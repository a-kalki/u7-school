# Troubleshoot — База известных проблем

## Директива

Ты ДОЛЖЕН следовать этому протоколу при столкновении с любой неожиданной ошибкой или странным поведением инструментов/библиотек.

## Инструкция для агента

### Когда читать
- При столкновении с **неожиданной ошибкой** (компиляции, runtime, тестов, линтера).
- При **странном поведении** библиотеки или инструмента.

### Как использовать
1. Найди запись в таблице ниже по тегу или описанию.
2. Перейди по ссылке → прочитай файл → примени решение.
3. Если запись была найдена и применена **Инкрементируй счётчики:**
   - `Всего` (глобальный) — **всегда** +1.
   - `Встреч` (у записи) — **всегда** +1.
   - `Помогло` (у записи) — +1 только если решение сработало.
4. Если запись в таблице **не найдена**:
   - реши проблему;
   - следуй инструкциям по добавлению записи.

### Как добавить новую запись
После решения проблемы, если задача заняла много ресурсов и решение может пригодиться в будущем:

1. Создай новый файл в `conductor/code_styleguides/troubleshoots/` с именем `<tag>-<short-slug>.md`.
2. Заполни: заголовок, симптомы, причина, решение.
3. **Добавь строку** в таблицу реестра:
   - `Встреч: 0`, `Помогло: 0`
   - `Добавлено на`: текущее значение `Всего`
4. **НЕ инкрементируй** `Всего` при добавлении записи — только при использовании.
5. В финальном отчёте задания сообщи пользователю о добавленной записи.


### Структура нового файла
```markdown
# <Библиотека>: <Краткое описание>

- **Симптомы:** ...
- **Причина:** ...
- **Решение:** ... (с примерами кода)
```

### Обслуживание

**Триггер ревизии:** когда `Всего` становится кратен 10 (т.е. 10, 20, 30…), после инкремента проведи ревизию полезности.

Для каждой записи с `Помогло = 0` вычисли **возраст** = `Всего − Добавлено_на`.

| Возраст | Статус |
|---|---|
| `< 20` | 🟢 актуально (ещё рано судить) |
| `>= 20` | 🟡 на заметке (сомнительная полезность) |
| `>= 40` | 🔴 кандидат на удаление |

- Записи с `Помогло > 0` — всегда 🟢, не трогай.
- Если есть записи с 🔴 — сообщи пользователю в итоговом отчёте, предложи удалить.
- При удалении: убери строку из реестра и удали файл из `troubleshoots/`.

Если `Всего` кратен 100 (100, 200, 300) в итоговом отчете предложи пользователю сделать расширенную ревизию полезности включая записи с `Помогло > 0`. При согласии пользователя сделай анализ полезности записей по метрикам. Дай статистику и твои выводы. Жди дальнейших инструкции.
