# Функция как единица программы синтаксис параметры и return

**Краткое содержание:**
Этот фрагмент видео-урока знакомит с фундаментальным понятием функции в программировании, объясняя её назначение как инструмента для создания сложных программ, основные правила и синтаксис объявления, а также концепции параметров, возвращаемых значений и различных терминов, связанных с функциями.

### Подробный конспект

#### 1. Что такое функция?
Функция — это первый инструмент, помогающий составлять сложные программы. Используя аналогию с изучением языка:
*   Простые части кода (переменные, операции) — это **буквы**.
*   Блоки `if` и циклы — это **слоги**.
*   **Функция** — это уже полноценное **слово**.

**Синтаксис функции:**
Функция объявляется с использованием оператора `function`, за которым следует имя функции, список параметров в круглых скобках (может быть пустым или содержать параметры), и тело функции, заключенное в фигурные скобки.

```javascript
function functionName ([...parameters]) {
  functionCode
}
```

**Эвристические правила (общие для программирования):**
1.  **Функция выполняет действие:** Это блок кода, имеющий своё имя, который умеет *что-то делать*.
    *   Имя функции должно быть глаголом, отражающим её действие.
2.  **Ответственность за один результат:** Функция должна быть ответственна только за один результат или одно действие.
3.  **Имя функции говорит, что она делает:** Имя функции должно явно говорить, какой результат мы получим (что она делает).

#### 2. Вызов (выполнение) функции

Чтобы запустить код, находящийся внутри функции, её необходимо вызвать по имени.

**Пример простой функции (без параметров):**
Эта функция демонстрирует базовый синтаксис и выполнение.

```javascript
function printHelloWorldTimes() {
  for (let i = 0; i < 3; i += 1) {
    console.log(`${i}: Hello world!`);
  }
}
```

**Вызов функции:**
Функцию можно вызывать (выполнять) любое количество раз, указывая её имя и круглые скобки.

```javascript
printHelloWorldTimes();
// Ожидаемый вывод в консоль:
// 0: Hello world!
// 1: Hello world!
// 2: Hello world!
```

**Объяснение выполнения:**
При вызове `printHelloWorldTimes();` интерпретатор переходит к объявлению функции и начинает выполнять код в её теле. В данном случае, это цикл `for`, который трижды выводит "Hello world!" с индексом итерации.

#### 3. Параметризованная (гибкая) функция

Простая функция, как показано выше, негибкая, так как всегда выполняет одно и то же действие. Для повышения гибкости используются **параметры**.

**Пример параметризованной функции:**
Эта функция принимает два параметра: `text` (что печатать) и `count` (сколько раз печатать).

```javascript
function printText(text, count) {
  for (let i = 0; i < count; i += 1) {
    console.log(`${i}: ${text}`);
  }
}
```

**Вызов параметризованной функции:**
При вызове функции мы передаем **аргументы** в круглых скобках, которые соответствуют параметрам.

```javascript
printText('Vivat!', 5);
// Ожидаемый вывод в консоль:
// 0: Vivat!
// 1: Vivat!
// 2: Vivat!
// 3: Vivat!
// 4: Vivat!

printText('Vivat!', 2);
// Ожидаемый вывод в консоль:
// 0: Vivat!
// 1: Vivat!

printText('Vivat!', 1);
// Ожидаемый вывод в консоль:
// 0: Vivat!
```

**Важные замечания о параметрах:**
1.  **Количество параметров не ограничено:** Можно указывать столько параметров, сколько необходимо.
2.  **Типы параметров не ограничены:** В JavaScript параметры могут быть любого типа (строки, числа, объекты, булевы значения, даже другие функции).
3.  **Имена параметров должны быть явными:** Имена параметров должны явно говорить, что в них должно быть указано, для лучшей читаемости кода.

#### 4. Пример плохой функции (нарушение эвристических правил)

Этот пример демонстрирует функцию, которая нарушает правила хорошего именования.

```javascript
function pTxt(t, n) {
  for (let i = 0; i < n; i += 1) {
    console.log(`${i}: ${t}`);
  }
}
```

**Почему это плохая функция:**
*   **Непонятное имя функции (`pTxt`):** Неясно, что именно делает функция.
*   **Непонятные имена параметров (`t`, `n`):** Неясно, какие данные ожидаются и для чего.

Несмотря на то, что для компьютера эта функция будет работать точно так же, как и `printText(text, count)`, для разработчика она гораздо менее понятна и поддерживаема.

**Дополнительные примеры плохих именований (упомянуто):**
*   `function text(txt, cnt)` — тоже плохое имя, так как `text` это существительное, а имя функции должно быть глаголом.
*   `function reverseAndPrintText(text, count)` — функция, выполняющая два действия (переворачивает текст И печатает), что является нарушением принципа единой ответственности функции.

#### 5. Предикаты (функции, отвечающие на вопрос)

Функции не всегда что-то "делают" (выполняют действия). Иногда они отвечают на вопросы, возвращая булево значение (`true` или `false`). Такие функции называют **предикатами**.

**Пример функции-предиката:**

```javascript
function isString(text) {
  return typeof text === 'string';
}
```

**Другие примеры предикатов (упомянуто):**
*   `function canKiss(person)` — ответит, умеет ли переданная персона целоваться.
*   `function hasChildren(person)` — ответит, есть ли у персоны дети.

#### 6. Параметры по умолчанию

Параметры по умолчанию позволяют указать часто используемое значение для параметра прямо в объявлении функции. Тогда код, вызывающий функцию, не обязан его передавать.

**Пример функции с параметром по умолчанию:**

```javascript
function printText(text, count = 1) { // count по умолчанию равен 1
  for (let i = 0; i < count; i += 1) {
    console.log(text);
  }
}
```

**Вызов функции с параметром по умолчанию:**

```javascript
printText('Vivate!', 3);
// Ожидаемый вывод в консоль (count будет 3):
// Vivate!
// Vivate!
// Vivate!

printText('JS is great!');
// Ожидаемый вывод в консоль (count по умолчанию будет 1):
// JS is great!
```

#### 7. Возвращаемое значение

Функции могут возвращать значение через оператор `return`. Это позволяет функции не только выполнять действия, но и предоставлять результат своей работы для дальнейшего использования. Оператор `return` также может использоваться, чтобы просто выйти из функции, тогда значение после оператора можно не ставить.

**Пример функции, возвращающей значение:**

```javascript
function repeatText(text, count) {
  let returnValue = ''; // Объявляем пустую строку для накопления результата
  for (let i = 0; i < count; i += 1) {
    returnValue += text; // Приклеиваем текст count раз
    // Или более явно: returnValue = returnValue + text;
  }
  return returnValue; // Возвращаем полученную строку
}
```

**Использование возвращаемого значения:**
Результат вызова функции с `return` можно присвоить переменной или использовать непосредственно.

```javascript
let consoleText = repeatText('Vivate!', 3);
console.log(consoleText);
// Ожидаемый вывод в консоль:
// Vivate!Vivate!Vivate!

console.log(repeatText('JS is great!', 2));
// Ожидаемый вывод в консоль:
// JS is great!JS is great!
```

### Ключевые выводы фрагмента

1.  **Функции — строительные блоки программ:** Они позволяют организовывать код в логические, переиспользуемые блоки, что важно для создания сложных программ.
2.  **Принципы чистоты функций:** Хорошая функция должна выполнять одно действие (быть глаголом), иметь понятное имя и ясные параметры.
3.  **Гибкость через параметры и возвращаемые значения:** Параметры делают функции универсальными, а возвращаемые значения позволяют им отдавать результаты своей работы.

### Термины

*   **Функция (Function):** Блок кода, имеющий имя, который может быть вызван для выполнения определенной задачи.
*   **Параметры функции (Parameters):** Переменные, которые объявляются в определении функции и используются для приема входных данных.
*   **Возвращаемое значение (Return Value):** Значение, которое функция возвращает после своего выполнения с помощью оператора `return`.
*   **Реализация (Implementation):** Внутреннее устройство функции, детали её работы.
*   **Чёрный ящик (Black Box):** Концепция, при которой пользователь функции знает только, что она принимает на вход и что возвращает на выходе, но не знает её внутреннего устройства.
*   **Контракт функции (Function Contract):** Соглашение между функцией и её вызывающим кодом, описывающее ожидаемые входные данные (параметры), гарантируемый вывод (возвращаемое значение) и побочные эффекты.
*   **Декомпозиция (Decomposition):** Процесс разбиения большой, сложной задачи на более мелкие, управляемые подзадачи, каждая из которых может быть реализована отдельной функцией.
*   **Предикаты (Predicates):** Функции, которые отвечают на вопрос, возвращая булево значение (`true` или `false`).
*   **Оператор `return`:** Ключевое слово, используемое для завершения выполнения функции и возврата значения (или просто для выхода из функции).
*   **Параметры по умолчанию (Default Parameters):** Значения, присваиваемые параметрам функции в её определении, которые будут использоваться, если вызывающий код не передаст соответствующий аргумент.
