# Функция как единица программы return предикаты и методы реализации

### Краткое содержание
Этот фрагмент видеоурока по JavaScript охватывает использование оператора `return` для возврата значений и досрочного выхода из функций, вводит концепцию "предикатов" как функций, отвечающих на вопросы, и объясняет разницу между "что функция должна делать" (задача) и "как она это делает" (реализация), демонстрируя несколько способов реализации одной и той же задачи.

### Подробный конспект

#### 1. Возвращаемое значение (`return`)
Функции могут возвращать значения с помощью оператора `return`. Это позволяет использовать результат выполнения функции в других частях кода, присваивать его переменным или передавать в другие функции.

**Пример кода:**
```javascript
function repeatText(text, count) {
  let returnValue = "";
  for (let i = 0; i < count; i = i + 1) {
    returnValue = returnValue + text;
  }
  return returnValue; // Возвращает объединенную строку
}

let consoleText = repeatText('Vivate!', 3);
console.log(consoleText); // Выведет: Vivate!Vivate!Vivate!

console.log(repeatText('JS is great!', 2)); // Выведет: JS is great!JS is great!
```
**Важные замечания:**
*   Значение, возвращаемое функцией, может быть присвоено переменной (как в `consoleText`) или использовано напрямую (как во втором `console.log`).
*   Когда поток выполнения достигает оператора `return`, функция немедленно завершает свою работу, и никакие инструкции после `return` не будут выполнены.

#### 2. Досрочный выход из функции с помощью `return`
Оператор `return` можно вставлять в любом месте функции и сколько угодно раз. Если поток выполнения добирается до `return`, функция немедленно выходит. Это полезно для валидации входных данных или обработки исключительных ситуаций.

**Пример кода (усовершенствованная `repeatText`):**
```javascript
function repeatText(text, count) {
  // Проверка входных данных: если текст пустой или количество меньше 1
  if (text === '' || count < 1) {
    return ''; // Выходим из функции и возвращаем пустую строку
  }

  let returnValue = "";
  for (let i = 0; i < count; i += 1) {
    returnValue += text;
  }
  return returnValue;
}

let s = repeatText('', 3);
console.log(s); // Выведет: "" (пустая строка, функция вышла досрочно)

console.log(repeatText('JS is great!', 0)); // Выведет: "" (пустая строка, функция вышла досрочно)
```
**Важные замечания:**
*   Функция **всегда** возвращает значение. Если оператор `return` не указан явно или указан без значения (например, `return;`), то после выполнения функции поток выполнения сам выходит из неё с возвращаемым значением `undefined`.
*   Пример функции `printText`, которая только печатает в консоль и не имеет явного `return`, будет неявно возвращать `undefined`.

**Пример кода (функция, неявно возвращающая `undefined`):**
```javascript
function printText(text, count) {
  for (let i = 0; i < count; i += 1) {
    console.log(text);
  }
  // Неявный return undefined
}

// Эквивалентно:
function printText(text, count) {
  for (let i = 0; i < count; i += 1) {
    console.log(text);
  }
  return; // Явный return undefined
}
```

#### 3. Предикаты — функции, отвечающие на вопрос
Функции, отвечающие на вопрос булевым ответом (`true` или `false`), называются "предикатами". Их названия часто начинаются с "is", "can", "has" и т.п.

**Пример кода:**
```javascript
function isString(text) {
  return typeof text === 'string'; // Возвращает true, если text - строка, иначе false
}

console.log(isString('true')); // Выведет: true
console.log(isString(true));   // Выведет: false
```
**Другие примеры предикатов:**
*   `function canKiss(person)` - Ответит: "Умеет ли переданная персона целоваться?"
*   `function hasChildrens(person)` - Ответит: "Есть ли у персоны дети?"

**Пример использования предиката для досрочного выхода:**
```javascript
function printText(text, count = 1) {
  if (!isString(text)) { // Если текст НЕ является строкой
    return; // Выходим из функции, ничего не печатая
  }
  for (let i = 0; i < count; i += 1) {
    console.log(text);
  }
}

printText(true, 3); // Ничего не выведет, так как true - не строка.
printText(5, 3);    // Ничего не выведет, так как 5 - не строка.
printText('5', 3);  /* Выведет:
                      5
                      5
                      5 */
```

#### 4. Функции в жизни (аналогия)
Функции в программировании можно сравнить с сервисами в реальной жизни (например, сантехник, автосервис).
1.  **Сервис умеет что-то делать.** (Функция выполняет определенное действие).
2.  **Обычно он делает что-то одно и делает это хорошо.** (Функция имеет одну конкретную ответственность).
3.  **Когда нам этот сервис понадобится, мы его вызываем.** (Мы вызываем функцию по её имени).
4.  **В результате его работы мы получаем какой-то результат.** (Функция возвращает результат или вызывает побочный эффект).

#### 5. Разработка функции это реализация
1.  **Задача (ЧТО нужно сделать):** Когда нам нужно разработать новую функцию, мы должны понимать, ЧТО функция должна сделать. Это задача, которая предстоит нам решить.
    *   Пример: для функции `repeatText` задача - вернуть текст, повторенный `count` раз.
2.  **Реализация (КАК это сделать):** Тело функции, или то, КАК он решает задачу, называется *реализацией*.
    *   **Вариантов реализации всегда очень много.** Какие-то будут удачные, какие-то не очень. Разные разработчики могут реализовать одну и ту же задачу по-разному.

**Пример кода (реализация `repeatText` с циклом):**
```javascript
function repeatText(text, count) {
  let returnValue = "";
  for (let i = 0; i < count; i += 1) {
    returnValue += text;
  }
  return returnValue;
}
```

**Пример кода (реализация `repeatText` через рекурсию):**
```javascript
function repeatText(text, count) {
  if (count < 1) return ''; // Базовый случай для завершения рекурсии
  return text + repeatText(text, count - 1); // Рекурсивный вызов
}

console.log(repeatText('Vivat!', 3)); // Выведет: Vivat!Vivat!Vivat!
```
Обе реализации выполняют одну и ту же *задачу*, но используют разные *способы* её решения.

#### 6. Реализация в жизни (аналогия)
**Задача (ЧТО нужно сделать):** попасть из дома в офис.
**Реализация (КАК я могу это сделать):**
*   Поехать на машине (и тут еще много вариантов: по какой улице, быстро или медленно, с остановками или без и т.д.)
*   Поехать на автобусе
*   Поехать на такси
*   Пойти пешком
*   Попросить друга довезти
*   Поработать удаленно (этот вариант не решает проблему "попасть в офис" напрямую, но решает лежащую в основе проблему, например, "выполнить работу", что может быть достигнуто удаленно).

Как и в программировании, в жизни для одной задачи существует множество вариантов реализации.

### Ключевые выводы фрагмента:
1.  Оператор `return` не только возвращает значение из функции, но и немедленно завершает её выполнение.
2.  Все JavaScript-функции всегда возвращают значение; если `return` не указан явно или без значения, функция возвращает `undefined`.
3.  Разработка функции включает в себя два этапа: определение *что* функция должна делать (задача) и написание кода, *как* она это делает (реализация), при этом для одной задачи может быть множество различных реализаций.

### Термины:
*   **`return`**: Оператор, используемый в функциях для возврата значения вызывающему коду и немедленного завершения выполнения функции.
*   **Предикат**: Функция, которая отвечает на вопрос булевым значением (`true` или `false`).
*   **Реализация**: Конкретный способ или код, который описывает, как функция выполняет поставленную перед ней задачу.
*   **Задача функции**: Описание того, *что* функция должна сделать, её цель или назначение.
*   **Рекурсия**: Метод, при котором функция вызывает сама себя для решения задачи.
