# Тернарный оператор

### Краткое содержание
В этом уроке мы углубленно изучаем оператор условного ветвления `if` в JavaScript, включая его обязательные и необязательные части, а также порядок выполнения. Кроме того, мы знакомимся с тернарным оператором, который предлагает более компактный синтаксис для простых условных выражений.

### Основные темы
1.  Структура оператора `if`, `else if`, `else`.
2.  Обязательные и необязательные части оператора `if`.
3.  Порядок и поток выполнения операторов `if-else if-else`.
4.  Различия между связанными `if-else if` и последовательными `if` операторами.
5.  Введение в тернарный оператор (условное выражение).

### Подробный конспект

#### 1. Структура оператора if, else if, else

Оператор `if` используется для выполнения блока кода только в том случае, если заданное условие истинно. Он может быть дополнен операторами `else if` для проверки дополнительных условий и `else` для выполнения кода, если ни одно из предыдущих условий не было истинным.

**Пример структуры (как в видео):**
```javascript
if (condition1) {
    code1
} [else if (condition2) {
    code2;
} else {
    code3;
}]
```

**Важное замечание:**
*   Оператор `if` является обязательным.
*   Блоки `else if` и `else` являются необязательными и могут быть использованы по желанию.
*   Нельзя использовать `else if` или `else` без предшествующего оператора `if`.
*   Можно добавлять множество блоков `else if` по мере необходимости.

**Пример допустимого использования (только `if`):**
```javascript
if (condition1) code1;
code2; // Этот код выполняется после if, если if выполняется или не выполняется
```
*Замечание: Фигурные скобки `{}` для тела `if` (и `else if`, `else`) могут быть опущены, если блок содержит только одно выражение. Однако для лучшей читаемости и предотвращения ошибок рекомендуется всегда использовать фигурные скобки.*

**Пример недопустимого использования (попытка использовать `else if` без `if`):**
```javascript
code1;
else if (condition1) code1; // Это вызовет синтаксическую ошибку
code2;
```
*Преподаватель демонстрирует это в консоли, где выдается ошибка `Uncaught SyntaxError: Unexpected token 'else'`.*

#### 2. Порядок и поток выполнения операторов if-else if-else

В цепочке `if-else if-else` выполняется только *первый* блок кода, условие которого истинно. После выполнения этого блока, интерпретатор выходит из всей цепочки условных операторов и продолжает выполнение кода, следующего за ней.

**Пример структуры и потока выполнения (как в видео):**
```javascript
if (condition1) {
    code1
} else if (condition2) {
    code2;
} else if (condition3) {
    code3;
} else {
    code4;
}
code5; // Этот код выполнится всегда, после того как выполнится один из блоков if/else if/else
```
**Важное замечание:** Если ни одно из условий `if` или `else if` не истинно, а блока `else` нет, то вся цепочка не выполнит ни одного блока кода.

**Демонстрация в консоли:**
Предположим, у нас есть переменная `i = 0`.
```javascript
let i = 0;
if (i < 0) { // i < 0 (0 < 0) -> false
    console.log(`${i} отрицательное число`);
} else if (i === 0) { // i === 0 (0 === 0) -> true. Этот блок выполнится.
    console.log(`${i} равно нулю`);
} else if (i === 1) { // Этот блок не будет проверен, так как предыдущий был true.
    console.log(`${i} равно одному`);
} else { // Этот блок также не будет проверен.
    console.log(`${i} число больше одного`);
}
```
**Вывод в консоли:**
```
0 равно нулю
```
*Это подтверждает, что выполняется только первый блок с истинным условием. После этого выполнение переходит к коду, следующему за всем блоком `if-else if-else`.*

#### 3. Различия между связанными if-else if и последовательными if операторами

Важно различать связанную цепочку `if-else if-else` и серию независимых операторов `if`.
*   **Связанная цепочка `if-else if-else`:** Только один блок кода будет выполнен.
*   **Последовательные `if` операторы:** Каждое условие `if` проверяется независимо. Если условие истинно, соответствующий блок кода выполняется. Может быть выполнено несколько блоков кода, если несколько условий истинны.

**Пример последовательных `if` (как в видео):**
```javascript
if (condition1) {
    code1
}
if (condition2) {
    code2;
}
code3; // Этот код выполнится после проверки всех if
```
**Демонстрация в консоли:**
Предположим, `let i = 0`.
```javascript
let i = 0;
if (i === 0) { // i === 0 (0 === 0) -> true. Этот блок выполнится.
    console.log(`${i} равен нулю`);
}
if (i >= 0) { // i >= 0 (0 >= 0) -> true. Этот блок выполнится.
    console.log(`${i} неотрицательное число`);
}
if (i < 0) { // i < 0 (0 < 0) -> false. Этот блок не выполнится.
    console.log(`${i} отрицательное число`);
}
```
**Вывод в консоли:**
```
0 равен нулю
0 неотрицательное число
```
*Здесь оба условия (`i === 0` и `i >= 0`) истинны, и поскольку это независимые операторы `if`, оба блока кода были выполнены.*

#### 4. Введение в тернарный оператор (условное выражение)

Тернарный оператор (`? :`) предоставляет компактный способ написания простых условных выражений и часто используется для присваивания значений переменным на основе условия.

**Синтаксис тернарного оператора:**
`condition ? exprIfTrue : exprIfFalse`

**Пример использования (как в видео):**
Представим, что нам нужно рассчитать скидку (`discount`) для клиента в зависимости от того, является ли он "хорошим клиентом" (`isGoodClient`).

**Обычная логика с `if-else`:**
```javascript
let discount;
if (isGoodClient) {
    discount = price * 0.15; // 15% скидка для хороших клиентов
} else {
    discount = price * 0.05; // 5% скидка для остальных
}
```

**Эквивалентный код с тернарным оператором:**
```javascript
let discount = isGoodClient ? price * 0.15 : price * 0.05;
```
*Преподаватель также показывает, что для читаемости тернарный оператор можно разбить на несколько строк:*
```javascript
let discount = isGoodClient ?
    price * 0.15 :
    price * 0.05;
```

**Как это работает:**
1.  **`condition`** (`isGoodClient`): Сначала проверяется условие.
2.  Если условие истинно (`true`), то вычисляется и возвращается значение **`exprIfTrue`** (`price * 0.15`).
3.  Если условие ложно (`false`), то вычисляется и возвращается значение **`exprIfFalse`** (`price * 0.05`).
4.  Результат выражения присваивается переменной `discount`.

### Ключевые выводы
1.  `if` является основой ветвления; `else if` и `else` — дополнения, используемые с `if`.
2.  В цепочке `if-else if-else` выполняется только *первый* блок кода, условие которого истинно, после чего вся цепочка прекращает работу.
3.  Независимые операторы `if` проверяются поочередно, и может быть выполнено несколько их блоков.
4.  Тернарный оператор (`? :`) — это компактный способ присвоить значение переменной на основе простого условия, эквивалентный короткому `if-else`.

### Термины
*   **Оператор ветвления (Conditional Statement)**: Конструкция в программировании, которая позволяет выполнять различные блоки кода в зависимости от истинности или ложности заданных условий.
*   **`if`**: Оператор ветвления, который выполняет блок кода, если его условие истинно.
*   **`else if`**: Дополнительный оператор ветвления, который проверяет условие, если предыдущие `if` или `else if` были ложны.
*   **`else`**: Оператор ветвления, который выполняет блок кода, если все предыдущие условия в цепочке `if-else if` были ложны.
*   **Интерпретатор (Interpreter)**: Компьютерная программа, которая напрямую выполняет инструкции, написанные на языке программирования, без предварительной компиляции.
*   **Тернарный оператор (Ternary Operator)**: Условный оператор, который принимает три операнда и возвращает одно из двух значений в зависимости от истинности условия. В JavaScript его синтаксис: `condition ? valueIfTrue : valueIfFalse`.
*   **Синтаксис (Syntax)**: Набор правил, определяющих, как должны быть написаны выражения и операторы в языке программирования, чтобы они были правильно интерпретированы.
*   **Эквивалентный код (Equivalent Code)**: Два или более фрагмента кода, которые, несмотря на различия в написании, дают одинаковый результат или имеют одинаковое поведение при выполнении.
