# Функции: аргументы, передача по значению и ссылке

### Часть 2: Передача данных в функции и типы данных

В этой части рассматриваются механизмы передачи аргументов в функции и влияние типов данных на это поведение.

**Ключевые идеи:**
*   **Параметры vs. Аргументы:** Параметры — это имена, указанные в определении функции; аргументы — фактические значения, передаваемые при вызове.
*   **Пропуск аргументов:** Если аргумент для параметра не передан, этот параметр получит значение `undefined`. JavaScript не выдаст ошибку на этапе компиляции, но проблемы могут возникнуть во время выполнения.
*   **Передача по значению (`pass-by-value`):** Для простых типов данных (числа, строки, булевы, `null`, `undefined`, `symbol`, `bigint`) в функцию передается *копия значения*. Изменения копии внутри функции не влияют на оригинальную переменную.
*   **Передача по ссылке (`pass-by-reference`) / Передача ссылки по значению:** Для сложных типов данных (объекты, массивы, функции) в функцию передается *ссылка* на объект в памяти. Изменения объекта через эту ссылку внутри функции **будут видны** извне.

**Примеры кода:**

```javascript
// Пропуск аргументов
function printText(text, count) {
    console.log(`text = ${text}, count = ${count}`);
    // Цикл for не будет выполнен, если count === undefined
}
printText('Hello'); // -> text = Hello, count = undefined
printText();        // -> text = undefined, count = undefined

// Передача простых типов (по значению)
let originalNumber = 10;
function changeNumber(num) {
    num = num + 5; // Изменяется копия
    return num;
}
console.log(changeNumber(originalNumber)); // -> 15
console.log(originalNumber);               // -> 10 (оригинал не изменился)

// Передача сложных типов (по ссылке)
let originalObject = { value: 10 };
function changeObject(obj) {
    obj.value = obj.value + 5; // Изменяется свойство оригинального объекта
}
changeObject(originalObject);
console.log(originalObject.value); // -> 15 (оригинал изменился)
```
