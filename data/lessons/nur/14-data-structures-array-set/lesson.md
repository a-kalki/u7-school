# Структура данных: последовательность (Array, Set)

## Массив (Array)

Массив — упорядоченная коллекция данных с числовыми индексами. В отличие от объектов, предоставляет методы управления порядком элементов.

### Создание
```javascript
let arr = [];                // литерал (основной способ)
let arr = new Array();       // конструктор (редко)
let fruits = ["Яблоко", "Апельсин", "Слива"];
```

### Доступ к элементам
- `arr[0]` — первый элемент (индексация с нуля)
- `arr[arr.length - 1]` — последний элемент
- `arr.at(-1)` — последний элемент (новый синтаксис, отрицательный индекс отсчитывает с конца)
- `arr.length` — количество элементов
- Присваивание: `arr[2] = 'Груша'` заменяет элемент
- `arr[3] = 'Лимон'` добавляет новый, если длина была <= 3

В массиве могут храниться элементы любых типов (строки, объекты, функции).

### Основные методы

**Работа с концом массива (быстрые — O(1)):**
- `arr.push(el)` — добавляет элемент(ы) в конец, возвращает новую длину
- `arr.pop()` — удаляет последний элемент, возвращает его

**Работа с началом массива (медленные — O(n), требуется переиндексация):**
- `arr.shift()` — удаляет первый элемент, сдвигая все остальные
- `arr.unshift(el)` — добавляет элемент(ы) в начало

### Внутреннее устройство
- Массив — подвид объекта; `typeof [] === 'object'`
- Копируется по ссылке
- Движок JS хранит элементы в непрерывной области памяти
- «Неправильное» использование (нечисловые свойства, «дыры») отключает оптимизации

### Перебор
```javascript
// Классический for
for (let i = 0; i < arr.length; i++) { ... }

// for..of — по значениям (рекомендуется для массивов)
for (let item of arr) { ... }

// for..in — технически работает, но не рекомендуется (перебирает все свойства)
```

### Очередь и стек
- **Очередь (FIFO):** push + shift
- **Стек (LIFO):** push + pop

## Множество (Set)

Set — коллекция **уникальных** значений (без ключей). Каждое значение может появляться только один раз.

### Создание
```javascript
let set = new Set();                    // пустое
let set = new Set(["a", "b", "a"]);     // Set { "a", "b" } — дубликаты удалены
```

### Методы
- `set.add(value)` — добавляет значение (если уже есть — ничего не делает), возвращает set (можно чейнить)
- `set.delete(value)` — удаляет, возвращает `true` если было
- `set.has(value)` — возвращает `true/false`
- `set.clear()` — очищает
- `set.size` — количество элементов

### Перебор
```javascript
// for..of
for (let value of set) { ... }

// forEach — сигнатура (value, valueAgain, set) для совместимости с Map
set.forEach(value => { ... });

// keys(), values(), entries() — для совместимости с Map
```

### Типичные применения
- Удаление дубликатов из массива: `[...new Set(arr)]`
- Хранение уникальных посетителей/ID
- Быстрая проверка наличия (`has`)

### Сравнение ключей
Использует алгоритм SameValueZero (почти как `===`, но `NaN` равен `NaN`).

## Сравнение Array и Set

| Характеристика | Array | Set |
|---|---|---|
| Упорядоченность | Да | Да (порядок вставки) |
| Доступ по индексу | Да (`arr[i]`) | Нет |
| Уникальность | Не гарантирует | Гарантирует |
| Поиск элемента | `indexOf`/`find` — O(n) | `has` — O(1) |
| Использование | Упорядоченные списки | Коллекции без дубликатов |
