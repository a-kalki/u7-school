# Типы данных исключения и особые случаи

**Краткое содержание:**
Этот урок по JavaScript знакомит с концепцией типов данных, объясняет, как интерпретатор обрабатывает различные операции в зависимости от типов операндов, и вводит понятие "исключения" — ошибки, возникающей при некорректном использовании типов, что приводит к остановке выполнения кода.

**Основные темы:**
*   Что такое типы данных
*   Поведение оператора сложения (`+`) с разными типами
*   Арифметические операции со строками
*   Методы строк
*   Что такое исключение (Exception)
*   Пример возникновения исключения и остановки потока выполнения

#### Подробный конспект:

##### 1. Что такое типы данных
Интерпретатор, выполняя код, оперирует данными. Ему необходимо знать, какой тип имеют данные, чтобы правильно выполнять операции. Разные типы данных ведут себя по-разному при одних и тех же операторах.

##### 2. Поведение оператора сложения (`+`) с разными типами

*   **Сложение двух чисел:**
    Когда оба операнда являются числами, оператор `+` выполняет арифметическое сложение.
    ```javascript
    5 + 5 // Результат: 10
    ```

*   **Сложение строки и числа (или двух строк):**
    Если один из операндов оператора `+` является строкой, интерпретатор выполняет **конкатенацию** (склеивание строк). В этом случае числовой операнд преобразуется в строку.
    ```javascript
    '5' + '5' // Результат: "55"
    '5' + 5   // Результат: "55"
    ```
    *Замечание:* Это демонстрирует, что JavaScript может неявно преобразовывать типы (type coercion) в зависимости от контекста операции.

##### 3. Арифметические операции со строками
Не все арифметические операции со строками приводят к конкатенации или ошибкам. Иногда JavaScript пытается преобразовать строку в число, чтобы выполнить операцию.

*   **Деление строки на число:**
    Если строка не может быть преобразована в число, результатом будет `NaN` (Not a Number).
    ```javascript
    'S' / 5 // Результат: NaN
    ```
    Если строка содержит числовое значение, она будет преобразована в число, и операция будет выполнена.
    ```javascript
    '5' / 5 // Результат: 1
    ```
    *Замечание:* Подробно о приведении типов данных будет рассказано в отдельном видео.

##### 4. Методы строк
Строки в JavaScript имеют встроенные методы (функции), которые позволяют выполнять различные операции со строками.

*   **Метод `toUpperCase()`:**
    Этот метод преобразует все символы строки в верхний регистр.
    ```javascript
    'apple'.toUpperCase() // Результат: "APPLE"
    ```
    *Замечание:* Вызов метода осуществляется через точку после строковой переменной или строкового литерала. Методы будут изучаться подробнее позже.

##### 5. Что такое исключение (Exception)
**Исключение (exception)** – это событие, которое возникает во время выполнения программы и нарушает её нормальный ход.
Интерпретатор **отказывается выполнять** код дальше, если встречается с исключением, то есть ошибкой. Поток выполнения останавливается, и никакие последующие строки кода после точки возникновения исключения не будут выполнены.

*   **Пример возникновения исключения:**
    Если мы попытаемся вызвать метод, который существует только для строк (например, `toUpperCase()`), на переменной числового типа, интерпретатор выдаст ошибку `TypeError`.
    Допустим, у нас есть переменная `bar`, которая содержит число `5`.
    ```javascript
    let bar = 5;
    bar.toUpperCase(); // Вызовет исключение TypeError
    // TypeError: bar.toUpperCase is not a function
    ```
    Сообщение об ошибке `TypeError: bar.toUpperCase is not a function` означает, что интерпретатор не может найти метод `toUpperCase` для числового типа данных.

##### 6. Пример возникновения исключения и остановки потока выполнения
Рассмотрим следующий пример кода:
```javascript
let personOld = '24';
let newOld = personOld + 1;
alert(`У вас день рождения, теперь вам: ${newOld.toUpperCase()}`);
const index = 1;
index.toUpperCase();
alert('after exception');
```

**Разбор выполнения кода:**
1.  `let personOld = '24';`
    Создается переменная `personOld` со строковым значением `"24"`.
2.  `let newOld = personOld + 1;`
    `personOld` (строка "24") конкатенируется с `1` (число). Результатом будет строка `"241"`, которая присваивается `newOld`.
3.  `alert(`У вас день рождения, теперь вам: ${newOld.toUpperCase()}`);`
    Вызывается функция `alert`, которая отображает диалоговое окно. Внутри строки используется интерполяция: `newOld` (которая является `"241"`) преобразуется в верхний регистр с помощью `toUpperCase()`. Так как `"241"` содержит только цифры, `toUpperCase()` вернёт ту же строку `"241"`.
    В итоге `alert` покажет: "У вас день рождения, теперь вам: 241".
    (На этом этапе код успешно выполняется, и пользователь видит окно оповещения).
4.  `const index = 1;`
    Создается переменная `index` с числовым значением `1`.
5.  `index.toUpperCase();`
    Пытаемся вызвать метод `toUpperCase()` на переменной `index`, которая имеет числовой тип. Поскольку метод `toUpperCase()` определен только для строк, а не для чисел, здесь произойдет **исключение**.
    В консоли будет выведено сообщение об ошибке: `TypeError: index.toUpperCase is not a function`.
6.  `alert('after exception');`
    Из-за того, что на предыдущей строке произошло исключение, поток выполнения кода **остановился**. Эта строка кода **не будет выполнена**, и пользователь не увидит `alert` с сообщением "after exception".

#### Ключевые выводы:
1.  **Типы данных важны:** Понимание типов данных критически важно для разработчика, чтобы писать корректный код, и для интерпретатора, чтобы правильно его выполнять.
2.  **Поведение операторов зависит от типов:** Один и тот же оператор (например, `+`) может выполнять разные действия (арифметическое сложение или конкатенацию) в зависимости от типов операндов.
3.  **Некорректное использование типов приводит к исключениям:** Попытка выполнить операцию или вызвать метод, который несовместим с типом данных, вызовет `exception`.
4.  **Исключения останавливают выполнение:** При возникновении исключения интерпретатор останавливает дальнейшее выполнение кода, и последующие инструкции игнорируются.

#### Термины:
*   **Типы данных (Data Types)**: Категории данных (например, числа, строки, булевы значения), которые определяют их свойства и доступные операции.
*   **Интерпретатор (Interpreter)**: Программа, которая читает и выполняет исходный код построчно, переводя его в машинный код во время выполнения.
*   **Конкатенация (Concatenation)**: Операция объединения нескольких строк в одну. В JavaScript обычно выполняется оператором `+` при работе со строками.
*   **NaN (Not a Number)**: Специальное числовое значение в JavaScript, обозначающее "не число". Возникает, например, при попытке выполнить математическую операцию с операндом, который не может быть преобразован в число.
*   **Метод (Method)**: Функция, связанная с объектом, которая выполняет определённое действие с данными этого объекта.
*   **Exception (Исключение)**: Ошибка, возникающая во время выполнения программы, которая прерывает нормальный ход выполнения кода и, если не будет обработана, приводит к аварийной остановке программы.
*   **Поток выполнения (Execution Flow)**: Последовательность операций или инструкций, которые выполняет программа. При исключении этот поток останавливается.
*   **Приведение типов (Type Coercion)**: Автоматическое или явное преобразование значений из одного типа данных в другой. (В JavaScript часто происходит неявно).
