# Область видимости и время жизни

### Часть 1: Что такое область видимости и время жизни

**Область видимости (scope)** — это часть программы, в которой переменная существует и доступна. Попытка обратиться к переменной за пределами её области видимости приводит к ошибке `ReferenceError`.

**Время жизни** переменной — период от момента создания до уничтожения. Переменная создаётся при входе в её область видимости и уничтожается при выходе.

В JavaScript три уровня области видимости:

| Уровень | Где объявлена | Когда живёт | Для каких объявлений |
|---|---|---|---|
| **Глобальная** | Вне функций и блоков | От запуска до завершения программы | `let`, `const`, `var` |
| **Функциональная** | Внутри функции | От вызова до возврата из функции | `var`, `let`, `const`, параметры |
| **Блочная** | Внутри `{ }` | От входа в блок до выхода | Только `let` и `const` |

### Часть 2: Глобальная область видимости

Переменная, объявленная вне любых функций и блоков — **глобальная**. Видна отовсюду. Живёт от запуска программы до её завершения.

```javascript
const appName = 'Калькулятор зарплаты';
let version = '1.0';
var legacyFlag = true;         // var тоже глобален, но не используй

function showInfo() {
  console.log(appName);        // 'Калькулятор зарплаты' — видна внутри функции
  console.log(version);        // '1.0'
}

showInfo();
console.log(appName);          // 'Калькулятор зарплаты' — и снаружи тоже
```

Глобальные переменные удобны для конфигурации приложения, но злоупотреблять ими опасно: в программе из сотен строк легко случайно перезаписать чужую глобальную переменную и получить трудноуловимую ошибку.

### Часть 3: Функциональная область видимости

Переменная, объявленная **внутри функции** (включая параметры), видна только внутри этой функции. Снаружи — ошибка. Живёт от момента вызова функции до `return` (или конца тела функции).

```javascript
function calculateIPN(salary) {
  const rate = 0.1;            // видна только внутри calculateIPN
  const ipn = salary * rate;
  return ipn;
}

console.log(calculateIPN(500000));  // 50000
console.log(rate);                  // ReferenceError: rate is not defined
console.log(salary);                // ReferenceError: salary is not defined
```

Параметры функции (`salary`) — это тоже локальные переменные функции. Их время жизни — один вызов.

**Вложенные функции.** Внутренняя функция видит переменные внешней:

```javascript
function printEmployeeInfo(name, salary) {
  const bonus = 5000;

  // Внутренняя функция — видит параметры и переменные внешней
  function formatLine() {
    const total = salary + bonus;
    return name + ': ' + total + ' тенге';
  }

  console.log(formatLine());  // результат используем здесь же
}

printEmployeeInfo('Алия', 400000);  // 'Алия: 405000 тенге'
console.log(bonus);                 // ReferenceError — bonus внутри функции
console.log(name);                  // ReferenceError — параметр внутри функции
```

Внутренняя `formatLine` имеет доступ к `name`, `salary` и `bonus` из внешней `printEmployeeInfo`. Но снаружи `printEmployeeInfo` ничего не видно.

**var в функции vs блоке.** `var` всегда создаёт **функциональную** область видимости. Блоки `{ }` для `var` прозрачны:

```javascript
function demoVar() {
  {
    var x = 10;
  }
  console.log(x);   // 10 — var «пробил» блок, но НЕ вышел за пределы функции
}

demoVar();
console.log(x);     // ReferenceError — за пределы функции var не выходит
```

А `let` и `const` внутри той же функции были бы ограничены блоком — снаружи блока `ReferenceError`.

**Почему var не нужен:**

```javascript
// Классический баг с var в цикле:
for (var i = 0; i < 3; i++) {
  // ...
}
console.log(i);  // 3 — переменная «утекла» из цикла

// let решает проблему:
for (let j = 0; j < 3; j++) {
  // ...
}
console.log(j);  // ReferenceError — переменная осталась в цикле
```

**Всегда используй `let` и `const`.** `var` оставлен для обратной совместимости и создаёт трудноуловимые ошибки.

### Часть 4: Сборщик мусора

Когда переменная выходит из области видимости, она становится недоступной и больше не нужна. **Сборщик мусора (Garbage Collector, GC)** — встроенный механизм JavaScript, который автоматически находит и удаляет такие переменные из памяти.

```javascript
function processData() {
  const hugeArray = new Array(1000000);  // заняли много памяти
  const result = hugeArray.length;
  return result;
}  // ← здесь функция завершилась, hugeArray больше не нужен

const size = processData();
// Сборщик мусора заметит, что hugeArray недоступен, и освободит память
```

Твоя ответственность — не держать ссылки на ненужные данные. Ответственность сборщика мусора — автоматически подчистить то, что стало недоступно. Если переменная вышла из области видимости и на неё никто не ссылается — GC её удалит. Ты не управляешь памятью вручную, в отличие от языков вроде C или C++.

**Возврат объекта из функции.** Когда функция создаёт объект и возвращает его, время жизни объекта зависит от того, сохранил ли вызывающий код ссылку:

```javascript
function createEmployee(name, salary) {
  const employee = {        // объект создан в куче (heap)
    name: name,
    salary: salary,
  };
  return employee;           // возвращаем ссылку на объект
}

const aliya = createEmployee('Алия', 500000);
// aliya держит ссылку на объект → объект жив, GC его не тронет

let bek = createEmployee('Бек', 350000);
bek = null;                  // ссылка потеряна → GC удалит объект Бека

createEmployee('Чингиз', 420000);
// Результат никуда не сохранён → объект недоступен → GC удалит
```

Переменные `name` и `salary` внутри функции живут один вызов и удаляются GC после `return`. Но сам **объект** `employee` продолжает жить, пока на него есть хотя бы одна ссылка снаружи. Как только ссылок не остаётся — GC его удаляет.

**Практический вывод:** объявляй переменные как можно ближе к месту использования, в самой узкой области видимости. Тогда они быстрее станут недоступны, и GC быстрее освободит память.

### Часть 5: Блочная область видимости

`let` и `const` создают **блочную** область видимости: переменная живёт только внутри ближайшего блока `{ ... }`. Это касается любого блока: тела `if`, `for`, `while`, просто фигурных скобок.

```javascript
const city = 'Алматы';           // глобальная

{
  const district = 'Бостандык';   // блочная
  console.log(city);              // 'Алматы' — глобальная видна внутри
  console.log(district);          // 'Бостандык'
}

console.log(city);                // 'Алматы'
console.log(district);            // ReferenceError — блок закрыт, переменная «умерла»
```

**Время жизни в цикле.** На каждой итерации `for` с `let` создаётся **новая** переменная:

```javascript
for (let i = 1; i <= 3; i++) {
  const doubled = i * 2;
  console.log(`Итерация ${i}: удвоенное = ${doubled}`);
}
// Итерация 1: удвоенное = 2
// Итерация 2: удвоенное = 4
// Итерация 3: удвоенное = 6

console.log(i);       // ReferenceError — переменная «умерла» после цикла
console.log(doubled); // ReferenceError
```

Это защищает данные: `doubled` из итерации 1 не «загрязняет» итерацию 2, потому что это **разные** переменные в разных итерациях.

**Вложенные блоки.** Внутренний блок видит переменные внешних блоков. Внешний — не видит внутренние:

```javascript
const country = 'Казахстан';

{
  const city = 'Алматы';

  {
    const street = 'Абая';
    console.log(street);   // 'Абая' — своя
    console.log(city);     // 'Алматы' — из внешнего блока
    console.log(country);  // 'Казахстан' — глобальная
  }

  console.log(city);       // 'Алматы' — ок
  console.log(street);     // ReferenceError — внутренний блок уже закрыт
}
```

### Часть 6: Сравнение всех трёх видов

```javascript
// ГЛОБАЛЬНАЯ — видна везде, живёт всю программу
const appName = 'Калькулятор';

// ФУНКЦИОНАЛЬНАЯ — видна внутри функции, живёт один вызов
function calc(salary) {
  const ipn = salary * 0.1;    // salary и ipn — локальные для calc
  return ipn;
}

// БЛОЧНАЯ — видна внутри {}, живёт пока открыт блок
if (true) {
  let temp = 'временная';      // temp умрёт после закрывающей }
  console.log(temp);           // 'временная'
}
console.log(temp);             // ReferenceError

// Сборщик мусора: temp, ipn и salary уже недоступны — GC может их удалить
```

### Ключевые идеи

- **Область видимости** определяет, где переменная доступна. Вне своей области — `ReferenceError`.
- **Время жизни** — от создания до уничтожения. Глобальные живут всю программу, функциональные — один вызов, блочные — пока открыт `{ }`.
- **Глобальная** — вне всего, доступна отовсюду. Используй умеренно.
- **Функциональная** — внутри функции (включая параметры). `var` ограничен функцией, а не блоком.
- **Блочная** (`let`, `const`) — внутри `{ }`. Каждая итерация `for` с `let` создаёт новую переменную.
- **Сборщик мусора** — автоматически удаляет переменные, когда они выходят из области видимости и становятся недоступны. Объявляй переменные в самой узкой области.
- `var` создаёт функциональную область и игнорирует блоки — **не используй**.
- Область видимости — не ограничение, а **защита**: данные не «утекают» и не конфликтуют.
