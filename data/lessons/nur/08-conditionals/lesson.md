# Ветвление

**Краткое содержание**
Этот урок знакомит с концепцией ветвления в программировании на JavaScript. Он объясняет, как использовать операторы `if`, `else if` и `else` для создания кода, который принимает решения и выполняет различные действия в зависимости от заданных условий, приводя аналогии из повседневной жизни и демонстрируя их синтаксис и поведение в коде.

**Основные темы**
*   Ветвление в жизни (аналогия)
*   Оператор `if`
*   Оператор `else if`
*   Оператор `else`
*   Оператор `if` без блока кода (однострочный)

---

### Подробный конспект

#### Ветвление в жизни
*   **Концепция:** Ветвление – это процесс принятия решений, основанный на определенных условиях. Мы постоянно используем его в повседневной жизни, даже не задумываясь об этом.
*   **Аналогии:**
    *   **Сборы на работу:**
        *   Если (сегодня рабочий день И (время 8 часов утра)), то Пора собираться на работу;
        *   иначе если (сегодня рабочий день И (время после 8 часов утра)), то надо бежать на работу;
        *   иначе еще рано, можно еще поспать!
    *   **Поездка на автомобиле:**
        *   Если (светофор красный ИЛИ желтый), то Остановиться перед светофором;
        *   иначе Проехать перекресток.
*   **Замечание:** Программы могут выполнять "что-то" в зависимости от того, произойдет ли какое-то условие или нет, точно так же, как мы принимаем решения в жизни.

#### Оператор if
*   **Концепция:** Оператор `if` используется для выполнения блока кода, если заданное условие (condition) истинно (`true`).
*   **Синтаксис:**
    ```javascript
    code0;
    if (condition1) {
        code1;
        code2;
    }
    code3;
    ```
*   **Объяснение:**
    *   Интерпретатор выполняет `code0`.
    *   Затем проверяется `condition1` (условие в скобках). Это условие должно возвращать булево значение (`true` или `false`).
    *   Если `condition1` истинно (`true`), выполняется блок кода внутри фигурных скобок (`code1` и `code2`). После этого выполнение продолжается с `code3`.
    *   Если `condition1` ложно (`false`), блок кода внутри фигурных скобок пропускается, и выполнение сразу переходит к `code3`.
*   **Пример кода (и демонстрация в консоли):**
    ```javascript
    let i = 0; // Инициализируем i
    if (i > 0) { // Условие: i больше 0? (0 > 0) -> false
        console.log(`${i} положительное число`); // Этот блок пропустится
    }
    i += 1; // i становится 1
    if (i > 0) { // Условие: i больше 0? (1 > 0) -> true
        console.log(`Повторно: ${i} положительное число`); // Этот блок выполнится
    }
    i -= 1; // i становится 0
    console.log(`i теперь равен ${i}`); // Выведет текущее значение i
    ```
*   **Ожидаемый вывод в консоли:**
    ```
    Повторно: 1 положительное число
    i теперь равен 0
    ```
*   **Важное замечание от преподавателя:** При демонстрации в консоли, из-за повторного объявления `let i = 0;` несколько раз, может возникнуть ошибка `Identifier 'i' has already been declared`. Чтобы избежать этого (и для демонстрации в видео), преподаватель временно сохранял значение `i` в `old_i` и затем восстанавливал его. В реальном коде такая ошибка указывает на неправильное объявление переменной в той же области видимости. `console.log()` используется для вывода информации в консоль разработчика, в отличие от `alert()`, который показывает всплывающее окно.

#### Оператор else if
*   **Концепция:** `else if` используется для проверки дополнительного условия, если предыдущее условие (`if` или предыдущий `else if`) оказалось ложным. Можно иметь несколько операторов `else if` в цепочке.
*   **Синтаксис:**
    ```javascript
    if (condition1) {
        code1;
    } else if (condition2) {
        code2;
    }
    code3;
    ```
*   **Объяснение:**
    *   Если `condition1` истинно, выполняется `code1`, затем `code3`.
    *   Если `condition1` ложно, то проверяется `condition2`.
    *   Если `condition2` истинно, выполняется `code2`, затем `code3`.
    *   Если оба условия (`condition1` и `condition2`) ложны, то оба блока кода пропускаются, и выполнение сразу переходит к `code3`.
*   **Пример кода:**
    ```javascript
    let i = 1; // Инициализируем i
    if (i === 0) { // Условие 1: i равно 0? (1 === 0) -> false
        console.log(`${i} равен нулю`);
    } else if (i > 0) { // Условие 2: i больше 0? (1 > 0) -> true
        console.log(`${i} положительное число`); // Этот блок выполнится
    }
    i += 1; // i становится 2
    ```
*   **Ожидаемый вывод в консоли:**
    ```
    1 положительное число
    ```

#### Оператор else
*   **Концепция:** Оператор `else` является завершающим звеном в цепочке `if-else if`. Его блок кода выполняется, если все предыдущие условия (`if` и все `else if`) оказались ложными. `else` не имеет собственного условия в скобках.
*   **Синтаксис:**
    ```javascript
    if (condition1) {
        code1;
    } else if (condition2) {
        code2;
    } else {
        code3;
    }
    code4;
    ```
*   **Объяснение:**
    *   Если `condition1` истинно, выполняется `code1`, затем `code4`.
    *   Если `condition1` ложно, а `condition2` истинно, выполняется `code2`, затем `code4`.
    *   Если `condition1` и `condition2` ложны, то выполняется `code3` (блок `else`), затем `code4`.
*   **Важное замечание:** Можно считать, что оператор `else` имеет всегда истинное условие и выполняется ВСЕГДА, если условия `if`, `else if` дали ложное утверждение (т.е. `condition1 === false` и `condition2 === false`).
*   **Пример кода:**
    ```javascript
    let i = -1; // Инициализируем i
    if (i === 0) { // Условие 1: i равно 0? (-1 === 0) -> false
        console.log(`${i} равен нулю`);
    } else if (i > 0) { // Условие 2: i больше 0? (-1 > 0) -> false
        console.log(`${i} положительное число`);
    } else { // Ни одно из предыдущих условий не выполнилось, выполняется else
        console.log(`${i} отрицательное число`); // Этот блок выполнится
    }
    i += 1; // i становится 0
    ```
*   **Ожидаемый вывод в консоли:**
    ```
    -1 отрицательное число
    ```

#### Оператор if без блока кода (однострочный)
*   **Концепция:** Если после `if`, `else if` или `else` следует только ОДНА инструкция, то фигурные скобки `{}` для блока кода можно опустить.
*   **Синтаксис:**
    ```javascript
    if (condition1) code1;
    else if (condition2) code2;
    else code3;
    code4;
    ```
*   **Замечание:** Такой синтаксис допускается, но может снизить читаемость кода, особенно если в будущем потребуется добавить больше инструкций в блок. Рекомендуется использовать фигурные скобки для ясности.
*   **Пример кода (с тем же логическим потоком, но без фигурных скобок):**
    ```javascript
    let i = -1;
    if (i === 0) console.log(`${i} равен нулю`);
    else if (i > 0) console.log(`${i} положительное число`);
    else console.log(`${i} отрицательное число`);
    i += 1;
    ```
*   **Ожидаемый вывод в консоли:**
    ```
    -1 отрицательное число
    ```

### Ключевые выводы
1.  **Условное выполнение:** Операторы `if`, `else if`, `else` позволяют коду динамически реагировать на различные сценарии.
2.  **Цепочка условий:** `if` начинает цепочку, `else if` предлагает альтернативные условия, а `else` служит универсальным вариантом по умолчанию.
3.  **Булевы значения:** Условия в операторах `if` и `else if` всегда должны возвращать `true` или `false`.
4.  **Поток выполнения:** Только один блок кода в цепочке `if-else if-else` будет выполнен. Как только условие истинно и его блок выполнен, остальная часть цепочки пропускается.
5.  **Читаемость кода:** Хотя для одиночных инструкций скобки `{}` можно опускать, их использование повышает читаемость и поддерживаемость кода.

### Термины
*   **Ветвление (Branching):** Структура управления потоком выполнения программы, позволяющая выбирать, какой блок кода выполнять, основываясь на логических условиях.
*   **Оператор `if`:** Базовая условная конструкция, которая выполняет блок кода, если его условие истинно.
*   **Оператор `else if`:** Расширение `if`, позволяющее проверять дополнительные условия, если предыдущие были ложны.
*   **Оператор `else`:** Выполняет блок кода, когда ни одно из условий в предшествующих `if` или `else if` не оказалось истинным.
*   **Условие (Condition):** Выражение, помещаемое в скобки после `if` или `else if`, которое оценивается как `true` или `false`.
*   **Блок кода (Code Block):** Набор из одной или нескольких инструкций, заключенных в фигурные скобки `{}`.
