# Рекурсия

## Определение

Рекурсия — приём программирования, при котором функция вызывает сама себя. Используется, когда задача может быть разделена на несколько таких же, но более простых подзадач.

### Два обязательных компонента

1. **База рекурсии** — условие, при котором функция завершается без рекурсивного вызова (тривиальный случай)
2. **Шаг рекурсии** — сведение задачи к более простому случаю через рекурсивный вызов

---

## Пример: возведение в степень

```javascript
function pow(x, n) {
  if (n == 1) {        // база рекурсии
    return x;
  } else {             // шаг рекурсии
    return x * pow(x, n - 1);
  }
}

pow(2, 4);
// pow(2, 4) = 2 * pow(2, 3)
// pow(2, 3) = 2 * pow(2, 2)
// pow(2, 2) = 2 * pow(2, 1)
// pow(2, 1) = 2 (база)
// Результат: 2 * 2 * 2 * 2 = 16
```

### Сравнение с итеративным подходом

```javascript
// Итеративно
function pow(x, n) {
  let result = 1;
  for (let i = 0; i < n; i++) {
    result *= x;
  }
  return result;
}

// Рекурсивно — короче
function pow(x, n) {
  return (n == 1) ? x : (x * pow(x, n - 1));
}
```

**Рекурсивное решение обычно короче и нагляднее**, но итеративное — экономичнее по памяти.

## Как работает рекурсия: контекст выполнения и стек

При каждом вызове функции создаётся **контекст выполнения** (execution context) с:
- Локальными переменными
- Позицией в коде
- Значением `this`

При рекурсивном вызове:
1. Контекст текущей функции сохраняется в **стеке контекстов**
2. Создаётся новый контекст для вложенного вызова
3. После завершения вложенного вызова — контекст восстанавливается из стека

**Глубина рекурсии** — количество одновременно хранимых в стеке контекстов. В примере `pow(2, 4)` глубина = 4.

### Ограничения
- Максимальная глубина рекурсии ограничена движком JS (~10000 вызовов)
- Рекурсия требует больше памяти, чем цикл (каждый контекст хранится в стеке)
- **Любая рекурсия может быть переписана в цикл**, но не всегда это оправдано

## Рекурсивный обход структур

Классическое применение — обход вложенных структур данных неизвестной глубины.

### Пример: сумма зарплат в компании с отделами

```javascript
let company = {
  sales: [{name: 'John', salary: 1000}, {name: 'Alice', salary: 600}],
  development: {
    sites: [{name: 'Peter', salary: 2000}, {name: 'Alex', salary: 1800}],
    internals: [{name: 'Jack', salary: 1300}]
  }
};

function sumSalaries(department) {
  if (Array.isArray(department)) {
    // База: массив сотрудников — суммируем
    return department.reduce((prev, curr) => prev + curr.salary, 0);
  } else {
    // Шаг: объект с отделами — рекурсивно обходим
    let sum = 0;
    for (let subdep of Object.values(department)) {
      sum += sumSalaries(subdep);
    }
    return sum;
  }
}

sumSalaries(company); // 6700
```

### Другие применения
- Обход DOM-дерева
- Обход файловой системы
- Парсинг вложенных JSON-структур
- Алгоритмы на графах и деревьях

## Рекурсивные структуры данных

Структура, определяемая через саму себя.

### Связный список (Linked List)

```javascript
let list = {
  value: 1,
  next: {
    value: 2,
    next: {
      value: 3,
      next: null
    }
  }
};
```

**Преимущества перед массивом:**
- Быстрая вставка/удаление в начале и середине (без переиндексации)
- Лёгкое разделение и объединение

**Недостатки:**
- Нет произвольного доступа по индексу (только последовательный)
- Больше расход памяти (хранение ссылок)

---

## Когда применять рекурсию

**Хорошо подходит:**
- Задачи с вложенными структурами неизвестной глубины
- Древовидные структуры (DOM, файлы, JSON)
- Алгоритмы «разделяй и властвуй» (быстрая сортировка, сортировка слиянием)
- Математические определения (факториал, Фибоначчи, обход графа)

**Лучше избегать:**
- Простые линейные задачи (цикл проще и эффективнее)
- Глубокие рекурсии (риск переполнения стека)
- Когда важна производительность и память
