# Обработка ошибок: throw и try-catch

**Краткое содержание:**
Урок знакомит с механизмом исключений в JavaScript. Студент учится выбрасывать ошибки через `throw`, проверять типы аргументов через `typeof` и ловить ошибки через `try-catch`. Эти навыки необходимы для написания надёжных функций и тестов к ним.

### Подробный конспект

#### 1. Зачем нужна обработка ошибок

Представь: ты написал функцию, которая ожидает строку, а ей передали число. Что произойдёт?

```javascript
function greet(name) {
  return 'Привет, ' + name + '!';
}

console.log(greet('Алия'));   // Привет, Алия!
console.log(greet(42));       // Привет, 42! — работает, но это неправильно
```

Код не падает, но результат бессмысленный. В сложной программе такая ошибка может проявиться через 10 шагов — и ты потратишь часы на поиск причины.

**Решение:** функция должна явно сообщить, что её вызвали неправильно. Для этого есть **исключения (exceptions)**.

#### 2. Оператор throw

`throw` выбрасывает ошибку и останавливает выполнение:

```javascript
function greet(name) {
  if (typeof name !== 'string') {
    throw new TypeError('Аргумент name должен быть строкой');
  }
  return 'Привет, ' + name + '!';
}

console.log(greet('Алия'));   // Привет, Алия!
console.log(greet(42));       // TypeError: Аргумент name должен быть строкой
```

**Ключевые моменты:**
- `throw` немедленно прекращает выполнение функции
- Код после `throw` не выполнится
- Ошибка «всплывает» вверх по стеку вызовов, пока её кто-нибудь не поймает

#### 3. Типы ошибок

В JavaScript есть встроенные типы ошибок:

| Тип | Когда использовать |
|-----|-------------------|
| `TypeError` | Неправильный тип аргумента |
| `RangeError` | Значение вне допустимого диапазона |
| `Error` | Общая ошибка (универсальный вариант) |
| `SyntaxError` | Ошибка синтаксиса (обычно не выбрасывают вручную) |
| `ReferenceError` | Обращение к несуществующей переменной |

```javascript
// TypeError — неправильный тип
throw new TypeError('Ожидалась строка');

// RangeError — выход за границы
throw new RangeError('Индекс должен быть от 0 до длины массива');

// Error — универсальный
throw new Error('Что-то пошло не так');
```

#### 4. Проверка типов через typeof

Оператор `typeof` возвращает строку с типом значения:

```javascript
typeof 'hello';   // 'string'
typeof 42;        // 'number'
typeof true;      // 'boolean'
typeof undefined; // 'undefined'
typeof null;      // 'object' (известная особенность JS)
typeof [1, 2, 3]; // 'object'
typeof {};        // 'object'
```

Для проверки, что аргумент — строка:
```javascript
if (typeof arg !== 'string') {
  throw new TypeError('Аргумент должен быть строкой');
}
```

#### 5. try-catch: ловим ошибки

`try-catch` позволяет перехватить ошибку и обработать её, не роняя всю программу:

```javascript
try {
  // Код, который может выбросить ошибку
  const result = greet(42);
  console.log(result);
} catch (error) {
  // Код, который выполнится если в try произошла ошибка
  console.log('Произошла ошибка:', error.message);
}
// Программа продолжает работать!
```

**Структура:**
- Блок `try` — опасный код, который исполняется первым
- Блок `catch` — выполняется, только если в `try` произошла ошибка
- Переменная в `catch(error)` содержит объект ошибки; у него есть свойства `.message`, `.name`, `.stack`

#### 6. try-catch в тестах

В тестах `try-catch` используют, чтобы проверить, что функция **действительно выбрасывает** ошибку в нештатной ситуации:

```javascript
// Тест: функция должна выбросить TypeError для не-строки
try {
  greet(42);
  // Если дошли сюда — ошибки не было, тест провален
  console.log('ТЕСТ ПРОВАЛЕН: greet(42) должна была выбросить ошибку');
} catch (error) {
  // Ошибка выброшена — проверяем её тип и сообщение
  if (error instanceof TypeError) {
    console.log('ТЕСТ ПРОЙДЕН: greet(42) выбросила TypeError');
  } else {
    console.log('ТЕСТ ПРОВАЛЕН: ожидался TypeError, получен ' + error.name);
  }
}
```

**Альтернатива в bun:test** (будет рассмотрена в уроке TDD):
```javascript
expect(() => greet(42)).toThrow(TypeError);
```

#### 7. Практика: функция len(str) с проверкой типа

Напишем заготовку функции `len(str)` — она возвращает длину строки (пока без TDD, просто код):

```javascript
function len(str) {
  // Проверка типа аргумента
  if (typeof str !== 'string') {
    throw new TypeError('Аргумент str должен быть строкой');
  }

  // Подсчёт символов (пока заглушка)
  let count = 0;
  for (let i = 0; str[i] !== undefined; i++) {
    count++;
  }
  return count;
}
```

Проверим через try-catch:
```javascript
try {
  console.log(len('hello')); // 5 — работает
  console.log(len(123));     // Должна выбросить ошибку
} catch (e) {
  console.log('Поймана ошибка:', e.message);
}
```
