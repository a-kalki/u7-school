# Шаги урока: Обработка ошибок — throw и try-catch

---

### Зачем нужна обработка ошибок

**kind:** `text`

Представь: ты написал функцию, которая ожидает строку, а ей передали число.

```javascript
function greet(name) {
  return 'Привет, ' + name + '!';
}

console.log(greet('Алия'));   // Привет, Алия!
console.log(greet(42));       // Привет, 42! — работает, но бессмысленно
```

Код не падает, но результат неверный. В сложной программе такая ошибка может проявиться через 10 шагов — и ты потратишь часы на поиск причины.

**Решение:** функция должна явно сообщать, что её вызвали неправильно. Для этого в JavaScript есть `throw`.

---

### Оператор throw и типы ошибок

**kind:** `text`

Оператор `throw` выбрасывает ошибку и **немедленно** останавливает выполнение  потока выполнения (на сленге: выкинуть исключение).

```javascript
function greet(name) {
  if (typeof name !== 'string') {
    throw new TypeError('Аргумент name должен быть строкой');
  }
  return 'Привет, ' + name + '!';
}

console.log(greet('Алия'));   // Привет, Алия!
console.log(greet(42));       // TypeError: Аргумент name должен быть строкой
```

Ключевые моменты:
- Код после `throw` **не выполнится**
- Ошибка «всплывает» вверх, пока её кто-нибудь не поймает

**Основные типы ошибок:**

| Тип | Когда использовать |
|-----|-------------------|
| `TypeError` | Неправильный тип аргумента |
| `RangeError` | Значение вне допустимого диапазона |
| `Error` | Общая ошибка (универсальный вариант) |

### try-catch — перехват ошибок

**kind:** `text`

`try-catch` позволяет **перехватить** ошибку и обработать её, не роняя всю программу:

```javascript
try {
  // Код, который может выбросить ошибку
  console.log(greet(42));
} catch (error) {
  // Выполнится, только если в try произошла ошибка
  console.log('Произошла ошибка:', error.message);
}
console.log('Программа продолжает работать!');
```

У объекта ошибки есть полезные свойства:
- `error.name` — тип ошибки (`'TypeError'`, `'RangeError'` и т.д.)
- `error.message` — текст сообщения
- `error.stack` — стек вызовов функции и методов
- `error instanceof TypeError` — проверка конкретного типа ошибки

### Предскажи вывод: практика с throw и try-catch

**kind:** `text`

Предскажи, что выведет этот код (запиши ответ, затем проверь в консоли):

```javascript
function divide(a, b) {
  if (typeof a !== 'number' || typeof b !== 'number') {
    throw new TypeError('Оба аргумента должны быть числами');
  }
  if (b === 0) {
    throw new Error('Деление на ноль');
  }
  return a / b;
}

try {
  console.log(divide(10, 2));
  console.log(divide(10, 0));
  console.log(divide('10', 2));
} catch (error) {
  console.log('Поймана ошибка:', error.message);
}

console.log('Конец программы');
```

Вопросы:
1. Выполнится ли `divide('10', 2)` после того, как `divide(10, 0)` выбросит ошибку?
2. Выполнится ли `console.log('Конец программы')`?

### Реализуй repeat() с проверкой аргументов

**kind:** `text`

Напиши функцию `repeat(text, count)`, которая повторяет строку `text` указанное количество раз. Добавь проверки:

- Если `text` не строка — `throw new TypeError('text должен быть строкой')`
- Если `count` не число — `throw new TypeError('count должен быть числом')`
- Если `count` меньше или равно 0 — `throw new RangeError('count должен числом больше 0')`

Проверь работу через `try-catch` для случаев:
- `repeat('JS', 3)` → `'JSJSJS'`
- `repeat(123, 3)` → ловится `TypeError`
- `repeat('JS', 'abc')` → ловится `TypeError`
- `repeat('JS', 0)` → ловится `RangeError`
