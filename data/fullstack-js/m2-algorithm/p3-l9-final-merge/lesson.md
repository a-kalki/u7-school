# Финальный мерж

**Краткое содержание:**
Финальный урок проекта 3: студент получает 2 аппрува, выполняет мерж PR на GitHub, подтягивает изменения в локальный `main` и чистит ветки.

### 1. Проверка перед мержем

Убедись, что:
- Получено **2 аппрува** (зелёные галочки в PR)
- Все обсуждения (conversations) помечены как resolved
- CI (если настроен) прошёл успешно
- Ты ответил на все комментарии ревьюеров

Если какой-то из пунктов не выполнен — доведи до конца, прежде чем мержить.

### 2. Мерж на GitHub

Есть два способа замержить PR:

**Способ 1 — через интерфейс GitHub (рекомендуется):**
1. Открой PR на GitHub
2. Нажми зелёную кнопку **Merge pull request**
3. Выбери тип слияния:
   - **Merge commit** — создаёт merge-коммит (сохраняет историю ветки). Хороший выбор по умолчанию
   - **Squash and merge** — сжимает все коммиты ветки в один (чистая история, но теряется детализация). В рамках модуля — не рекомендуется, чтобы была видна работа
   - **Rebase and merge** — перекладывает коммиты ветки на верхушку main (линейная история). Тоже ок, но merge commit — проще для понимания
4. Подтверди: нажми **Confirm merge**
5. После мержа GitHub предложит удалить ветку — нажми **Delete branch** (это удалит ветку на GitHub)

**Способ 2 — через терминал (если хочешь потренироваться):**
```bash
git checkout main
git pull origin main          # подтянуть актуальный main
git merge feat/substring-search
git push origin main
```

Но раз ты уже создал PR, способ 1 проще и правильнее.

### 3. Обновление локального репозитория

После мержа на GitHub твой локальный `main` отстаёт. Подтяни изменения:

```bash
git checkout main
git pull origin main
```

Проверь историю: коммиты из `feat/substring-search` должны быть видны в `main`:

```bash
git log --oneline
```

### 4. Удаление старых веток

**Локальные ветки:**
```bash
git branch -d feat/substring-search
```
Флаг `-d` удалит ветку, только если она уже слита. Если Git ругается — используй `-D` (принудительно), но проверь, что ветка действительно не нужна.

**Удалённые ветки (если не удалил через интерфейс GitHub):**
```bash
git push origin --delete feat/substring-search
```

**Очистка информации об удалённых ветках:**
```bash
git fetch --prune
```
Эта команда удаляет из локального репо ссылки на ветки, которых уже нет на GitHub.

### 5. Итоги проекта 3

Ты завершил проект 3. В твоём арсенале теперь:

- **Git:** ветки (branch, switch), слияние (merge), конфликты и их разрешение, fix-ветки
- **Алгоритмы:** наивный поиск подстроки, проверки начала/конца/наличия, переворот и повторение строки
- **Процесс:** полный цикл — ветка → код → тесты → push → PR → ревью → правки → 2 аппрува → мерж
