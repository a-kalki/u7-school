### О ресурсе w3-school

**kind:** `text`

В этом модуле для изучения Git мы будем использовать ресурс **W3Schools** — один из крупнейших образовательных сайтов для разработчиков. Вот что нужно о нём знать.

**Что такое W3Schools**

W3Schools — это учебный портал, существующий с 1998 года. На нём собраны уроки по всем основным технологиям веба: HTML, CSS, JavaScript, Python, SQL, Git, React и десяткам других. Каждый урок состоит из теории, живых примеров кода (можно редактировать и запускать прямо в браузере) и интерактивных упражнений.

**Регистрация**

1. Перейди на [w3schools.com](https://www.w3schools.com/) и нажми **Sign Up** в правом верхнем углу
2. Заполни email и пароль (или войди через Google/GitHub)
3. Подтверди email — на почту придёт письмо со ссылкой
4. После входа в правом верхнем углу появится твой аватар — регистрация завершена

Регистрация **обязательна**. Без неё ты не сможешь:
- Отслеживать свой прогресс (пройденные уроки отмечаются зелёной галочкой)
- Выполнять упражнения с автопроверкой
- Получать сертификаты

**Бесплатно vs PRO**

| Возможность | Бесплатно | PRO |
|------------|-----------|-----|
| Читать уроки | ✅ | ✅ |

| Отслеживание прогресса | ✅ | ✅ |

Для нашего модуля бесплатной версии **достаточно**. PRO имеет смысл, если ты хочешь выполнять из задания-экзамены и/или получить их сертификат для портфолио.

**Язык уроков**

Все уроки на W3Schools — **на английском языке**. Это не баг, а фича:

- Технический английский в IT — базовый навык. Вся документация, статьи, ответы на StackOverflow — на английском. Чем раньше ты начнёшь читать технические тексты в оригинале, тем быстрее привыкнешь
- Если сложно — включи встроенный перевод браузера (правая кнопка → «Перевести на русский»), но **старайся запоминать термины в оригинале**: repository, commit, branch, stage
- Все примеры кода в уроках на английском — это стандарт в программировании

**Как устроен урок на W3Schools**

Каждый урок имеет одинаковую структуру:
1. **Теория** — текст с пояснениями и диаграммами
2. **Try it Yourself** — зелёная кнопка, открывающая живой редактор кода. Меняй код и сразу видишь результат
3. **Exercises** (только с PRO) — задания в конце урока для закрепления
4. **Навигация** — кнопки Previous/Next внизу страницы для последовательного прохождения

### Что такое Git

**kind:** `text`

Прочитай введение в Git на W3Schools. Обрати внимание на три вопроса (ответь на них себе после прочтения):

1. Что такое **система контроля версий** и какую проблему она решает?
2. Что такое **репозиторий**, **коммит** и **stage**?
3. Что такое **ветка** и зачем она нужна?

Изучи:
- [Git Intro](https://www.w3schools.com/git/git_intro.asp) — что такое Git
- [Git Install](https://www.w3schools.com/git/git_install.asp) — установка Git
- [Git Config](https://www.w3schools.com/git/git_config.asp) — настройка имени и email

После прочтения установи Git (если ещё не установлен) и выполни базовую настройку:

```
git config --global user.name "Твоё Имя"
git config --global user.email "твой.email@example.com"
```

Проверь:

```
git config --global user.name
git config --global user.email
```

### Контрольные вопросы: коммит, stage и ветки

**kind:** `text`

Ответь на вопросы (устно или в конспекте):

1. Что такое **коммит**? Чем он отличается от простого сохранения файла через Ctrl+S?
2. Зачем нужен **stage** (индекс)? Почему нельзя просто коммитить все изменения разом, без подготовки?
3. Ты работаешь над новой фичей, но она экспериментальная и может сломать проект. Как ветки помогают в этой ситуации?
4. Что произойдёт, если удалить скрытую папку `.git` из проекта?

Если есть все еще сложности с пониманием этих терминов и в общем с git, то скопируй эти вопросы и задай их своему любимому ИИ ассистенту.
