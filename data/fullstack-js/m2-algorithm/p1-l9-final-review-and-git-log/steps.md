### Шаг 3

**kind:** `text`

Изучи историю своего проекта через Git:

```
git log --oneline
```

Ты должен увидеть список всех коммитов — по одному на каждую реализованную функцию. Попробуй также:

```
git log -3           # последние 3 коммита
git log --graph      # графическое представление (пока одна ветка — main)
```

### Шаг 4

**kind:** `text`

Проверь свои commit-сообщения через `git log --oneline`. Соответствуют ли они правилам хорошего тона?

**Хорошее сообщение:**
- Глагол в повелительном наклонении: «Реализовать», «Исправить», «Добавить»
- Кратко — первая строка до 72 символов
- Понятно без контекста: **что** сделано, а не **как**

Примеры:

```
✅ Реализовать isMore — лексикографическое сравнение
✅ Исправить обработку пустых строк в isEqual
❌ fix
❌ обнова
❌ чё-то поменял
```

Если есть незакоммиченные правки — добавь их и сделай финальный коммит:

```
git add .
git commit -m "Завершить проект 1: 7 функций сравнения строк"
```
