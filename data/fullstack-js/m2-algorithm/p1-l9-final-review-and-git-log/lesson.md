### 1. Полный прогон тестов

К этому моменту в проекте должны быть реализованы все 7 функций:

- `len(str)` — длина строки
- `isEqual(a, b)` — посимвольное равенство
- `isNotEqual(a, b)` — отрицание равенства
- `isMore(a, b)` — лексикографическое «строго больше»
- `isLess(a, b)` — лексикографическое «строго меньше»
- `isMoreOrEqual(a, b)` — больше или равно
- `isLessOrEqual(a, b)` — меньше или равно

### 2. Просмотр истории Git

Команда `git log` показывает историю коммитов:

```bash
git log
```

Вывод по умолчанию — полная информация о каждом коммите: хеш, автор, дата, сообщение.

**Полезные флаги:**

```bash
# Компактный вывод: по одному коммиту на строку
git log --oneline

# Показать последние 5 коммитов
git log -5

# Показать коммиты с изменениями (diff)
git log -p

# Графическое представление веток (пока одна ветка)
git log --graph --oneline
```

Пример вывода `git log --oneline`:

```
a1b2c3d Реализовать isLessOrEqual и isMoreOrEqual
e4f5g6h Реализовать isNotEqual, isMore и isLess
i7j8k9l Реализовать isEqual
m0n1o2p Реализовать функцию len(str) через TDD
```

### 3. Правила хороших commit-сообщений

Хорошее commit-сообщение — то, по которому через месяц понятно, что было сделано и зачем.

**Принципы:**

- **Повелительное наклонение:** «Добавить», «Исправить», «Реализовать» (как будто ты приказываешь коду измениться)
- **Первая строка до 72 символов** — краткая суть
- **Понятно без контекста:** не «фикс», а «Исправить сравнение пустых строк в isMore»
- **На русском или английском** — как принято в команде

**Плохие примеры:**
```
fix
WIP
обнова
чё-то поменял
```

**Хорошие примеры:**
```
Реализовать isMore с лексикографическим сравнением
Исправить обработку пустых строк в isEqual
Добавить тесты на кириллицу для len
```

### 4. Финальный коммит проекта

Если есть незакоммиченные изменения (новые тесты, правки) — добавь их и сделай финальный коммит:

```bash

```

Проверь финальную историю:

```bash
git log --oneline
```
