### Запуск тестов: фильтрация

**kind:** `text`

К этому моменту у тебя накопилось много тестовых файлов. Каждый раз ждать прогон всех тестов неудобно — Bun умеет запускать только то что нужно.

**Запуск конкретного файла**

```
bun test is-more.test.js
```

Указываешь полное имя файла — Bun выполнит только его.

**Запуск по совпадению в имени файла**

```
bun test more
```

Bun найдёт все файлы, в имени которых встречается «more», и запустит их. Например: `is-more.test.js` и `is-more-or-equal.test.js`. Удобно когда помнишь часть имени файла или работаешь над группой связанных функций.

**Запуск по описанию теста: флаг `-t`**

Флаг `-t` (test name) фильтрует тесты по тексту в `test('...', ...)` или `describe('...', ...)`.

```
bun test -t "пустых"
```

Запустит только те тесты, в описании которых есть «пустых».

Примеры фильтрации:

```
# Конкретный тест
test('должна вернуть 0 для пустой строки', () => { ... })
bun test -t "пустой строки"       # найдёт именно этот тест

# Все тесты внутри describe
describe('Тесты isMore', () => { ... })
bun test -t "Тесты isMore"         # запустит все тесты из этого describe

# Если описание совпадает у нескольких тестов — запустятся все
test('должна выбросить TypeError если аргумент не строка', () => { ... })
test('должна выбросить TypeError если аргумент не строка', () => { ... })
bun test -t "аргумент не строка"   # запустит оба теста
```

### Запуск тестов: режимы

**kind:** `text`

Ещё два флага, которые пригодятся в работе:

**Watch mode — автоматический перезапуск**

```
bun test --watch
```

Bun будет следить за файлами. Как только ты сохранишь изменения в коде или тестах — тесты перезапустятся автоматически. Не нужно каждый раз вручную набирать `bun test`. Выйти из watch mode: `Ctrl+C`.

**Подробный вывод при ошибке**

```
bun test --verbose
```

При падении теста показывает не только `Expected / Received`, но и полный стек вызовов — на какой строке и в какой функции произошла ошибка. Полезно при отладке.

### Функция isMoreOrEqual(a, b)

**kind:** `text`

Реализуй `isMoreOrEqual(a, b)` — вернуть `true`, если `a` **больше или равно** `b`.

**Подсказка:** `a >= b` означает «`a` больше `b` ИЛИ `a` равна `b`»:

Тестовые случаи:
- Должна вернуть `true` если `a` больше (`'cat'`, `'car'`)
- Должна вернуть `true` если строки равны (`'hello'`, `'hello'`)
- Должна вернуть `false` если `a` меньше (`'car'`, `'cat'`)
- Должна вернуть `true` если `a` длиннее и символы совпадают (`'hello!'`, `'hello'`)
- Должна вернуть `false` если `a` короче и символы совпадают (`'hello'`, `'hello!'`)
- Должна вернуть `true` для пустых строк (`''`, `''`)
- Должна выбросить `TypeError` если аргумент(ы) не строка

Закоммить:

```
git add is-more-or-equal.js is-more-or-equal.test.js
git commit -m "Реализовать isMoreOrEqual"
```
