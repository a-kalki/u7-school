# Ветки в Git — краткая выжимка

- **Ветка** — «параллельная вселенная» для кода. `main` остаётся стабильным, работа идёт в feature-ветках
- Ветка — это указатель на коммит, а не копия файлов. Создание/переключение — мгновенно
- `HEAD` — указатель на текущую ветку
- **Команды:**
  - `git branch` — список веток, `*` — текущая
  - `git branch имя` — создать ветку
  - `git switch имя` — переключиться (или `git checkout`)
  - `git switch -c имя` — создать и сразу переключиться
  - `git branch -d имя` — удалить слитую ветку
  - `git branch -D имя` — удалить принудительно (даже если не слита)
- **Удаление ветки ≠ удаление коммитов.** Если коммиты не слиты — становятся «висячими» и со временем чистятся сборщиком мусора
- **`git reflog`** — локальный журнал всех действий, позволяет восстановить «потерянные» коммиты (~30 дней)
- Имена веток: `feat/`, `fix/`, `chore/` — осмысленные префиксы
