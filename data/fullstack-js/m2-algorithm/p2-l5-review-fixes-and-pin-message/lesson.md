# Процесс доработки по ревью и pin-сообщение

**Краткое содержание:**
Студент узнаёт, как организовать процесс исправления замечаний ревью: отвечать на комментарии, пушить правки, запрашивать повторное ревью. Также настраивает быстрое взаимодействие через pin-сообщение в чате.

> **Важно:** этот урок ты читаешь **до** того, как ревьюеры оставят комментарии. Ревьюеры — живые люди, они ответят не сразу. Прочитай урок сейчас, чтобы понимать процесс. Сами действия (ответы на комментарии, правки, повторный запрос ревью) будешь выполнять позже, когда появятся реальные замечания.

### 1. Как отвечать на комментарии ревью

Когда ревьюер оставляет комментарий, у тебя есть два варианта ответа:

**Согласен — исправляю:**
- Пишешь краткий ответ: «Согласен, исправил»
- Вносишь правку в код
- Делаешь коммит и push
- После исправления комментарий можно отметить как **resolved** (кнопка «Resolve conversation»)

**Не согласен — аргументирую:**
- Пишешь аргументированный ответ: почему твой вариант лучше или почему предложение ревьюера нельзя применить
- Обсуждаете в ветке комментария, пока не придёте к согласию
- Не отмечай как resolved, пока ревьюер не согласится с твоими аргументами

### 2. Пуш правок — PR обновляется автоматически

После того как ты исправил код, сделал коммит и запушил — **PR на GitHub обновляется автоматически**. Никаких новых PR создавать не нужно.

Проверь:
1. После `git push` — обнови страницу PR
2. Во вкладке **Commits** должен появиться новый коммит с правками
3. Во вкладке **Files changed** — diff обновился

### 3. Повторный запрос ревью

После того как ты исправил все замечания, нужно явно запросить ревьюера пересмотреть код:

1. В правой панели PR найди раздел **Reviewers**
2. Рядом с именем ревьюера, который запросил правки, появится иконка 🔄 (re-request review)
3. Нажми на неё

Ревьюер получит уведомление, что ты внёс правки и ждёшь повторной проверки.

### 4. Pin-сообщение в чате

Чтобы ревьюеры быстро находили твой PR, закрепи сообщение со ссылкой в общем чате группы.

**Что писать в pin-сообщении:**

```
🔍 PR на ревью: [7 функций сравнения строк]
📎 Ссылка: https://github.com/твой-username/js-algorithms/pull/1
👥 Нужны ревью от: @username1, @username2
📋 Правила ревью: 2 аппрува для мержа
```

**Обновляй pin-сообщение при изменениях:**
- После правок по ревью: «Правки внесены, жду повторное ревью»
- После получения аппрува от одного ревьюера: «✅ 1/2 аппрувов получено»
- После второго аппрува: «✅ PR одобрен, мерж!»

Это держит всех в курсе и экономит время — ревьюеры не гадают, нужно ли им сейчас что-то смотреть или нет.

### 5. Что делать, если ревьюер не отвечает

В реальной работе такое бывает. Алгоритм действий:

1. **Напомнить** через 1–2 дня в личном сообщении или в чате
2. **Пингануть** в общем чате: «@username, посмотри PR пожалуйста»
3. **Найти другого ревьюера** — если первый занят/в отпуске
4. **Эскалировать** — сообщить ментору или тимлиду, если ревью критично задерживается

Не нужно молча ждать неделю — напоминай вежливо. Всем иногда нужно напоминание.
