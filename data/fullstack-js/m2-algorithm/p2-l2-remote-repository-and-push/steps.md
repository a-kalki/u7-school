### Создание репозитория и первый push

**kind:** `text`

Теперь свяжи свой локальный проект с GitHub.

**1. Создай репозиторий на GitHub:**

- Нажми **+** → **New repository**
- Имя: `js-algorithms`
- Выбери **Public** (или Private — как хочешь)
- **НЕ** добавляй README, .gitignore и license (у тебя уже есть файлы)
- Нажми **Create repository**

**2. Привяжи локальный репозиторий:**

Скопируй SSH-адрес со страницы нового репозитория (начинается с `git@github.com:`) и выполни в папке проекта:

```
git remote add origin git@github.com:твой-username/js-algorithms.git
```

Проверь, что remote добавился:

```
git remote -v
```

Должен быть вывод с двумя строками: `origin ... (fetch)` и `origin ... (push)`.

**3. Сделай первый push:**

```
git push -u origin main
```

Git отправит все твои коммиты на GitHub. Обнови страницу репозитория — ты должен увидеть свои файлы и историю коммитов.

### Сверка истории коммитов

**kind:** `text`

Проверь историю коммитов на GitHub: перейди на вкладку **commits** (значок часов над списком файлов).

Сравни её с локальной историей:

```
git log --oneline
```

Вывод должен совпадать — на GitHub видны те же коммиты с теми же сообщениями, что и локально.
