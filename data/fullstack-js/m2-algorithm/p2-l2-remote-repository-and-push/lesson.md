# Создание удалённого репозитория и git push

**Краткое содержание:**
Студент создаёт репозиторий на GitHub, связывает его с локальным Git через `git remote add` и делает первый `git push`.

### 1. Удалённый репозиторий (remote)

До этого момента весь твой проект `js-algorithms` жил только на твоём компьютере — в локальном Git-репозитории. Теперь мы свяжем его с **удалённым репозиторием** на GitHub.

**Удалённый репозиторий (remote)** — копия твоего Git-репозитория, которая хранится на сервере (GitHub). Он нужен чтобы:
- Другие разработчики могли скачать твой код
- Ты мог работать с разных компьютеров
- Твой код был защищён от потери (бэкап)
- Ты мог создавать Pull Request для ревью

Связь локального и удалённого репозитория — двусторонняя:
- `push` — отправляешь изменения **из** локального репо **в** удалённый
- `pull` — забираешь изменения **из** удалённого репо **в** локальный

### 2. Создание репозитория на GitHub

1. Нажми на **+** в правом верхнем углу GitHub → **New repository**
2. В поле **Repository name** введи: `js-algorithms`
3. **Description** (описание) — необязательно, можно оставить пустым
4. Выбери **Private** (приватный) или **Public** (публичный):
   - **Public** — твой код видят все (портфолио для работодателей)
   - **Private** — код виден только тебе и тем, кому дашь доступ
5. **НЕ ставь** галочки:
   - ❌ Add a README file — у тебя уже есть файлы в проекте
   - ❌ Add .gitignore — у тебя уже есть `.gitignore`
   - ❌ Choose a license — не нужно сейчас
6. Нажми **Create repository**

После создания GitHub покажет инструкцию. Нам нужен блок **«…or push an existing repository from the command line»**.

### 3. Привязка локального репозитория к GitHub: git remote add

Открой терминал в папке проекта `js-algorithms` и выполни:

```bash
git remote add origin git@github.com:твой-username/js-algorithms.git
```

Разберём команду по частям:
- `git remote add` — добавить новый удалённый репозиторий
- `origin` — стандартное имя для основного удалённого репозитория (можно любое, но `origin` — общепринятое)
- `git@github.com:твой-username/js-algorithms.git` — SSH-адрес репозитория

**Проверка:**

```bash
git remote -v
```

Должен вывести:
```
origin  git@github.com:твой-username/js-algorithms.git (fetch)
origin  git@github.com:твой-username/js-algorithms.git (push)
```

Две строки с `origin` — fetch (откуда забирать) и push (куда отправлять). Обычно они совпадают.

### 4. Первый push

```bash
git push -u origin main
```

Разберём:
- `git push` — отправить изменения
- `-u` (или `--set-upstream`) — запомнить связку «локальная ветка main → удалённая ветка origin/main». После этого можно просто писать `git push` без аргументов
- `origin` — куда отправляем (имя удалённого репозитория)
- `main` — какую локальную ветку отправляем

**Что произойдёт:**
1. Git отправит все твои коммиты на GitHub
2. На GitHub появится ветка `main` с теми же файлами и историей коммитов
3. Связка `main ↔ origin/main` запомнится (благодаря `-u`)

**Проверка:** обнови страницу репозитория на GitHub — ты должен увидеть свои файлы (`index.js`, `len.js`, `isEqual.js` и т.д.) и историю коммитов.

### 5. Просмотр удалённых веток

```bash
git branch -r      # удалённые ветки
git branch -a      # все ветки (локальные + удалённые)
```

Вывод:
```
* main
  remotes/origin/main
```
