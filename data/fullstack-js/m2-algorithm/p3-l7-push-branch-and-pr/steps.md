# Шаги урока: Пуш ветки и создание PR

---

### Шаг 1 — Финальная проверка

**kind:** `text`

Перед отправкой кода на GitHub убедись, что всё в порядке:

1. Все 6 функций готовы и протестированы
2. `bun test` — все тесты зелёные
3. `git status` — working tree clean, ты в ветке `feat/substring-search`
4. `git log --oneline` — осмысленная история коммитов

Если есть незакоммиченные изменения — закоммить их.

---

### Шаг 2 — Пуш ветки

**kind:** `text`

Отправь ветку на GitHub:

```
git push -u origin feat/substring-search
```

Проверь, что ветка появилась на GitHub: перейди в репозиторий, открой выпадающий список веток (слева над списком файлов, где написано `main`). Ты должен увидеть `feat/substring-search`.

---

### Шаг 3 — Создание PR

**kind:** `text`

Создай Pull Request:

1. На GitHub нажми зелёную кнопку **Compare & pull request** (обычно появляется автоматически)
2. Или зайди во вкладку **Pull requests** → **New pull request**, выбери base: `main`, compare: `feat/substring-search`

Заполни:
- **Заголовок:** `Проект 3: реализованы функции поиска подстрок и утилиты для строк`
- **Описание:** распиши, что сделано (6 функций), как запустить тесты

Пример описания:

```
## Что сделано

Реализованы 6 функций для работы со строками:

- indexOf(str, search) — наивный поиск подстроки
- includes(str, search) — проверка наличия подстроки
- startsWith(str, search) — проверка начала строки
- endsWith(str, search) — проверка конца строки
- reverse(str) — переворот строки
- repeat(str, count) — повторение строки

## Как запустить тесты

bun test
```

### Шаг 4 — Приглашение ревьюеров и pin

**kind:** `text`

1. В правой панели PR нажми **Reviewers** → выбери 2 и более ревьюеров
2. Скопируй ссылку на PR из адресной строки
3. Закрепи (pin) сообщение со ссылкой в чате группы

**Пример сообщения для чата:**
```
Проект 3: поиск подстрок и утилиты.
PR: https://github.com/username/js-algorithms/pull/2
Нужно 2 и болле аппрува 🙏
```
