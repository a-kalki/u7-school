# Шаги урока: Пуш и Pull Request для проекта 4

---

### Финальная проверка

**kind:** `text`

Перед отправкой кода на GitHub убедись, что всё в порядке:

1. Все 8 функций готовы и протестированы
2. JSDoc добавлен ко всем функциям (включая `len` и `isEqual` из проекта 1)
3. `bun test` — все тесты зелёные
4. `git status` — working tree clean, ты в ветке `feat/string-transform`
5. `git log --oneline` — осмысленная история коммитов

Если есть незакоммиченные изменения — закоммить их.

---

### Пуш ветки

**kind:** `text`

Отправь ветку на GitHub:

```
git push -u origin feat/string-transform
```

Проверь, что ветка появилась на GitHub — открой репозиторий, посмотри выпадающий список веток.

---

### Создание PR

**kind:** `text`

Создай Pull Request:

1. На GitHub нажми **Compare & pull request**
2. Выбери base: `main`, compare: `feat/string-transform`

Заполни заголовок и описание:

**Заголовок:** `Проект 4: JSDoc и функции трансформации строк`

**Описание:** перечисли все 8 функций с краткими пояснениями, укажи что добавлен JSDoc, укажи как запустить тесты (`bun test`), отметь особенности реализации (разрешённые методы, нерекурсивная замена в replaceAll, разница slice/substring).

### Приглашение ревьюеров и pin
