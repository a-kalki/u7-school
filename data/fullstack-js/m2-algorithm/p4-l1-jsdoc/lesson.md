# JSDoc: документирование функций

**Краткое содержание:**
Урок объясняет, зачем нужно документировать код, знакомит со стандартом JSDoc — основным способом документирования JavaScript-функций. Студент изучает синтаксис тегов (`@param`, `@returns`, `@throws`, `@example`) и практикуется, документируя уже реализованные функции `len()` и `isEqual()` из проекта 1.

### Подробный конспект

#### 1. Зачем документировать код

Код читают намного чаще, чем пишут. Через неделю ты сам можешь забыть, что делает твоя функция и какие аргументы принимает. Другие разработчики (в том числе твои ревьюеры) вообще видят твой код впервые.

**Хорошая документация экономит часы** — вместо чтения всего тела функции достаточно прочитать её описание.

Документация отвечает на три вопроса:
1. **Что делает** функция? (одно предложение)
2. **Что принимает** на вход? (параметры и их типы)
3. **Что возвращает** и когда кидает ошибки?

#### 2. JSDoc как стандарт

**JSDoc** — это стандарт документирования JavaScript-кода. Это обычные комментарии, начинающиеся с `/**` (две звёздочки), которые располагаются **перед** объявлением функции.

```javascript
/**
 * Посимвольное сравнение двух строк на равенство.
 *
 * @param {string} a — первая строка
 * @param {string} b — вторая строка
 * @returns {boolean} — true если строки равны, false если нет
 * @throws {TypeError} — если любой из аргументов не строка
 *
 * @example
 *   isEqual('hello', 'hello'); // true
 *   isEqual('hello', 'world'); // false
 */
export function isEqual(a, b) {
  // тело функции
}
```

**Преимущества JSDoc:**
- Редактор (VS Code) показывает документацию при наведении на вызов функции
- Инструменты вроде `jsdoc-to-markdown` генерируют из JSDoc красивую документацию в HTML/Markdown
- Это стандарт, принятый во всей JS-экосистеме

#### 3. Основные теги JSDoc

| Тег | Описание | Пример |
|-----|----------|--------|
| `@param {type} name` | Описывает параметр функции | `@param {string} str — исходная строка` |
| `@param {type} name` (с `—`) | Тире отделяет имя от описания | `@param {number} count — количество повторений` |
| `@returns {type}` | Описывает возвращаемое значение | `@returns {boolean} — true если найдено` |
| `@throws {type}` | Описывает исключение | `@throws {TypeError} — если аргумент не строка` |
| `@example` | Пример использования (с новой строки) | `@example\n//   isEqual('a', 'b'); // false` |

**Типы в JSDoc:**
- `{string}`, `{number}`, `{boolean}` — примитивные типы
- `{any}` — любой тип
- `{string[]}` — массив строк
- `{object}` — объект
- `{undefined}` — отсутствие значения

#### 4. Стиль оформления

Принятые соглашения (используем эти правила):

1. **Описание функции** — одно предложение на отдельной строке после `/**`
2. **Пустая строка** перед каждой группой тегов (для читаемости)
3. **`@param`** — сначала тип, потом имя, потом `—` и описание
4. **`@returns`** — тип, потом `—` и описание
5. **`@throws`** — после `@returns`
6. **`@example`** — в самом конце, с отступом в 2 пробела для кода
7. **Закрывающий `*/`** — на отдельной строке

**Пример плохой документации:**
```javascript
/** @param a @param b @returns boolean */
function isEqual(a, b) { ... }
```

**Пример хорошей документации:**
```javascript
/**
 * Посимвольное сравнение двух строк на равенство.
 *
 * @param {string} a — первая строка
 * @param {string} b — вторая строка
 * @returns {boolean} — true если строки посимвольно равны, false если нет
 * @throws {TypeError} — если любой из аргументов не строка
 *
 * @example
 *   isEqual('hello', 'hello'); // true
 *   isEqual('hi', 'hello');    // false
 */
export function isEqual(a, b) { ... }
```

#### 5. Что документируем в этом проекте

Начиная с этого проекта, **каждая** написанная функция должна быть задокументирована в стиле JSDoc **перед реализацией** (сразу после заглушки). Это часть TDD-цикла:

1. RED — написал тесты, заглушка возвращает неверный результат
2. Написал JSDoc к заглушке
3. GREEN — реализовал функцию
4. REFACTOR — улучшил код, проверил что документация всё ещё верна

#### 6. Инструменты и IDE

VS Code (и другие редакторы) понимают JSDoc и показывают подсказки:

- **Наведи мышку** на вызов `isEqual(...)` — увидишь сигнатуру и описание
- **Начни печатать** `/**` перед функцией и нажми Enter — VS Code сгенерирует шаблон
