# Шаги урока: Функции includes(), startsWith(), endsWith()

---

### Шаг 1 — includes()

**kind:** `text`

> Убедись, что ты в ветке `feat/substring-search`: `git switch feat/substring-search`

Реализуй функцию `includes(str, search)` — проверка наличия подстроки. Создай файлы `includes.js` и `includes.test.js`.

**Функция принимает:**
- `str`: `string` — строка, в которой ищем
- `search`: `string` — подстрока, которую ищем

**Функция возвращает:** `boolean` — `true` если подстрока найдена, `false` если нет.

Тестовые случаи:
- Должна вернуть `0` для подстроки в начале (`'hello'`, `'he'`)
- Должна вернуть `2` для подстроки в середине (`'hello'`, `'ll'`)
- Должна вернуть `3` для подстроки в конце (`'hello'`, `'lo'`)
- Должна вернуть `-1` если подстрока не найдена (`'hello'`, `'li'`)
- Должна вернуть `0` для пустой поисковой строки (`'hello'`, `''`)
- Должна вернуть `-1` если поисковая строка длиннее исходной (`'hi'`, `'hello'`)
- Должна вернуть `0` для одинаковых строк (`'abc'`, `'abc'`)
- Должна вернуть индекс первого вхождения при нескольких совпадениях (`'ababa'`, `'ba'`) → `1`
- Должна найти подстроку, которая начинается внутри предыдущего почти-совпадения (`'abababc'`, `'ababc'`) → `2`
- Должна вернуть `0` для поиска одного символа (`'hello'`, `'h'`)
- Должна вернуть `2` для поиска повторяющегося символа (`'hello'`, `'l'`)
- Должна работать с кириллицей (`'привет'`, `'иве'`) → `2`
- Должна выбросить `TypeError` если первый аргумент не строка (`123`, `'hello'`)
- Должна выбросить `TypeError` если второй аргумент не строка (`'hello'`, `123`)

### Шаг 2 — startsWith()

**kind:** `text`

Реализуй функцию `startsWith(str, search)` — проверка, начинается ли строка с подстроки. Создай файлы `starts-with.js` и `starts-with.test.js`.

**Функция принимает:**
- `str`: `string` — исходная строка
- `search`: `string` — подстрока для проверки начала

**Функция возвращает:** `boolean` — `true` если `str` начинается с `search`.

Тестовые случаи:
- Должна вернуть `true` для одного символа `('hello', 'h')`
- Должна вернуть `true` для подстроки `('hello', 'hel')`
- Должна вернуть `false` для несовпадающего вхождения `('hello', 'el')`
- Должна вернуть `true` для пустой поисковой строки (`'hello'`, `''`)
- Должна вернуть `false` если поисковая строка длиннее (`'hell'`, `'hello'`)
- Должна вернуть `true` для точного совпадения (`'abc'`, `'abc'`)
- Должна выбросить `TypeError` если первый аргумент не строка

### Шаг 3 — endsWith()

**kind:** `text`

Реализуй функцию `endsWith(str, search)` — проверка, заканчивается ли строка подстрокой. Создай файлы `ends-with.js` и `ends-with.test.js`.

**Функция принимает:**
- `str`: `string` — исходная строка
- `search`: `string` — подстрока для проверки конца

**Функция возвращает:** `boolean` — `true` если `str` заканчивается на `search`.

Тестовые случаи:
- Должна вернуть `true` для одного символа `('hello', 'o')`
- Должна вернуть `true` для подстроки `('hello', 'llo')`
- Должна вернуть `false` для несовпадающего вхождения `('hello', 'ell')`
- Должна вернуть `true` для пустой поисковой строки
- Должна вернуть `false` если поисковая строка длиннее
- Должна вернуть `true` для точного совпадения (`'abc'`, `'abc'`)
- Должна вернуть `true` для одного символа в конце (`'hello'`, `'o'`)
- Должна выбросить `TypeError` если первый аргумент не строка

**kind:** `text`

```
git add includes.js includes.test.js starts-with.js starts-with.test.js ends-with.js ends-with.test.js

```
