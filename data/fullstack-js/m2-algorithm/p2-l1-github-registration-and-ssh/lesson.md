# Регистрация на GitHub и SSH-ключ

**Краткое содержание:**
Урок знакомит с платформой GitHub. Студент создаёт аккаунт, настраивает SSH-доступ и проверяет соединение с GitHub.

### 1. Что такое GitHub

GitHub — крупнейший веб-сервис для хостинга Git-репозиториев. Если Git — это программа для отслеживания версий на твоём компьютере, то GitHub — это место, где эти версии хранятся в облаке и доступны другим разработчикам.

**Зачем нужен GitHub:**
- **Резервная копия** — код не потеряется, даже если сломается компьютер
- **Совместная работа** — коллеги видят твой код, могут предлагать изменения
- **Pull Request (PR)** — механизм предложения изменений с последующим ревью
- **Портфолио** — твой публичный код могут увидеть работодатели
- **Issues, Projects, Actions** — инструменты для управления задачами и автоматизации

**Аналогия:**
- Git = редактор документов с историей изменений (на твоём компьютере)
- GitHub = Google Docs (облачный сервис, где документы доступны всем, у кого есть ссылка)

### 2. Регистрация на GitHub

1. Перейди на [github.com](https://github.com/)
2. Нажми **Sign up**
3. Введи email, пароль и имя пользователя (username)
4. Пройди верификацию (капча или пазл)
5. Подтверди email — на почту придёт письмо с кодом
6. На странице настройки выбери **Free** план (его достаточно)

**Рекомендации по username:**
- Используй что-то профессиональное: `ivan-petrov`, а не `coolgamer2005`
- Username будет частью URL твоих репозиториев: `github.com/ivan-petrov/мой-проект`
- Его можно будет изменить в настройках, но лучше выбрать сразу

### 3. Что такое SSH и зачем он нужен

**SSH (Secure Shell)** — протокол безопасного соединения между компьютерами. GitHub использует SSH, чтобы ты мог отправлять (push) и получать (pull) код без постоянного ввода пароля.

**Как это работает:**
1. Ты генерируешь **пару ключей** на своём компьютере: приватный (секретный) и публичный (открытый)
2. Публичный ключ добавляешь в настройки GitHub
3. При push/pull GitHub проверяет, что у тебя есть соответствующий приватный ключ — и пускает

**Аналогия:** SSH-ключи — это как замок и ключ. Публичный ключ = замок (вешаешь на GitHub), приватный = ключ (хранишь на компьютере).

### 4. Генерация SSH-ключа

Открой терминал и выполни:

> **Windows:** используй **Git Bash** (устанавливается вместе с Git) — в нём работают все команды этого урока. Обычный cmd или PowerShell не подойдут.

```bash
ssh-keygen -t ed25519 -C "твой.email@example.com"
```

Если система скажет, что `ed25519` не поддерживается (старые версии), используй RSA:

```bash
ssh-keygen -t rsa -b 4096 -C "твой.email@example.com"
```

**На вопросы инсталлятора:**
- `Enter file in which to save the key` — нажми Enter (сохранит в папку по умолчанию `~/.ssh/`)
- `Enter passphrase` — можно оставить пустым (Enter) или ввести пароль для дополнительной защиты
- `Enter same passphrase again` — повтори (или Enter)

После генерации у тебя появятся два файла:
- `~/.ssh/id_ed25519` — **приватный** ключ (никому не показывай!)
- `~/.ssh/id_ed25519.pub` — **публичный** ключ (его нужно добавить на GitHub)

### 5. Добавление публичного ключа на GitHub

1. Выведи содержимое публичного ключа:
   ```bash
   cat ~/.ssh/id_ed25519.pub
   ```
   (или `cat ~/.ssh/id_rsa.pub`, если использовал RSA)

2. Скопируй вывод — он выглядит примерно так:
   ```
   ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... твой.email@example.com
   ```

3. На GitHub:
   - Нажми на аватар в правом верхнем углу → **Settings**
   - В левом меню: **SSH and GPG keys**
   - Нажми **New SSH key**
   - В поле **Title** напиши что-то понятное: «Мой ноутбук» или «Домашний ПК»
   - В поле **Key** вставь скопированный ключ
   - Нажми **Add SSH key**

### 6. Проверка соединения

```bash
ssh -T git@github.com
```

При первом подключении система спросит:
```
The authenticity of host 'github.com' can't be established.
ED25519 key fingerprint is SHA256:...
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

Введи `yes` и нажми Enter.

Ожидаемый вывод:
```
Hi username! You've successfully authenticated, but GitHub does not provide shell access.
```

Сообщение «does not provide shell access» — это нормально. Оно означает, что GitHub подтвердил твой ключ, но не даёт доступ к командной строке сервера (она тебе и не нужна).
