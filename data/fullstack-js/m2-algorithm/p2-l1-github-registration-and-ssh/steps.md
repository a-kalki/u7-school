### Шаг 2 — Регистрация и SSH

**kind:** `text`

Выполни регистрацию на GitHub и настрой SSH-доступ.

**Регистрация:**

1. Перейди на [github.com](https://github.com/) и нажми **Sign up**
2. Придумай username — профессиональный, который не стыдно показать работодателю (`ivan-petrov`, а не `xXx_cool_guy_2005`)
3. Подтверди email (придёт письмо с кодом)
4. Выбери бесплатный план

**SSH-ключ:**

Сгенерируй SSH-ключ в терминале:

> **Windows:** используй **Git Bash** (ставится вместе с Git), не cmd/PowerShell.

```
ssh-keygen -t ed25519 -C "твой.email@example.com"
```

На все вопросы нажимай Enter (сохранить в папку по умолчанию, без passphrase).

Выведи публичный ключ и скопируй его:

```
cat ~/.ssh/id_ed25519.pub
```

Добавь ключ на GitHub:
- Settings → SSH and GPG keys → New SSH key
- Title: «Мой ноутбук»
- Key: вставь скопированный ключ

### Шаг 3 — Проверка соединения

**kind:** `text`

Проверь, что всё настроено правильно:

```
ssh -T git@github.com
```

Если появится вопрос «Are you sure you want to continue connecting?» — введи `yes`.

Ты должен увидеть:

```
Hi твой-username! You've successfully authenticated, but GitHub does not provide shell access.
```

Сообщение «does not provide shell access» — это нормально. GitHub подтвердил, что ключ рабочий, но не даёт командную строку (она не нужна).

Если видишь `Permission denied` — проверь, что:
- Публичный ключ скопирован полностью (от `ssh-ed25519` до конца строки с email)
- Ключ добавлен именно в **SSH keys**, а не в GPG keys
- Команда `ssh -T` выполнена с `git@github.com` (именно пользователь `git`)
