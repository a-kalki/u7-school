# Шаги урока: Ветка для исправлений по ревью

---

### Шаг 1 — Анализ комментариев ревью

**kind:** `text`

> ⏳ **Этот урок — отложенное действие.** Ты не можешь выполнить его прямо сейчас — нужно дождаться, пока ревьюеры оставят комментарии к твоему PR (урок 3.7). Прочитай шаги чтобы понимать что делать, и смело жми «Выполнено». Когда получишь комментарии ревью — вернись к этому уроку и пройди шаги по-настоящему.

Дождись, пока ревьюеры оставят комментарии к твоему PR проекта 3. Когда получишь комментарии:

1. Прочитай **каждый** комментарий внимательно
2. Раздели их на категории:
   - 🐛 Баги (логические ошибки, неправильное поведение)
   - 🎨 Стиль (имена переменных, форматирование)
   - 📝 Документация (не хватает пояснений)
   - 💡 Предложения по улучшению (можно сделать, но не обязательно)

3. Определи, с чем ты согласен, а с чем — нет. По каждому пункту сформулируй (мысленно или в конспекте) своё решение.

### Шаг 2 — Внесение правок

**kind:** `text`

Для первого опыта используй упрощённый подход: правки прямо в feature-ветке (PR обновится автоматически).

```
git checkout feat/substring-search
```

Внеси исправления. Для каждого исправления — отдельный коммит с осмысленным сообщением:

```
git add файл.js
git commit -m "Исправлено: проверка на Infinity в repeat"
```

Примеры хороших commit-сообщений для правок:
- `Исправлено: проверка типов в indexOf`
- `Добавлен тест на палиндром в reverse`
- `Убран дублирующийся код проверки типов`
- `Поправлены имена аргументов для ясности`

После всех правок:

```
git push
```

(Без `-u` — связь с удалённой веткой уже настроена.)

### Шаг 3 — Ответы на комментарии

**kind:** `text`

На каждый комментарий ревьюера нужно ответить.

**Шаблоны ответов:**

Если исправил:
```
> Исправил, добавил проверку на Infinity. Спасибо, что заметил!
```

Если не согласен и объясняешь:
```
> Оставил как есть, потому что [причина]. [Ссылка на документацию или логическое объяснение].
```

Если комментарий — предложение, и ты его принял:
```
> Хорошая идея, переименовал. Так действительно понятнее.
```

После ответа нажми **Resolve conversation** (кнопка под комментарием) — это покажет, что вопрос закрыт.

---

### Шаг 4 — Запрос повторного ревью

**kind:** `text`

Когда все правки внесены и на все комментарии отвечено:

1. В правой панели PR найди ревьюеров
2. Рядом с именем каждого — иконка «повторного запроса» (две стрелки по кругу)
3. Нажми её для каждого ревьюера

Также можешь написать комментарий в чате:
```
Правки по ревью внесены, жду повторной проверки 🙏
```
