# Ветка для исправлений — краткая выжимка

> Повторение и практика навыков из уроков 2.4–2.5. Выполняется когда ревьюеры оставят комментарии к PR проекта 3.

- **Рабочий процесс:** комментарии ревью → правки в feature-ветке → коммит + пуш → ответ на комментарий → resolve → re-request review
- **Для простоты на первом опыте:** правки прямо в `feat/substring-search`, PR обновляется автоматически при пуше
- **Продвинутый подход:** fix-ветка от feature-ветки, отдельный PR
- **Ответы:** на каждый комментарий. «Исправил, спасибо» или объяснение, почему не согласен
- **Resolve conversation:** закрывает обсуждение, показывает что вопрос решён
- **Re-request review:** кнопка рядом с именем ревьюера в правой панели PR
