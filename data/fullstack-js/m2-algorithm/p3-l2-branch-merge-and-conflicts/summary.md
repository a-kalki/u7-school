# Слияние веток и конфликты — краткая выжимка

- **Merge** — объединение изменений из одной ветки в другую
- **Fast-forward** — если в целевой ветке не было новых коммитов. Git просто двигает указатель вперёд
- **Merge commit** — если были изменения в обеих ветках. Создаётся коммит с двумя родителями
- **Конфликт** — одна и та же строка изменена по-разному в двух ветках. Git не может выбрать — просит разработчика
- Маркеры конфликта: `<<<<<<< HEAD`, `=======`, `>>>>>>> ветка`
- Разрешение: открыть файл → оставить нужное → удалить маркеры → `git add` → `git commit`
- `git merge --abort` — отменить незавершённое слияние
- **Откат (reset):** `git reset --hard <хэш>` — переместить указатель ветки назад. Коммиты становятся недостижимыми, но восстанавливаются через `reflog`
- **Откат (revert):** `git revert <хэш>` — создать коммит-отмену, не трогая историю. Безопасно всегда, используется в shared-ветках
- `git status` показывает конфликтующие файлы
