### Добавление ревьюеров

**kind:** `text`

Добавь ревьюеров в свой PR:

1. Открой PR в репозитории `js-algorithms`
2. В правой панели найди раздел **Reviewers**
3. Добавь двух ревьюеров из своей группы (узнай их GitHub-username в чате)

Если ревьюеры не появляются в поиске — попроси их зайти в репозиторий (тогда GitHub начнёт предлагать их в списке). Для приватного репозитория также потребуется дать им доступ: Settings → Collaborators.

### Практика: проведи мини-ревью

**kind:** `text`

Чтобы понять, как чувствует себя ревьюер, проведи мини-ревью для воображаемого кода. Ниже — реализация функции `isEqual`. Найди и запиши:
- Как минимум одно **предложение** (можно улучшить, но код работает)
- Как минимум одно **критическое замечание** (нужно исправить)

Формулируй по правилам команды:

- Предложение: «Предлагаю сделать X, потому что Y»
- Критическое: «Нужно сделать X, потому что Y»

```js
// Функция сравнивает две строки посимвольно
// и возвращает true если они равны
function isEqual(a, b) {
  if (typeof a !== 'string' || typeof b !== 'string') {
    throw new TypeError('Аргументы должны быть строками');
  }

  for (let i = 0; a[i] !== undefined; i++) {
    let code1 = a[i].charCodeAt(0);
    let code2 = b[i].charCodeAt(0);

    if (code1 !== code2) {
      return false;
    }
  }

  return true;
}
```

Подсказки:
- Есть ли проблема, если строки разной длины?
- Все ли имена переменных понятны?
- Есть ли нарушения правил модуля?

После того как запишешь замечания — сверься с ответами в следующем шаге.

### Разбор мини-ревью

**kind:** `text`

Сверь свои замечания с эталонными:

**Критическое замечание:**
> Нужно добавить проверку на длину строк. Сейчас цикл идёт по `a`, и если `b` короче `a`, то `b[i]` вернёт `undefined`, `undefined.charCodeAt(0)` вызовет ошибку. А если `b` длиннее `a`, то цикл остановится раньше и функция вернёт `true` для неравных строк.

**Предложение:**
> Предлагаю переименовать переменные `code1` и `code2` в `charCodeA` и `charCodeB`. Имена `code1`/`code2` не несут смысловой нагрузки — неясно, чей это код и зачем он.

**Примечание:** `.charCodeAt(0)` — это разрешённый метод по правилам модуля (см. урок 1.5), так что здесь нарушения нет.

**Итог ревью:**
- Есть критическое замечание → **Request changes**
- Автор должен исправить проблему с длинами строк

### Правила ревью: памятка

**kind:** `text`

Сохрани себе памятку по правилам ревью. Ты будешь использовать их во всех следующих проектах модуля.

**Тон комментариев:**
- 🟢 **Предложение:** «Предлагаю...», «Можно...» — автор сам решает
- 🔴 **Критическое:** «Нужно...», «Необходимо...» — обязательно к исправлению

**Итог ревью:**
- Только предложения → **Approve** ✅ (PR можно принимать)
- Есть критические замечания → **Request changes** ❌ (PR блокируется до исправления)

**Что проверяем:**
1. Корректность — правильно ли работает код?
2. Граничные случаи — пустые строки, `undefined`, отрицательные числа
3. Правила модуля — не используются ли запрещённые методы?
4. Читаемость — понятны ли имена?
5. Тесты — покрыты ли основные и граничные случаи?

**Правило модуля:** нужно одобрение **двух** ревьюеров.
