# Шаги урока: Таблица символов — ASCII и Unicode

---

### Шаг 1

**kind:** `text`

Компьютер понимает только числа. Каждый символ — это **число** в таблице символов. Когда ты нажимаешь клавишу «A», клавиатура отправляет число 65, а видеокарта знает: «число 65 — нарисовать букву A».

**ASCII** — первая стандартная таблица (1960-е), 128 символов, коды 0–127.

Основные группы ASCII:

| Диапазон | Что |
|----------|-----|
| 10 | Перевод строки (`\n`) |
| 32 | Пробел |
| 48–57 | Цифры `'0'`–`'9'` |
| 65–90 | Заглавные латинские: `'A'`–`'Z'` |
| 97–122 | Строчные латинские: `'a'`–`'z'` |

Полная таблица ASCII с кодами и символами: [asciitable.com](https://www.asciitable.com/)

Таблица Unicode (все символы, включая кириллицу, эмодзи, иероглифы): [unicode-table.com](https://unicode-table.com/)

Ключевая закономерность: **строчная буква = заглавная + 32**. `'a'` = 97, `'A'` = 65. Разница всегда 32.

**Unicode** — расширенная таблица (149 000+ символов), включает кириллицу, иероглифы, эмодзи. Первые 128 символов Unicode совпадают с ASCII.

### Шаг 2

**kind:** `text`

В этом модуле **разрешены** два метода как исключение из правил:

- `.charCodeAt(0)` — получить код символа
- `String.fromCodePoint(код)` — получить символ по коду

Выполни в консоли и проверь коды:

```javascript
console.log('A'.charCodeAt(0));  // 65
console.log('a'.charCodeAt(0));  // 97
console.log('0'.charCodeAt(0));  // 48
console.log(' '.charCodeAt(0));  // 32 (пробел)

// Обратно:
console.log(String.fromCodePoint(65));  // 'A'
console.log(String.fromCodePoint(97));  // 'a'
console.log(String.fromCodePoint(32));  // ' '
```

Проверь закономерность: `'b'.charCodeAt(0) - 'B'.charCodeAt(0)` — всегда 32 для любой латинской пары.

Эти методы пригодятся позже при реализации `upperCase` и `lowerCase`.

### Шаг 3

**kind:** `text`

Строки сравниваются **посимвольно слева направо** по кодам символов. Как только символы различаются — результат определён. Если одна строка закончилась раньше — она меньше.

Этот принцип тебе уже знаком из жизни — именно так упорядочены слова в **словарях** и фамилии в **телефонном справочнике**:

- «Иванов» идёт раньше «Петров», потому что `'И' < 'П'` — на первом же символе порядок определён
- «Иванов» и «Иванчук» — первые три буквы одинаковые, смотрим четвёртую: `'а'` и `'н'` → `'а' < 'н'` → Иванов раньше
- «Иван» и «Иванов» — первые 4 буквы совпадают, в «Иван» символы кончились, в «Иванов» ещё есть → «Иван» раньше (короткое слово идёт перед длинным)
- «иванов» и «Иванов» — а вот тут сюрприз: строчная `'и'` имеет код **больше** чем заглавная `'И'`, поэтому в компьютерной сортировке «Иванов» пойдёт раньше «иванов». В бумажном словаре обычно такого разделения нет — это особенность компьютерного сравнения

Именно поэтому в программировании такое упорядочивание называют **лексикографическим** (от греческого lexikon — словарь + grapho — пишу).

**Разбор на примерах:**

`'ab' < 'ac'`
Первые символы `'a'` равны. Вторые: `'b'` (98) < `'c'` (99) → `true`.

`'aa' < 'aaa'`
Первые два символа равны. В первой строке символы кончились, во второй ещё есть → первая меньше → `true`.

`'b' > 'aaaaa'`
Первый символ: `'b'` (98) > `'a'` (97) → `true`. Длина не имеет значения — первый символ решил.

`'aa' < 'aa '`
Первые два символа равны. В `'aa'` символы кончились, в `'aa '` остался пробел (32) → `'aa'` короче → `true`.

`'Zoo' < 'apple'`
Первый символ: `'Z'` (90) < `'a'` (97) → `true`. Хотя в алфавите Z позже A, код `'Z'` меньше кода `'a'`, потому что **все заглавные меньше всех строчных**.
