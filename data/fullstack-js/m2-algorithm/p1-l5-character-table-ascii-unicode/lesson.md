# Таблица символов: ASCII и Unicode

**Краткое содержание:**
Урок объясняет, как компьютер хранит и сравнивает символы. Студент узнает про кодовые таблицы ASCII и Unicode, поймёт лексикографический порядок и научится получать код символа без использования `.charCodeAt()`. Все граничные случаи сравнения строк разбираются на примерах.

### Подробный конспект

#### 1. Как компьютер хранит символы

Компьютер понимает только числа. Каждый символ (буква, цифра, знак) — это просто **число** в таблице символов. Когда ты нажимаешь клавишу «A», клавиатура отправляет не букву, а число 65. Видеокарта знает, что число 65 — это форма буквы «A», и рисует её на экране.

**Код символа (code point)** — это число, соответствующее символу в таблице.

#### 2. Таблица ASCII

**ASCII (American Standard Code for Information Interchange)** — первая стандартная таблица символов, созданная в 1960-х годах. Содержит 128 символов (коды от 0 до 127).

**Основные группы ASCII:**

| Диапазон | Что хранит |
|----------|------------|
| 0–31 | Управляющие символы (невидимые: перевод строки, табуляция, backspace) |
| 32 | Пробел |
| 48–57 | Цифры `'0'`–`'9'` |
| 65–90 | Заглавные латинские буквы `'A'`–`'Z'` |
| 97–122 | Строчные латинские буквы `'a'`–`'z'` |

**Важные коды (запомнить):**
- `'0'` = 48 (цифры идут подряд: 48–57)
- `'A'` = 65 (заглавные идут подряд: 65–90)
- `'a'` = 97 (строчные идут подряд: 97–122)
- `' '` (пробел) = 32

**Ключевое наблюдение:** строчные буквы на 32 больше заглавных. `'A'` = 65, `'a'` = 97. Разница всегда 32.

#### 3. Таблица Unicode

ASCII покрывает только английский алфавит. Для всех остальных языков (русский, казахский, китайский, арабский, эмодзи) создан **Unicode** — он включает более 149 000 символов.

Первые 128 символов Unicode совпадают с ASCII — это сделано для обратной совместимости. Кириллица (`'а'`–`'я'`, `'А'`–`'Я'`) находится в диапазоне кодов 1040–1103.

#### 4. Как получить код символа БЕЗ charCodeAt

Поскольку нам запрещено использовать `.charCodeAt()`, придумаем свой способ. Идея: создать строку-таблицу из всех возможных символов и найти индекс символа в ней.

```javascript
// Строим таблицу символов: строка, где каждый символ — на своём месте
// Индекс в строке = код символа

function buildCharTable() {
  // Начинаем с ASCII: коды 32–126 (печатные символы)
  // Но проще — создать полную таблицу до нужного диапазона
  // через String.fromCodePoint — обратный метод

  // ЭТОТ МЕТОД МЫ НЕ МОЖЕМ ИСПОЛЬЗОВАТЬ в функциях!
  // Но для демонстрации в уроке — ок.
  let table = '';
  for (let i = 32; i < 1104; i++) {
    table += String.fromCodePoint(i);
  }
  return table;
}
```

**Исключение из правил:** методы `.charCodeAt()` и `String.fromCodePoint()` **разрешены** в этом модуле. Нет смысла заставлять студентов переписывать всю таблицу символов вручную — это отвлечёт от основной темы (алгоритмы сравнения строк).

Для функций сравнения строк (`isMore`, `isLess` и т.д.) мы будем использовать прямое сравнение символов через `<` и `>` — это проще и работает для любых символов Unicode:

```javascript
// Сравнение одиночных символов — РАЗРЕШЕНО
'a' < 'b';   // true  (97 < 98)
'A' < 'a';   // true  (65 < 97)
'я' < 'а';   // false (1103 > 1072 — в кириллице порядок обратный алфавитному)
```

#### 5. Лексикографическое сравнение строк

Строки сравниваются **посимвольно слева направо**. На каждом шаге сравниваются коды символов. Как только символы различаются — результат определён. Если одна строка закончилась раньше — она меньше.

**Правила лексикографического порядка:**

1. **Сравнение идёт посимвольно:** `'ab' < 'ac'` — первые символы равны (`'a' === 'a'`), вторые сравниваются: `'b'` (98) < `'c'` (99) → true

2. **Больше — не значит длиннее:** `'Z'` (90) > `'apple'` — первый символ `'Z'` (90) > `'a'` (97)? Нет! 90 < 97 → `'Z' < 'apple'`

3. **Короткая строка может быть больше длинной:** `'b'` (98) > `'aaaaa'` — первый символ `'b'` (98) > `'a'` (97) → true, несмотря на длину

4. **Если все символы равны, но одна строка длиннее — она больше:** `'aa' < 'aaa'` — первые два символа равны, в первой строке символы кончились, во второй ещё есть → `'aa' < 'aaa'`

5. **Пробел (код 32) меньше любой буквы:** `'aa ' > 'aa'` — два символа равны, в `'aa'` символы кончились, в `'aa '` остался пробел → `'aa' < 'aa '`

#### 6. Разбор граничных случаев

##### Случай 1: Заглавные vs строчные

```javascript
// 'A' = 65, 'a' = 97
'A' < 'a';   // true — все заглавные МЕНЬШЕ строчных

// Следствие:
'Apple' < 'apple';   // true — на первом символе: 65 < 97
'Zoo' < 'apple';     // true — 'Z' (90) < 'a' (97)
```

**Важно:** при сортировке строк с заглавными буквами порядок может быть неочевидным:
```
Сортировка по кодам: Apple, Banana, Zoo, apple, banana, zoo
Алфавитная сортировка: Apple, apple, Banana, banana, Zoo, zoo
```

##### Случай 2: Пробелы и специальные символы

```javascript
' ' < 'A';   // true — пробел (32) < 'A' (65)
' ' < '0';   // true — пробел (32) < '0' (48)

// Строка с пробелом в конце:
'aa' < 'aa '; // true — 'aa' закончилась, в 'aa ' ещё пробел
'aa' < ' a';  // true — первый символ: 'a'(97) > ' '(32)? Нет. Пробел (32) < 'a'(97) — true

// Строка с пробелом в начале:
' a' < 'aa';  // true — первый символ: пробел (32) < 'a' (97)
```

##### Случай 3: Цифры как символы

```javascript
// '0' = 48, '9' = 57
'0' < '9';   // true
'10' < '2';  // true — сравниваем '1' (49) и '2' (50): 49 < 50

// Это НЕ числовое сравнение! Как строки:
'10' < '2';  // true, но как числа: 10 > 2
```

##### Случай 4: Разная длина

```javascript
'hello' < 'hello!';    // true — первые 5 символов равны, первая строка кончилась
'hello!' < 'hello!!';  // true — первые 6 равны, первая кончилась
'aaa' < 'b';           // true — 'a' (97) < 'b' (98)
```

#### 7. Получение кода символа вручную (без charCodeAt)

В этом модуле **разрешено** использовать `.charCodeAt()` и `String.fromCodePoint()`. Вот как с ними работать:

```javascript
// Получение кода символа
'a'.charCodeAt(0);  // 97
'A'.charCodeAt(0);  // 65
'0'.charCodeAt(0);  // 48
' '.charCodeAt(0);  // 32

// Обратно: из кода в символ
String.fromCodePoint(97);  // 'a'
String.fromCodePoint(65);  // 'A'
String.fromCodePoint(32);  // ' '
```

Эти методы пригодятся при реализации `upperCase` и `lowerCase` (проект 8), где нужно превращать `'a'` (97) в `'A'` (65) вычитанием 32.

Для функций сравнения строк (`isMore`, `isLess` и т.д.) мы используем прямое сравнение символов через `<` и `>` — это проще и работает для любых символов Unicode:

```javascript
// Так мы будем сравнивать в isMore и isLess:
if (strA[i] < strB[i]) return false; // или true, зависит от функции
if (strA[i] > strB[i]) return true;
```

#### 8. Сводка: как работает сравнение строк

```
Сравнение 'cat' и 'car':

Шаг 1: 'c' === 'c' → равны, идём дальше
Шаг 2: 'a' === 'a' → равны, идём дальше
Шаг 3: 't' (116) > 'r' (114) → результат: 'cat' > 'car'

Сравнение 'aa' и 'aa ':

Шаг 1: 'a' === 'a' → равны
Шаг 2: 'a' === 'a' → равны
Шаг 3: в 'aa' символы кончились, в 'aa ' остался пробел (32)
       → 'aa' < 'aa ' (короткая строка меньше)

Сравнение 'Z' и 'a':

Шаг 1: 'Z' (90) < 'a' (97) → результат: 'Z' < 'a'
```
