# Сложность алгоритмов — краткая выжимка

⏱ **Видео:** ~20 мин

- **O-нотация** — оценка верхней границы скорости роста алгоритма (константы отбрасываются)
- **O(1)** — константная: `at()`, `push()`. Время не зависит от размера данных
- **O(n)** — линейная: `len()`, `isEqual()`, `map()`, `trim()`. Время растёт прямо пропорционально
- **O(n²)** — квадратичная: `indexOf()` (наивный). Вложенный цикл. 10 000× на больших данных
- **O(log n)** — логарифмическая: деление данных пополам на каждом шаге (бинарный поиск)
- **Правила оценки:** нет циклов → O(1); один цикл → O(n); вложенные → O(n²); рекурсия с делением → O(log n)
