# Сложность алгоритмов: O-нотация

**Краткое содержание:**
Урок знакомит с понятием алгоритмической сложности и O-нотацией. Студент учится оценивать, насколько быстро работает алгоритм при росте входных данных, и анализирует сложность уже реализованных в модуле функций.

---

### 1. Зачем оценивать сложность

Два разработчика написали функцию поиска подстроки. Первый — простым перебором (наш `indexOf`). Второй — алгоритмом Кнута-Морриса-Пратта. На строке из 10 символов разница незаметна. На строке из 10 000 символов первый алгоритм работает 3 секунды, второй — 5 миллисекунд. **Разница в 600 раз.**

Сложность алгоритма — это способ предсказать, как будет расти время выполнения при увеличении объёма данных, **не запуская код**.

### 2. Что такое O-нотация

**O-нотация (Big O)** описывает **верхнюю границу** скорости роста алгоритма. Она отвечает на вопрос: «если входных данных станет в N раз больше, во сколько раз дольше будет работать алгоритм?»

Важно: O-нотация отбрасывает константы. O(2n) = O(n), O(n + 1000) = O(n). Интересует только **порядок роста**.

### 3. Основные классы сложности

#### O(1) — константная

Время выполнения **не зависит** от размера входных данных. Всегда одно и то же количество операций.

**Примеры из модуля:**
- `at(arr, index)` — доступ к элементу по индексу. Одна операция, неважно какой длины массив.
- `push(arr, item)` — добавить в конец. Одно присваивание.

**График:** прямая горизонтальная линия. 1 элемент = 1 мкс, 1 000 000 элементов = 1 мкс.

#### O(n) — линейная

Время выполнения растёт **прямо пропорционально** размеру входных данных. N данных → N операций.

**Примеры из модуля:**
- `len(str)` — цикл по каждому символу. Строка из 10 символов = 10 итераций, из 100 = 100.
- `len(arr)` — то же для массивов.
- `isEqual(a, b)` — в худшем случае (строки равны) проходит по каждому символу.
- `isMore(a, b)` — в худшем случае проходит до конца одной из строк.
- `forEach(arr, callback)` — вызывает callback для каждого элемента.
- `map(arr, callback)` — создаёт новый массив, обрабатывая каждый элемент.
- `filter(arr, callback)` — проверяет каждый элемент.
- `startsWith(str, search)` — сравнивает search.length символов.
- `endsWith(str, search)` — сравнивает search.length символов.
- `reverse(arr)` — проходит половину массива = всё ещё O(n), так как константа ½ отбрасывается.
- `reverse(str)` — проходит всю строку.
- `trim(str)` — два прохода (левый и правый) = O(n).
- `upperCase(str)` / `lowerCase(str)` — посимвольная замена.
- `fill(arr, value)` — заполняет каждый элемент.
- `repeat(str, count)` — создаёт строку длины n×count → O(n×count).

**График:** прямая линия под углом. 10 элементов = 10 мкс, 1 000 000 элементов = 1 секунда.

#### O(n²) — квадратичная

Время выполнения растёт как **квадрат** размера входных данных. Обычно означает **вложенный цикл**: для каждого из N элементов делаем ещё N операций.

**Примеры из модуля:**
- **Наивный `indexOf(str, search)`** — для каждой позиции в строке (N) проверяем все символы подстроки (M). В худшем случае (подстрока не найдена) — примерно N×M операций.
  ```javascript
  for (let i = 0; i < len(str); i++) {        // N итераций
    for (let j = 0; j < len(search); j++) {    // M итераций
      if (str[i + j] !== search[j]) break;
    }
  }
  ```
- **`splice(arr, start, deleteCount)`** — удаление из середины требует сдвига всех последующих элементов:
  ```javascript
  // Удаление первого элемента = сдвиг всех оставшихся
  for (let i = 0; i < len(arr) - 1; i++) {   // N итераций
    arr[i] = arr[i + 1];                      // каждый сдвиг
  }
  ```

**На практике O(n²) ощущается так:**

| Размер данных | O(n) | O(n²) |
|--------------|------|-------|
| 10 | 10 оп. | 100 оп. |
| 100 | 100 оп. | 10 000 оп. |
| 1 000 | 1 000 оп. | 1 000 000 оп. |
| 10 000 | 10 000 оп. | 100 000 000 оп. |

На 10 000 элементах разница между O(n) и O(n²) — **в 10 000 раз**.

#### O(log n) — логарифмическая

Время выполнения растёт как **логарифм** от размера данных. Означает, что на каждом шаге алгоритм **отбрасывает половину** оставшихся данных.

**Пример:** бинарный поиск в отсортированном массиве. Ищем число в массиве из 1 000 000 элементов:

1. Смотрим в середину — число меньше середины? Отбрасываем правую половину (осталось 500 000)
2. Смотрим в середину оставшегося — отбрасываем половину (250 000)
3. ...и так далее. Всего ~20 шагов для 1 000 000 элементов.

Функций с O(log n) в нашем модуле нет (мы не реализовывали бинарный поиск), но это важно знать для общего понимания.

**График:** растёт очень медленно. При N = 1 000 000: O(n) = 1 000 000, O(log n) = 20.

#### Сводная таблица

| Нотация | Название | Рост при N×100 | Примеры из модуля |
|---------|----------|---------------|-------------------|
| O(1) | Константная | Без изменений | `at()`, `push()` |
| O(log n) | Логарифмическая | ×7 | (бинарный поиск) |
| O(n) | Линейная | ×100 | `len()`, `isEqual()`, `map()`, `trim()` |
| O(n²) | Квадратичная | ×10 000 | `indexOf()` (наивный), `splice()` |

### 4. Как оценить сложность своей функции

**Правило 1 — найди циклы:**
- Нет циклов → O(1)
- Один цикл по всем данным → O(n)
- Два последовательных цикла → всё ещё O(n) (константа 2 отбрасывается)
- Вложенный цикл → O(n²) (если оба до n) или O(n×m) (если до разных размеров)

**Правило 2 — рекурсия:**
- Каждый вызов делит данные пополам → O(log n)
- Каждый вызов обрабатывает все элементы и делает рекурсивный вызов → может быть O(n×log n) или O(n²)

**Правило 3 — отбрасывай константы и младшие члены:**
- O(2n + 5) → O(n)
- O(n² + 100n) → O(n²) (n² доминирует при больших n)
- O(n + log n) → O(n)

### 5. Практика: оцени сложность

Для каждой функции из модуля определи её сложность по O-нотации:

| Функция | Сложность | Почему |
|---------|----------|--------|
| `len(str)` | O(n) | Один цикл по всем символам |
| `isEqual(a, b)` | O(n) | В худшем случае сравниваем все символы |
| `isMore(a, b)` | O(n) | В худшем случае проходим до конца одной из строк |
| `indexOf(str, search)` | O(n×m) | Вложенный цикл: каждая позиция × символы подстроки |
| `reverse(str)` | O(n) | Проходим строку целиком |
| `repeat(str, count)` | O(n×count) | count повторений строки длины n |
| `trim(str)` | O(n) | Два прохода по строке |
| `replaceAll(str, s, r)` | O(n×m) | Каждое вхождение подстроки ищем через indexOf |
| `push(arr, item)` | O(1) | Присваивание в конец |
| `unshift(arr, item)` | O(n) | Сдвиг всех элементов вправо |
| `splice(arr, s, d)` | O(n) | Сдвиг элементов после позиции удаления |
| `map(arr, cb)` | O(n) | Один вызов callback на каждый элемент |
| `filter(arr, cb)` | O(n) | Один вызов callback на каждый элемент |
| `flat(arr, depth)` | O(всего элементов) | Каждый элемент обрабатывается один раз |
