# Основы JavaScript: Типы данных, оператор typeof и исключения

## Краткое содержание
В этом уроке мы познакомимся с основными типами данных в JavaScript, разберем, как интерпретатор работает с информацией и почему важно понимать типы данных. Также будет рассмотрен оператор `typeof` для определения типа данных и концепция исключений.

## Подробный конспект

### 1. Типы данных — Пример из жизни и Первое знакомство

**Объяснение концепции:**
В повседневной жизни человек оперирует различными объектами и понимает их типы (например, продукты питания и вещи). В зависимости от типа объекта, мы знаем, что с ним можно делать, а что нельзя (например, продукты можно есть, вещи — нет).

Точно так же, в программировании, интерпретатор (среда выполнения кода) оперирует информацией. Эта информация делится на типы, которые мы называем **типами данных**. В зависимости от типа данных, интерпретатор понимает, какие действия допустимо с ними выполнять, а какие — нет.

**Примеры кода (показаны в консоли браузера):**

```javascript
// Пример 1: Сложение двух чисел
5 + 5;
// Вывод: 10
```
*Замечание:* Интерпретатор понимает, что это числа, и выполняет арифметическое сложение.

```javascript
// Пример 2: Сложение строки и числа
'5' + 5;
// Вывод: "55"
```
*Замечание:* В этом случае, так как один из операндов является строкой, интерпретатор выполняет конкатенацию (склеивание) строк, а не арифметическое сложение. Число `5` неявно преобразуется в строку.

```javascript
// Пример 3: Умножение строки и числа
's' * 5;
// Вывод: NaN
```
*Замечание:* Интерпретатор пытается выполнить арифметическое умножение, но не может преобразовать букву 's' в число. В результате получается `NaN` (Not a Number).

**Основные типы данных в JavaScript:**
*   `number`
*   `string`
*   `boolean`
*   `object`
*   `null` (будет объяснено, почему он относится к типу `object`)
*   `undefined`

### 2. Тип данных: `number`

**Объяснение концепции:**
Тип `number` позволяет хранить любое число: целые, отрицательные, дробные. В JavaScript, в отличие от многих других языков программирования, нет отдельных типов для целых и дробных чисел (например, `int` и `float`). Все числа имеют единый тип `number`. Дробные числа разделяются точкой.

**Примеры кода (со слайда):**

```javascript
let bar = 5;
let temperature = -25;
let pi = 3.1415;
let infinityValue = 5 / 0;
```

**Важные замечания:**
*   **`NaN` (Not a Number):** Это специальное числовое значение, которое обозначает "не число". Оно обычно появляется в ходе ошибок в вычислениях, когда математическая операция не может дать корректное числовое значение (как в примере `'s' * 5;`).
*   **`Infinity`:** Это специальное числовое значение, представляющее математическую бесконечность. Оно может быть получено, например, при делении на ноль (как в примере `5 / 0;`).
*   Как `NaN`, так и `Infinity` имеют тип `number`.

### 3. Оператор `typeof`

**Объяснение концепции:**
Оператор `typeof` позволяет в момент выполнения кода (runtime) узнать тип данных любой переменной или значения. Это очень полезно для отладки и для написания гибкого кода, который может по-разному обрабатывать данные в зависимости от их типа.

**Примеры кода (показаны в консоли):**

```javascript
// Объявим переменную bar и присвоим ей число
let bar = 5;

// Проверим тип переменной bar
typeof bar;
// Вывод: "number"

// Проверим тип отрицательного числа
typeof -5;
// Вывод: "number"

// Проверим тип дробного числа
typeof 3.14;
// Вывод: "number"

// Проверим тип Infinity (результат деления на ноль)
typeof (5 / 0);
// Вывод: "number"

// Проверим тип NaN (результат некорректной операции)
typeof ('s' * 5);
// Вывод: "number"
```
*Замечание:* Несмотря на то, что `NaN` и `Infinity` представляют собой особые состояния, `typeof` все равно возвращает для них `"number"`, подтверждая, что они являются частью числового типа в JavaScript.

### 4. Тип данных: `string` и обратные кавычки

**Объяснение концепции:**
Тип `string` соответствует текстовым данным или строке. Чтобы интерпретатор понимал значение как строку, его необходимо обернуть в одинарные кавычки (`'...'`), двойные кавычки (`"..."`) или обратные кавычки (`` `...` ``).

**Важные замечания:**
*   **Обратные кавычки** (шаблонные строки, template literals) обладают особой функциональностью: они позволяют вставлять значения других переменных прямо внутрь строки с помощью синтаксиса `${переменная}`. Это называется интерполяцией строк.

**Примеры кода (показаны в консоли):**

```javascript
// Объявление строки с одинарными кавычками
let s = 'apple';

// Проверка типа переменной s
typeof s;
// Вывод: "string"

// Объявление числа как строки с двойными кавычками
let n = "5";

// Проверка типа переменной n
typeof n;
// Вывод: "string"
```
*Замечание:* Значение `'5'` хоть и хранит число, но имеет тип `string`. Интерпретатор будет работать с ним как со строкой.

**Пример использования обратных кавычек:**

```javascript
// Объявим числовую переменную
let aCount = 5;

// Объявим строку с использованием обратных кавычек и интерполяции
let f = `У нас есть ${aCount} яблок`;

// Просмотрим содержимое f
f;
// Вывод: "У нас есть 5 яблок"
```
*Замечание:* Интерпретатор подставил значение переменной `aCount` (число `5`) в строку, динамически создав текстовое сообщение.

### 5. Тип данных: `boolean`

**Объяснение концепции:**
Тип `boolean` (или булев тип) предназначен для хранения данных, отвечающих на вопрос в виде "Да" (истина) или "Нет" (ложь). Он имеет всего два возможных значения: `true` (истина) и `false` (ложь).

**Примеры кода (со слайда и показаны в консоли):**

```javascript
// Объявление булевой переменной со значением true
let bool = true;

// Объявление булевой переменной со значением false
let isApple = false;

// Проверка типа переменной isApple
typeof isApple;
// Вывод: "boolean"
```

### 6. Тип данных: `object`

**Объяснение концепции:**
В коде мы можем определять "виртуальные" и сложные "вещи" и работать с ними. Их мы называем **объектами**. Тип данных `object` — это сложный тип данных, позволяющий хранить в одном месте много информации, имеющей один контекст. Объекты реального мира (как велосипед на примере) обладают различными характеристиками (свойствами или атрибутами) и могут выполнять действия (поведение или методы).

**Пример кода (со слайда):**

```javascript
let bike = {
    manufacturer: 'Sparta',
    model: 'Sprinter',
    color: 'Yellow',
    wheels: 2
};
```
*Замечание:* В этом примере объект `bike` (велосипед) имеет свойства `manufacturer`, `model`, `color` (строковые значения) и `wheels` (числовое значение). Все эти свойства объединены в одном объекте, так как они описывают одну "вещь" — велосипед. В JavaScript объекты объявляются с помощью фигурных скобок `{}`.

**Важные замечания:**
*   Подробное изучение объектов, включая их поведение (методы), будет посвящено отдельным урокам.

### 7. Типы данных: `undefined`, `null`

**Объяснение концепции:**
`undefined` и `null` оба говорят об отсутствии значения, но используются в разных ситуациях.

*   **`undefined`:** Означает, что переменная была объявлена, но ей еще не было присвоено никакого значения. Это тип `undefined`.

**Примеры кода (показаны в консоли):**

```javascript
// Объявим переменную person, но не присвоим ей значение
let person;

// Проверим тип переменной person
typeof person;
// Вывод: "undefined"
```

*   **`null`:** Означает, что переменной **намеренно** присвоено "пустое" или "неизвестное" значение. С точки зрения JavaScript, `null` имеет тип `object`. Это историческая особенность языка.

**Примеры кода (показаны в консоли):**

```javascript
// Присвоим переменной person объект (в данном случае пустой объект с именем и возрастом)
person = { name: "Bill", age: 45 };

// Допустим, аккаунт пользователя person был удален, и мы хотим обнулить его данные
person = null;

// Проверим тип переменной person после присвоения null
typeof person;
// Вывод: "object"
```
*Замечание:* Несмотря на то, что `null` семантически означает "ничего", `typeof null` возвращает `"object"`. Это признанная особенность JavaScript.
*   **Их объединяет то, что они оба говорят об отсутствии данных в этой переменной.**
*   **Использование `null` нежелательно:** В некоторых командах программистов принято избегать использования `null` и вместо него использовать `undefined` для обозначения отсутствия значения, чтобы не путать с типом `object`.

### 8. Исключение

**Объяснение концепции:**
Исключение (exception) — это ситуация, когда интерпретатор JavaScript сталкивается с ошибкой, которую не может обработать, и отказывается выполнять код дальше. Поток выполнения программы останавливается.

**Пример кода (со слайда):**

```javascript
5.toUpperCaseCase() + '5'; // exception
```
*Замечание:* Метод `.toUpperCaseCase()` предназначен для строковых значений. Попытка вызвать его на числовом значении `5` приведет к `TypeError`, что и является исключением. Интерпретатор не сможет выполнить эту операцию, так как число не имеет такого метода.

## Ключевые выводы
1.  **Типы данных определяют поведение:** Интерпретатор JavaScript использует типы данных для определения того, какие операции допустимы с конкретными значениями.
2.  **`number` включает дробные, отрицательные, `NaN` и `Infinity`:** Все числовые значения в JS относятся к единому типу `number`, включая специальные значения для некорректных или бесконечных вычислений.
3.  **Строки гибкие:** Строки могут быть объявлены одинарными, двойными или обратными кавычками. Обратные кавычки (шаблонные строки) позволяют легко вставлять переменные.
4.  **`typeof` для интроспекции:** Оператор `typeof` — это основной инструмент для определения типа данных переменной или значения в runtime.
5.  **`undefined` vs `null`:** `undefined` означает, что значение не было присвоено, тогда как `null` — это преднамеренное отсутствие значения, при этом `typeof null` возвращает "object".

## Термины
*   **Интерпретатор (Interpreter):** Программа, которая читает и выполняет код построчно, не компилируя его предварительно в машинный код.
*   **Тип данных (Data Type):** Категория, определяющая тип значения, которое может хранить переменная, и набор операций, которые можно с ним выполнять.
*   **`number`:** Тип данных для числовых значений (целых, дробных, отрицательных, `NaN`, `Infinity`).
*   **`string`:** Тип данных для текстовых значений.
*   **`boolean`:** Тип данных, имеющий два значения: `true` (истина) или `false` (ложь).
*   **`object`:** Сложный тип данных, используемый для хранения коллекций данных и более сложных сущностей.
*   **`undefined`:** Значение, указывающее на то, что переменная была объявлена, но ей не было присвоено никакого значения. Это также самостоятельный тип данных.
*   **`null`:** Значение, указывающее на преднамеренное отсутствие какого-либо значения объекта. `typeof null` возвращает `"object"`.
*   **`NaN` (Not a Number):** Специальное числовое значение, обозначающее результат некорректной или неопределенной математической операции.
*   **`Infinity`:** Специальное числовое значение, представляющее математическую бесконечность.
*   **Оператор `typeof`:** Оператор, возвращающий строку, указывающую тип данных своего операнда.
*   **Шаблонные строки (Template Literals):** Строковые литералы, позволяющие использовать встроенные выражения (интерполяцию строк) и многострочные строки. Обозначаются обратными кавычками (`` `...` ``).
*   **Исключение (Exception):** Событие, которое нарушает нормальный поток выполнения программы и приводит к ее остановке, если оно не было обработано.
