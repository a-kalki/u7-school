# Функция: чёрный ящик, контракт, декомпозиция — краткая выжимка

⏱ **Видео:** 11:45

- **Чёрный ящик** — используем функцию, зная *что* она делает, но не *как* (не вникая в реализацию)
- **Имя функции** должно говорить ЧТО делает, а не КАК; плохо: `recursiveRepeat()`, хорошо: `repeatText()`
- **Контракт функции:** имя (обещание) + параметры (вход) + возврат (результат); клиент обязан соблюдать
- **Декомпозиция** — разбиение сложной задачи на мелкие подзадачи; функции — первый инструмент для этого
- **Принцип:** одна функция — одна ответственность, делает что-то одно и делает хорошо
