# Устройство и работа Backend-слоя

Слой `src/api/` отвечает за оркестрацию бизнес-логики и взаимодействие с внешними интерфейсами.

## Основные компоненты

### Сценарии использования (UseCase)
Центральный элемент API-слоя. Каждый `UseCase` наследуется от базового класса и инкапсулирует одну бизнес-операцию.
- **Ответственность:**
    - Валидация входящей команды через `inputSchema`.
    - Проверка на авторизацию.
    - Проверка политик доступа согласно правил домена.
    - Взаимодействие с другими модулями через фасады модулей.
    - Вызов агрегатов домена.
    - Взаимодействие с репозиториями через `resolve`.
    - Выброс типизированных ошибок (`throwNotFound`, `throwValidation` и т.д.).

### Модуль (Module)
Агрегатор всех UseCase-ов конкретной предметной области.
- **Ответственность:**
    - Регистрация и инициализация списка UseCase-ов.
    - Хранение и передача зависимостей (`Resolve`) в UseCase-ы.
    - Диспетчеризация команд через метод `handle(command)`.
    - Предоставление метаданных команд через `getCommands()` (для генерации OpenAPI).

## Поток обработки запроса

1. **Контроллер ui (REST/CLI):** Принимает внешний запрос, формирует `ModuleCommand`.
2. **Module:** Принимает команду, находит соответствующий `UseCase` по `command.name`.
3. **UseCase:** `execute()`: выполняет логику (права -> домен -> репозиторий).
4. **Контроллер:** Ловит результат или исключение, форматирует ответ в стандартный JSON.
