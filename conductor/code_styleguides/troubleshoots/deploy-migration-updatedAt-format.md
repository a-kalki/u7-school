# Deploy: миграция пишет `updatedAt` в неверном формате

- **Симптомы:** после миграции в логах `json-repo` предупреждение «Некорректный формат даты обновления», поле `updatedAt` содержит пробел вместо `T` (например `"2026-07-09 19:08:07"`).
- **Причина:** valibot `isoDateTime()` требует формат `YYYY-MM-DDTHH:mm` (без секунд, без `Z`). Использование `.replace('T', ' ')` в скрипте миграции ломает формат.
- **Решение:** использовать `new Date().toISOString().slice(0, 16)` — даст `"2026-07-09T19:08"`. Исправлять существующие записи: `s.updatedAt.replace(' ', 'T').slice(0, 16)`.
