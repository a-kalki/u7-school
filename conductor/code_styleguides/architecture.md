# Общая архитектура приложения

## Обзор

Платформа **u7-school** построена по модульному принципу. Приложение разделено на доменные модули, каждый из которых инкапсулирует свою предметную область. Взаимодействие с модулями происходит через команды (commands), что позволяет единообразно подключать разные интерфейсы: консоль, Telegram-бота и REST API.

## Доменные модули

Приложение делится на независимые доменные модули (core, w3school и др.). Каждый модуль — это отдельный npm-пакет в директории `packages/`.

## Двухслойная архитектура модуля

Каждый модуль внутри разделен на три слоя:

```
packages/<module>/
  src/
    domain/       # Слой предметной области (чистый код, без инфраструктуры)
    api/          # Слой приложения (оркестрация, репозитории, use cases)
    ui/           # Слой интерфейса пользователя (фронтенд-компоненты, если применимо)
```

### Слой api: Структура папок

```
src/api/
  uc/             # Объекты сценариев использования (Use Cases)
  repositories/   # Интерфейсы репозиториев
  controllers/    # Точки входа (console, bot, ui/rest)
  module/         # Entry point модуля (module.ts)
```

## Вход в модуль: Команды и Entry Point

Каждый модуль предоставляет единый **entry point** — класс `Module`, который принимает команду и возвращает результат:

```typescript
// packages/<module>/src/api/module/module.ts
module.handle(command: ModuleCommand): Promise<unknown>;
```

### Поток выполнения команды

1. **Interface (Console / Bot / REST):** Формирует `ModuleCommand { name, actorId?, attrs }`.
2. **Entry point модуля `handle(command)`:** Диспетчер команд модуля находит нужный Use Case.
3. **Use Case:**
    - Проверка валидации команды (через `inputSchema`).
    - Проверка прав (через `Policy`).
    - Изменение состояния через агрегат (`Ar`).
    - Сохранение через репозиторий (`Repo`).
4. **Result:** Возврат результата вызывающей стороне.
