# Фасад (Facade)

## Назначение

Файл `domain/facade.ts` содержит интерфейс фасада — контракт для доступа внешних модулей к функциональности текущего модуля. Фасад инкапсулирует внутреннее устройство модуля и предоставляет только необходимые методы.

## Правила

1. Фасад размещается в том модуле, который он представляет (например, `UserFacade` — в `@u7/user/domain`).
2. Интерфейс фасада экспортируется из `domain/index.ts` пакета и доступен для импорта другими модулями через `@u7/<module>/domain`.
3. Фасад содержит только методы, необходимые внешним модулям (принцип минимального интерфейса).
4. Методы фасада возвращают доменные типы (Entity), а не внутренние детали реализации.
5. Фасад не должен раскрывать внутренние репозитории, схемы или API-модули.
6. Фасад не обязан повторять в точности контракт usecase который он оборачивает, например `getUser` может вернуть `User | undefined` обернув обработку исключения `UserNotFound`;
7. Руководствуйся при проектировании api фасада удобством для клиента (другие модули);

## Пример

```typescript
// packages/user/src/domain/facade.ts
import type { User } from "./user/entity";

/**
 * Фасад модуля пользователей для внешних модулей.
 * Предоставляет методы получения информации о пользователях,
 * не раскрывая внутреннее устройство модуля @u7/user.
 */
export interface UserFacade {
  /** Получить пользователя по UUID */
  getUserByUuid(uuid: string, actorId?: string): Promise<User | undefined>;

  /** Проверить, существует ли пользователь с указанным UUID */
  userExists(uuid: string, actorId?: string): Promise<boolean>;
}
```

## Использование в других модулях

```typescript
// Внешний модуль (например, @u7/course) импортирует фасад
import type { UserFacade } from "@u7/user/domain";

// Использует фасад через резолвер зависимостей
export interface CourseApiModuleResolver {
  courseRepo: CourseRepo;
  userFacade: UserFacade;
}
```
