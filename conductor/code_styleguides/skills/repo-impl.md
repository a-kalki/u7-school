# Реализация репозитория — Styleguide

**Назначение:** файл `infra/db/<entity-name>-<type>-repo.ts` — конкретная реализация интерфейса `Repo` из `domain/<entity>/repo.ts`. Например: `UserInmemoryRepo`, `StreamJsonRepo`.

---

## 1. Ключевые правила

1. Реализует интерфейс из `domain/<entity>/repo.ts` — никаких лишних методов.
2. **Только хранение и извлечение** — без бизнес-логики.
3. Методы поиска возвращают `undefined` (не `null`) при отсутствии результата.
4. In-memory реализация обычно на `Map` с индексами по полям поиска (`#byUuid`, `#byTelegramId`).
5. `save()` должен обновлять все индексы (включая перезапись при смене ключевого поля).

Живые примеры: [`packages/stream/src/infra/db/stream-json-repo.ts`](../../../packages/stream/src/infra/db/stream-json-repo.ts), `packages/course/src/infra/db/lesson-json-repo.ts`.

---

## 2. Тестирование

- Каждый метод покрывается тестом; группируйте по методам на втором уровне.
- Тестируйте граничные случаи: пустой репозиторий, перезапись, смена ключевого поля, несуществующие ключи.
- Корректность репозитория проверяется **здесь**, на уровне UC всё мокается.
- Реальная работа с репозиторием дополнительно проверяется на уровне тестов модуля (ограниченный тест подключения).
- Выносите повторяющуюся логику в хелперы (`createTestUser`, `assertUserExists`); тест — не более 10 строк.

Живой пример теста: `packages/stream/src/infra/db/stream-json-repo.test.ts`.
