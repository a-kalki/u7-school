# Агрегат (Aggregate) — Styleguide

**Назначение:** файл `domain/<entity>/a-root.ts` — богатый доменный объект, инкапсулирующий состояние и бизнес-логику изменений. Наследуется от `Aggregate<ArMeta>` из `@u7/core`.

---

## 1. Ключевые правила

1. Принимает состояние через конструктор, проверяет инварианты через схему из `entity.ts`.
2. **Только** фабричные методы создания и методы изменения состояния.
3. Не содержит схему валидации (импортирует из `entity.ts`), не обращается к репозиторию/БД/внешним сервисам.
4. `createdAt`/`updatedAt` — поля **только корня**; value-objects внутри агрегата их не имеют.
5. `updateState`/`safeUpdate` автоматически обновляют `updatedAt`.

Живой пример: [`packages/stream/src/domain/stream/a-root.ts`](../../../packages/stream/src/domain/stream/a-root.ts).

---

## 2. Состояние и мутации

| Геттер | Доступ | Назначение |
|---|---|---|
| `state` | внешний (иммутабельный) | Возвращает `structuredClone` — внешний код не может мутировать состояние |
| `_state` | только внутренние методы | Прямая ссылка на `#state` для мутаций без копирований |

- **После прямых мутаций через `_state`** вызывайте `this.safeUpdate({})` — обновит `updatedAt` и запустит валидацию.
- `updateState(newState)` — полная замена + валидация + авто-`updatedAt`.
- `safeUpdate(partial)` — частичное обновление полей со значением не `undefined`; поля из `safeAttrs` (`uuid`, `createdAt`) не перезаписываются.

```typescript
addModule(command: AddModuleCmd): void {
	this._state.modules.push({ ... });
	this.safeUpdate({}); // триггер updatedAt + валидация
}
```

> Прямая мутация разрешена: агрегат не используется для реактивного рендеринга, иммутабельные обновления не нужны.

### Ошибки консистентности данных

Если метод, возвращающий данные, не находит ожидаемый объект — это ошибка консистентности, выбрасывайте `throwBadRequest`. **Нельзя молча возвращать пустой массив/undefined** — это маскирует баги.

---

## 3. Тестирование

- Тест инвариантов — проверка, что схема подключена (1 тест); обязательны тесты инвариантов с доп. логикой.
- Каждый метод покрывается тестом (кроме очевидных); группируйте по методам.
- Граничные случаи и неудачные сценарии; тест — не более 10 строк.

Живой пример теста: `packages/stream/src/domain/stream/a-root.test.ts`.
