# Спецификация трека Stateless Bot & Controller

## 1. Обзор
Цель трека — переписать взаимодействие между Telegram-ботом и слоем UI Controller. Удаляется использование `grammy/conversations` (чтобы избавиться от нестабильности, связанной с локальным состоянием), и вводится строго типизированный контракт между ботом (слой маршрутизации) и контроллером (слой чистой логики отображения). Бот становится полностью "тупым".

## 2. Функциональные требования
- **Игнорирование групповых чатов**: Бот должен обрабатывать команды и сообщения только из приватных чатов (`ctx.chat?.type === 'private'`). Запросы из групп и каналов должны игнорироваться.
- **Удаление локального состояния**: Бот не должен хранить состояние текущей анкеты (кроме `telegramId`). Модуль `onboardingConversation` удаляется.
- **Типизированный контракт**: В `packages/onboarding/src/ui/bot/app.ts` (или аналогичном файле) определяются типы `BotUpdate` (события от юзера) и `BotResponse` (инструкции для бота).
- **Чистый Контроллер**: `OnboardingController` переписывается для работы как конечный автомат:
    - Метод `handleUpdate(update: BotUpdate): Promise<BotResponse[]>`
    - Внутри использует `get-onboarding-state-uc` для понимания контекста.
    - Возвращает конкретные инструкции (какое сообщение отправить, обновить, какую клавиатуру показать).
- **Умная Маршрутизация (в боте)**: В `apps/u7-bot` настраивается middleware:
    - Все события проверяются: если у пользователя статус `in_progress` в анкете (через UC) или он нажал кнопку в рамках анкеты — событие отправляется в `OnboardingController.handleUpdate`.
    - Иначе обрабатывается главное меню (`/start`, `be_in_the_know`, `become_student` и т.д.).
- **Поддержка /cancel**: Команда прерывания анкеты должна корректно обрабатываться в контроллере или боте, сбрасывая состояние в домене и возвращая в главное меню.

## 3. Критерии приёмки
- Зависимость `@grammyjs/conversations` может быть удалена из проекта бота (или не используется в onboarding).
- Тесты `OnboardingController` переписаны и подтверждают корректное формирование `BotResponse[]` на различные `BotUpdate`.
- Бот корректно проходит сценарий анкеты, включая множественный выбор, без зависаний и крашей при клике на старые кнопки.
- Команда `/cancel` работает корректно в любой момент прохождения анкеты.

## 4. За рамками
- Изменения в домене и API Onboarding (это сделано в первом треке).
- Разработка других контроллеров (для курсов/модулей).
