# Спецификация трека: Витрина для любопытного

> **Контекст эволюции:** `conductor/architecture-evolution.md` (§2.1, §2.2, §2.10, §2.11). Прочитать перед началом.
> **Зависимости:** трек 0 (Course), трек 2 (ContentPath — curious-уровень доступа).

## Обзор

Дать «любопытному» (GUEST/CANDIDATE) полную картину для принятия решения о записи:
- «Программы курсов» — что и как учат (каталог + drill-down структуры).
- «Потоки курсов» — curious-витрина запусков, без менторских инструментов.
- Прозрачность студентов — прогресс и исходы видны всем.

**Роли:** GUEST, CANDIDATE (любопытный); MENTOR/ADMIN — тоже видят curious-ветки, но управление — в треке `mentor_tools_20260713`.

## Функциональные требования

### F1. «Программы курсов» — каталог и структура (S00 → S00a → S00b)
- Главное меню: «📖 Программы курсов» (новый top-level, всем ролям).
- **S00** — список опубликованных курсов: title, краткое описание, эмодзи направления (tech/business), сводка объёма.
- **S00a** — карточка курса: title, описание, автор, phases (этапы с направлениями), сводка «N этапов · M модулей · K проектов · L уроков». Кнопка «📖 Развернуть программу» → S00b. Кнопка «📚 Найти поток» → переход в «Потоки курсов» (отфильтрованный по модулям курса).
- **S00b** — программа курса (drill-down): этап → модуль → проект → урок → **заголовки шагов** (curious-уровень ContentPath, трек 2). У урока: «📝 Урок (5 шагов)», типы шагов иконками. У шага: только заголовок + тип, **без тела** (`content`/`code` скрыты). Если у модуля есть enrollment-поток → «📝 Записаться» прямо отсюда → S10.
- Пагинация при превышении лимита 4096 символов; «⬅️ Назад» на каждом уровне.

### F2. Расширенная программа модуля (внутри S00b и S03)
- Число шагов урока, типы шагов иконками (текст/код/файл), сводка объёма.
- Контент шагов — скрыт, видны только заголовки (curious-уровень ContentPath).

### F3. «Потоки курсов» — curious-режим карточки потока (S02)
- Главное меню: «📚 Потоки курсов» (переименование «Потоки курсов», всем).
- **S02 curious-режим**: только curious-функциональность, **без lifecycle-кнопок ментора** (см. §2.10):
  - `📖 Программа` (модуля, любой статус) → S03. Сейчас только `enrollment` — снять ограничение.
  - `📋 Детали` (любой статус) → S04. Сейчас только `enrollment` не-владельцу — снять ограничение.
  - `📊 Студенты` → S07 (прозрачность, всем).
  - `📚 Курс: Fullstack JS` — строчка в теле карточки (техдолг: захардкожено, ждёт `courseId` в Stream).
    В будущем: `courseId` → `appApi.execute('get-course')` → реальное имя + кнопка-ссылка на S00a.
  - Гостевая часть (только не-владелец): `📝 Записаться` (`enrollment`).
- Ментор в этой ветке — обычный зритель, без инструментов управления.

### F3a. Прозрачность студентов (S07, S08 — всем ролям)
- S07 (список) и S08 (карточка студента) доступны **всем ролям**, не только ментору.
- Curious видит: прогресс-бары, статусы (`active`/`abandoned`/`advanced`/`not_advanced`), позиции, историю шагов, динамику и исходы. Цель — показать, как учатся студенты, кого отчислили за халтуру, кто каких успехов достиг.
- Кнопки **действий** над студентом (`⚠️ Неактивен` → `mark-abandoned`, `✅ Завершить`/`🔄 Сменить исход` → `complete-student`) — только ментор потока / ADMIN (в mentor-режиме S02m, трек `mentor_tools_20260713`).
- Контент S07/S08 остаётся как есть; богатая переработка (сводка здоровья, сортировка отстающих, история шагов полноценно, динамика) — трек `mentor_tools_20260713`.

## Критерии приёмки
- [x] «📖 Программы курсов» → единое 5-уровневое дерево (курсы→этапы→модули→проекты→уроки+шаги, без тел).
- [x] Программа курса показывает объём (проекты, уроки, шаги, типы).
- [x] «📚 Потоки курсов» → S02 curious-режим: Программа/Детали/Студенты на любом статусе.
- [x] В S02 curious-режиме **нет** lifecycle-кнопок ментора (Запустить/Завершить/Архив).
- [x] S07/S08 доступны всем ролям; curious видит прогресс/статусы/исходы студентов. (✅ код уже был публичным, добавлены тесты на GUEST/CANDIDATE/все-статусы)
- [x] Кнопки действий над студентом — только ментору/админу.
- [x] Покрытие >80%, TDD (1366 pass, 4 skip, 0 fail).
