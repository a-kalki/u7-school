# Спецификация трека: Витрина для любопытного + Инструменты ментора

> **Контекст эволюции:** `conductor/architecture-evolution.md` (§2.1, §2.2, §2.8, §2.10, §2.11). Прочитать перед началом.
> **Зависимости:** трек 0 (Course), трек 2 (ContentPath — curious-уровень доступа).

## Обзор

Дать «любопытному» (GUEST/CANDIDATE) полную картину для решения о записи и выстроить чёткое разделение веток: «Программы» (что учат), «Потоки» (curious-витрина запусков, без менторских инструментов), «Инструменты ментора» (управление потоками). Студенты и их прогресс/исходы **открыты всем** (прозрачность: кто как учился, кого отчислили, кто каких успехов достиг).

**Роли:** GUEST, CANDIDATE (любопытный); MENTOR/ADMIN — управление; AUTHOR — отложено (§2.8).

## Функциональные требования

### F1. «Программы» — каталог курсов и структура (S00 → S00a → S00b)
- Главное меню: «📚 Программы» (новый top-level, всем ролям).
- **S00** — список опубликованных курсов: title, краткое описание, эмодзи направления (tech/business), сводка объёма.
- **S00a** — карточка курса: title, описание, автор, phases (этапы с направлениями), сводка «N этапов · M модулей · K проектов · L уроков». Кнопка «📖 Развернуть программу» → S00b. Кнопка «📚 Найти поток» → переход в «Потоки» (отфильтрованный по модулям курса).
- **S00b** — программа курса (drill-down): этап → модуль → проект → урок → **заголовки шагов** (curious-уровень ContentPath, трек 2). У урока: «📝 Урок (5 шагов)», типы шагов иконками. У шага: только заголовок + тип, **без тела** (`content`/`code` скрыты). Если у модуля есть enrollment-поток → «📝 Записаться» прямо отсюда → S10.
- Пагинация при превышении лимита 4096 символов; «⬅️ Назад» на каждом уровне.

### F2. Расширенная программа модуля (внутри S00b и S03)
- Число шагов урока, типы шагов иконками (текст/код/файл), сводка объёма.
- Контент шагов — скрыт, видны только заголовки (curious-уровень ContentPath).

### F3. «Потоки» — curious-режим карточки потока (S02)
- Главное меню: «📚 Потоки» (переименование «Наши потоки», всем).
- **S02 curious-режим**: только curious-функциональность, **без lifecycle-кнопок ментора** (см. §2.10):
  - `📖 Программа` (модуля, любой статус) → S03. Сейчас только `enrollment` — снять ограничение.
  - `📋 Детали` (любой статус) → S04. Сейчас только `enrollment` не-владельцу — снять ограничение.
  - `📊 Студенты` → S07 (прозрачность, всем).
  - `📚 Курсы` → S00a (курсы, в которые входит модуль потока). Новая кнопка.
  - Гостевая часть (только не-владелец): `📝 Записаться` (`enrollment`), `🔔 Уведомить о наборе` (`active`), `🔔 Ждать новый набор` (`completed`, F4).
- Ментор в этой ветке — обычный зритель, без инструментов управления.

### F3a. Прозрачность студентов (S07, S08 — всем ролям)
- S07 (список) и S08 (карточка студента) доступны **всем ролям**, не только ментору.
- Curious видит: прогресс-бары, статусы (`active`/`abandoned`/`advanced`/`not_advanced`), позиции, историю шагов, динамику и исходы. Цель — показать, как учатся студенты, кого отчислили за халтуру, кто каких успехов достиг.
- Кнопки **действий** над студентом (`⚠️ Неактивен` → `mark-abandoned`, `✅ Завершить`/`🔄 Сменить исход` → `complete-student`) — только ментор потока / ADMIN (в mentor-режиме S02m).
- Контент S07/S08 остаётся как есть; богатая переработка (сводка здоровья, сортировка отстающих, история шагов полноценно, динамика) — трек `mentor_monitoring_20260708`.

### F4. Предзапись на completed-поток
- «🔔 Ждать новый набор» на completed-потоке (модуль которого есть в курсе) → подписка на уведомление о новом потоке этого модуля.
- (Хранилище подписок — мини-репо; уведомление — через TgFacade при появлении enrollment-потока, если решим.)

### F5. «Инструменты ментора» (главное меню + mentor-режим S02m)
- Главное меню MENTOR: «🛠️ Инструменты ментора» → подменю:
  - «📋 Мои потоки» — потоки, где я ментор → **S02m mentor-режим**.
  - «➕ Создать поток» → S09 (перенос `CreateStreamStory.handleStart` на 2-й уровень).
  - «👥 Мониторинг» — быстрый вход в активные потоки → S07.
- **S02m (mentor-режим)**: содержимое карточки + панель управления `🚀 Запустить` / `✅ Завершить` / `📁 В архив` + `👥 Студенты` → S07/S08 **с кнопками действий ментора** + `📊 Сводка потока` (счёт по статусам; медиана/отстающие/сортировка — трек 6).
- `CreateStreamStory.handleStart` → второй уровень (под «Инструментами ментора»), не в корне меню.
- Секция role-gated (`MENTOR`).

> **«Инструменты автора» (AUTHOR) в этот трек НЕ входят** — отложено (§2.8). В `ui-spec.md` фиксируется краткое описание ответственности, реализации нет.

## Критерии приёмки
- [ ] «📚 Программы» → каталог курсов → карточка курса → программа (drill-down до заголовков шагов, без тел).
- [ ] Программа модуля показывает объём (шаги, типы).
- [ ] «📚 Потоки» → S02 curious-режим: Программа/Детали на любом статусе, кнопка «Курсы», «Студенты».
- [ ] В S02 curious-режиме **нет** lifecycle-кнопок ментора (Запустить/Завершить/Архив).
- [ ] S07/S08 доступны всем ролям; curious видит прогресс/статусы/исходы студентов.
- [ ] Кнопки действий над студентом — только ментору/админу.
- [ ] «🛠️ Инструменты ментора» → «Мои потоки» → S02m (lifecycle + Студенты с действиями) + «Создать поток» + «Мониторинг».
- [ ] Предзапись работает (или заглушка с уведомлением «сообщим»).
- [ ] Покрытие >80%, TDD.

## За рамками
- «Инструменты автора» (UI) — отложено, только в документации (§2.8).
- Переработка контента S07/S08 (сводка здоровья, сортировка, история шагов, динамика) — трек 6.
- Отзывы выпускников (бэклог).
- Полноценная веб-версия витрины.
- Реальная рассылка уведомлений о наборе (нужен TgFacade + scheduler — бэклог).
