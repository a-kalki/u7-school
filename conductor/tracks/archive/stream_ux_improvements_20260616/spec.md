# Спецификация трека: Stream UX Improvements

## Обзор
Два UX-улучшения модуля stream:
1. **Мгновенный переход к следующему шагу** — после нажатия «Выполнено» студент сразу видит следующий шаг с кнопками «Выполнено» + «Мой прогресс», без промежуточного сообщения «Шаг выполнен!».
2. **Автозаполнение полей потока из модуля** — при создании потока через wizard поля `title`, `description`, `goal`, `result`, `rules`, `targetAudience`, `additional` подтягиваются из выбранного модуля как значения по умолчанию.

**Роли:** STUDENT, MENTOR, ADMIN.

## Функциональные требования

### FR-1: Мгновенный переход к следующему шагу
- После вызова `complete-step` студент **сразу** видит карточку следующего шага с кнопками `[✅ Выполнено]` и `[📊 Мой прогресс]`.
- Промежуточное сообщение «Шаг выполнен! Следующее задание уже ждёт.» **не показывается**.
- При завершении урока (`level === 'lesson'`) — показывается поздравление вида «🎉 Урок «...» завершён!» и кнопка `[▶️ Начать следующий урок]`.
- При завершении проекта (`level === 'project'`) — показывается поздравление вида «🚀 Проект «...» завершён!» и кнопка `[▶️ Начать следующий проект]`.
- При завершении всего потока (`level === 'stream'`) — текущее поведение сохраняется: «🏆 Поток полностью завершён!».
- Кнопка «Мой прогресс» остаётся доступной на любом шаге.

### FR-2: Автозаполнение полей потока из модуля
- В wizard создания потока (CreateStreamStory) после выбора модуля:
  - Поле «Название потока» предзаполняется значением `module.title`.
  - Поле «Описание» предзаполняется значением `module.description`.
- Пользователь может изменить предзаполнённые значения (оставить как есть или ввести свои).
- После шага с датой старта **каждое** необязательное поле модуля (`goal`, `result`, `rules`, `targetAudience`, `additional`) запрашивается **отдельным шагом**:
  - Если в модуле есть значение для поля — показывается «По умолчанию: ...» + кнопка **«Принять»** (использует значение модуля).
  - Если в модуле нет значения — просто предлагается ввести текст.
  - На каждом шаге есть кнопка **«Пропустить»** (поле остаётся пустым).
  - Порядок шагов: цель → результат → правила → аудитория → дополнительно → группа.
- При confirm, если `create-stream` возвращает ошибку валидации — показывается сообщение вида «Ошибка валидации: поле X требует Y», данные сбрасываются, пользователь начинает заново.

## Нефункциональные требования
- Изменения не ломают существующие тесты LearningStory и CreateStreamStory.
- Покрытие нового кода тестами >80%.

## Критерии приёмки
- [ ] После «Выполнено» студент сразу видит следующий шаг (без промежуточного сообщения)
- [ ] При завершении урока — поздравление + кнопка «Начать следующий урок»
- [ ] При завершении проекта — поздравление + кнопка «Начать следующий проект»
- [ ] При создании потока поля title и description предзаполнены из модуля
- [ ] Каждое необязательное поле модуля запрашивается отдельным шагом с подсказкой «По умолчанию» (если есть) и кнопками «Принять» / «Пропустить»
- [ ] Нажатие «Принять» подставляет значение модуля, «Пропустить» — оставляет пустым
- [ ] При ошибке валидации create-stream показывается сообщение с деталями и данные сбрасываются
- [ ] Все существующие тесты проходят
