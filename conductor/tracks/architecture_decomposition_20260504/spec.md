# Спецификация трека: Декомпозиция архитектурной документации

## Обзор
Текущий файл архитектуры проекта (`docs/architecture.md`) является монолитным (более 200 строк), что затрудняет поиск специфической информации для разработчиков и агентов ИИ. Цель данного трека — разделить этот документ на набор небольших, сфокусированных руководств (guidelines) и разместить их в директории `conductor/code_styleguides/`, а также обновить ссылки в корневом `README.md` и `conductor/product-guidelines.md` для обеспечения прямого и удобного доступа.

## Функциональные требования

### FR1: Декомпозиция монолита
Извлечь содержимое `docs/architecture.md` и распределить его по следующим новым файлам в папке `conductor/code_styleguides/`:
1. **`architecture.md`** — Общая архитектура (модули, слои domain/api, entry points).
2. **`ddd.md`** — Принципы DDD (агрегаты, инварианты, команды, схемы).
3. **`errors.md`** — Хранение и обработка ошибок (иерархия исключений, AppException, уровни).
4. **`naming.md`** — Соглашения по именованию (kebab-case, суффиксы Ar, Uc, Schema).
5. **`testing.md`** — Правила тестирования (изоляция, проверка ошибок, размер тестов до 10-20 строк).
6. **`integration.md`** — Межмодульное взаимодействие & DI (фасады, передача ModuleResolve).

### FR2: Обновление ссылок
- Файл `docs/architecture.md` должен быть удален после миграции.
- В корневой файл `README.md` (или `docs/README.md`, если он есть) добавить раздел со ссылками на новые декомпозированные файлы.
- В файл `conductor/product-guidelines.md` добавить ссылки на новые архитектурные гайдлайны для удобства ИИ-агента.

## Нефункциональные требования
- Никакой потери информации: все важные концепции из оригинального файла должны быть сохранены в новых документах.
- Документы должны быть написаны в формате Markdown (.md).

## Критерии приёмки
- [ ] Созданы 6 новых файлов в папке `conductor/code_styleguides/`.
- [ ] Содержимое `docs/architecture.md` полностью перенесено в новые файлы.
- [ ] Монолитный файл `docs/architecture.md` удалён.
- [ ] Обновлены ссылки в `README.md`.
- [ ] Обновлены ссылки в руководствах Conductor.

## За рамками
- Изменение текущих архитектурных правил (это просто миграция документации).
