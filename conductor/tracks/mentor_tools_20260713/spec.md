# Спецификация трека: Инструменты ментора

> **Контекст эволюции:** `conductor/architecture-evolution.md` (§2.3, §2.8, §2.10, §2.11). Прочитать перед началом.
> **Зависимости:** трек 0 (Course), трек 1 (ЖЦ студента), трек 2 (ContentPath), трек 5 (curious_showcase_20260708 — прозрачность S07/S08).

## Обзор

Объединённый трек: всё, что нужно ментору для управления потоками и мониторинга студентов.

**Часть A — Управление потоками:** выделить управление в отдельное role-gated пространство «🛠️ Инструменты ментора». Сейчас кнопка «➕ Создать поток» висит в корне главного меню, а lifecycle-кнопки смешаны с curious-кнопками. Цель — навести порядок: curious-ветка «Потоки курсов» (трек `curious_showcase_20260708`) — только обзорная, без менторских инструментов.

**Часть Б — Мониторинг студентов:** автоматическое выявление отстающих, сводка здоровья группы, сортировка проблемных сверху, детальная история шагов студента и динамика обучения.

**Роли:** MENTOR, ADMIN — управление потоками. S07/S08 контент — всем ролям (прозрачность из трека 5). Кнопки действий над студентом — только ментор потока / ADMIN.

## Часть А: Управление потоками

### F1. Главное меню «🛠️ Инструменты ментора»
- Подменю (только для MENTOR/ADMIN):
  - «📋 Мои потоки» — потоки, где пользователь является ментором → **S02m mentor-режим**.
  - «➕ Создать поток» — перенос `CreateStreamStory.handleStart` с корневого уровня на второй.
  - «👥 Мониторинг» — быстрый вход в активные потоки → S07 (список студентов).

### F2. S02m — mentor-режим карточки потока
- Содержимое карточки + панель управления lifecycle:
  - `🚀 Запустить` (enrollment → active)
  - `✅ Завершить` (active → completed)
  - `📁 В архив`
- `👥 Студенты` → S07/S08 **с кнопками действий ментора**.
- `📊 Сводка потока`: счёт по статусам (active/advanced/not_advanced/abandoned/enrolled). Медиана, отстающие, сортировка — см. Часть Б.

### F3. Curious-ветка — БЕЗ lifecycle-кнопок (✅ уже сделано)
- В треке `curious_showcase_20260708` lifecycle-кнопки убраны из `ViewStreamStory.#buildKeyboard`.

### F4. Перенос «Создать поток» на второй уровень
- `CreateStreamStory.handleStart` → `null` (не участвует в главном меню). Вход — через подменю.

## 🏗️ Реализация S02m — варианты и контекст

**Что уже готово (трек `curious_showcase_20260708`):**
- `ViewStreamStory` — curious-карточка: просмотр, программа, детали, студенты. Без lifecycle-кнопок.
- `ViewStreamMentorStory` — менторские хендлеры: `complete`, `archive`, `complete-confirm`, `archive-confirm` + confirm-диалоги. Лежит в `packages/stream/src/ui/bot/stories/view-stream-mentor.story.ts`, тесты в `describe.skip`.
- `ActivateStreamStory` — уже зарегистрирован в `StreamController`, работает.
- Кнопки S02m описаны в `ui-spec.md` (строки 192–199): Запустить/Завершить/В архив + Студенты с действиями.

**Вариант А: Зарегистрировать ОБА сторис в MentorController**
- `MentorController` регистрирует `ViewStreamStory` (curious) + `ViewStreamMentorStory` (lifecycle).
- Ментор видит curious-карточку (свою) и lifecycle-кнопки (mentor-сторис).
- Плюс: без дублирования кода, `ViewStreamStory` используется как есть.
- Минус: две разные сторис в одном контроллере → клавиатура собирается из двух мест.

**Вариант Б: S02m = дочерний сторис с собственной клавиатурой**
- `ViewStreamMentorStory` не только обрабатывает lifecycle, но и строит полную S02m-клавиатуру (curious + lifecycle).
- Либо наследует и переопределяет `#buildKeyboard` (но `#`-приватные методы в JS не наследуются — потребуется рефакторинг `#` → `protected`).
- Либо дублирует `#handleView` в менторском сторис. Плюс: одна точка входа, одна клавиатура.
- Минус: дублирование или рефакторинг приватных методов.

**Рекомендация:** Вариант А проще в реализации, вариант Б чище архитектурно (одна клавиатура). Решение — на усмотрение разработчика трека.

## Часть Б: Мониторинг студентов

> **Важно:** действия ментора в S08 (`mark-abandoned`, `complete-student`) уже реализованы в треке 1. Эта часть **не создаёт их заново** — добавляет bot-level тесты и перерабатывает **контент** S07/S08.

### F5. Логика «Отстаёт» (S07, S08)
- Статус «⚠️ Отстаёт» если:
  - `completedAt` последнего шага старше 5 дней (константа, configurable), статус `active`; ИЛИ
  - прогресс отстаёт от медианного по группе на 30%+.
- В S07 — эмодзи-маркер (виден всем). В S08 — причина.
- Ментор может поставить `mark-abandoned` через существующую кнопку `⚠️ Неактивен` (cause=`inactivity`).

### F6. Сводка по группе (S07)
- Кроме «Всего: N»: медиана прогресса, число отстающих, advanced, not_advanced, active, enrolled.
- Видна всем ролям (прозрачность здоровья группы).

### F7. Сортировка списка студентов (S07)
- Отстающие вверху, затем по убыванию прогресса.

### F8. История шагов студента (S08)
- Реализовать `monitor:history:{studentId}` (сейчас заглушка).
- Список шагов: позиция (ContentPath из трека 2), статус (issued/completed), даты выдачи/завершения.
- Виден всем (прозрачность). Ментор видит темп и где студент застрял.

### F9. Действия ментора в S08 — bot-level тесты
Действия уже в коде (трек 1). Требуются bot-level интеграционные тесты через `BotRouter.handleCallback`:
- `✅ Завершить` → выбор исхода → confirm → UC → статус изменился, `−STUDENT`
- `⚠️ Неактивен` → `mark-abandoned` (cause: `inactivity`) → end-to-end
- `🔄 Сменить исход` для завершённого → смена outcome

## Критерии приёмки
- [ ] «🛠️ Инструменты ментора» → подменю (Мои потоки / Создать поток / Мониторинг), role-gated.
- [ ] «Мои потоки» → S02m с lifecycle-кнопками.
- [ ] «➕ Создать поток» из подменю, не в корне меню.
- [ ] В curious-режиме lifecycle-кнопок нет.
- [ ] Отстающие помечаются с причиной (видно всем).
- [ ] S07 показывает сводку здоровья группы (всем), сортировка отстающих сверху.
- [ ] История шагов студента работает.
- [ ] `complete-student` UI работает end-to-end через BotRouter.
- [ ] Bot-level тесты действий ментора в `tests/bot`.
- [ ] Покрытие >80%, TDD.
