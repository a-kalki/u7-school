# Обзор
Рефакторинг архитектурных слоев `Module` и `App` (в пакете `core` и зависимых пакетах) для обеспечения строгой типизации вызовов usecase-ов со стороны контроллеров через `ApiApp`.

# Функциональные требования
1. **Переименование:** Переименовать `ModuleMeta` в `ApiModuleMeta`.
2. **Типизация Meta-классов:**
   - `ApiModuleMeta` должен регистрировать и типизировать набор связанных с ним `UcMeta`.
   - Добавить новый слой метаданных `AppMeta`, который будет регистрировать в себе список `ApiModuleMeta`.
3. **Строгий контракт модулей:** Класс `ApiModule` должен на уровне системы типов TypeScript гарантировать, что передаваемые ему экземпляры usecase-ов строго соответствуют контрактам, описанным в `ApiModuleMeta`.
4. **Строгий контракт App:** Класс `ApiApp` должен использовать `AppMeta` и гарантировать, что регистрируемые модули соответствуют контрактам. Базовый `App` остается без привязки к дженерикам метаданных, так как он может переиспользоваться (например, в UiApp).
5. **Type-Safe вызовы в ApiApp:** Метод обработки запросов (например, `apiApp.execute(...)`) должен использовать систему типов TypeScript для валидации аргументов (команд, акторов) и выводимого типа результата на основе переданного `UcMeta`. Необходимо продумать DX вызова, чтобы минимизировать количество передаваемых аргументов (например, избежать явной передачи имени модуля, если оно выводится из метаданных) при сохранении строгой типизации.
6. **DX (Developer Experience):** Интерфейс вызова в `ApiApp` должен быть максимально эргономичным, позволяя автокомплиту и проверке типов работать без излишнего дублирования передачи типов.
7. **Обновление контроллеров:** 
   - `u7-cli`: определить `ApiApp` для него (включить все модули, кроме `onboarding`).
   - `onboarding`: исправить контроллер/бота, чтобы он работал через свой экземпляр `ApiApp` (включающий модули `onboarding` и `user`).
   - Заменить прямые вызовы UseCase в этих контроллерах на вызовы через `ApiApp`.

# Нефункциональные требования
1. Обновление и дополнение документации об архитектуре в папке `conductor/code_styleguides/skills/`. Удалить все упоминания устаревшего `auto-ui`, заменив их на актуальный `cli-controller`.
2. Сохранение работоспособности всех тестов (проверка через компилятор `tsc` и `bun test`).
3. Максимально строгая типизация в `ApiApp` и `ApiModule` (минимизация `any` и `as`). Основная проверка правильности будет производиться компилятором `tsc`.

# Критерии приёмки
1. Сущность `ModuleMeta` переименована в `ApiModuleMeta` во всей кодовой базе.
2. Создан и используется `AppMeta`.
3. Вызов метода у `ApiApp` строго типизирован: IDE подсказывает ожидаемую команду, а TypeScript выбрасывает ошибку при несовпадении типов. Количество аргументов в методе вызова сведено к минимуму для удобства DX.
4. Контроллеры используют типизированный `ApiApp` (с соответствующим их контексту набором модулей), а не напрямую обращаются к UseCase.
5. Документация обновлена, очищена от упоминаний `auto-ui`.
6. Сборка `tsc` проходит без новых ошибок типизации.
7. Все тесты (`bun test`) проходят успешно.

# За рамками
- Изменение логики внутри самих `UseCase`, Domain или Infra слоев.
- Добавление новых UseCase (только миграция существующих).
