# Спецификация: Telegram-бот onboarding

## Обзор
Реализовать Telegram-флоу onboarding в `apps/u7-bot` через grammY + @grammyjs/conversations.

## Функциональные требования
### /start
- Получить `telegramId` из `ctx.from.id`.
- Вызвать `GetUserByTelegramIdUc`. Если нет — `RegisterUserUc`.
- Отправить приветствие с inline-клавиатурой: «Быть в курсе» / «Стать студентом».

### Логика повторного /start
- **CANDIDATE** → предложить заполнить новую заявку (создаётся новая, история сохраняется).
- **GUEST (не candidate, не subscriber)** → показать меню с «Быть в курсе» / «Стать студентом».
- **SUBSCRIBER (не candidate)** → ничего не показываем (или короткое «Ты уже с нами!»).
- **Новый пользователь** → приветствие + меню.

### «Быть в курсе»
- Показать ссылку на новостную группу (из `.env`).
- Роль SUBSCRIBER выдаётся при вступлении в группу — за рамками этого трека.
- Кнопка «Вернуться в меню».

### «Стать студентом» — Conversation
1. Приветствие, объяснение анкеты.
2. Пошаговый опросник с inline-клавиатурами:
   - Множественный выбор: кнопки обновляются с ✅, кнопка «Далее ➡️».
   - Одиночный выбор: сразу переход к следующему вопросу.
   - Ветвление: после `intensity` показывать `baseDays`+`baseTime` ИЛИ `intensiveTime`.
3. Предпросмотр ответов перед отправкой.
4. Кнопки: «Отправить» / «Изменить ответы» (частичное изменение — возврат к нужному шагу).
5. По «Отправить»:
   - `CreateApplicationUc` (через ApplicationDs — атомарно создаёт заявку + CANDIDATE).
   - Сообщение-confirmation + ссылка на группу.

### /cancel
- Прерывает текущую conversation.
- Удаляет промежуточные данные из session.
- Возвращает в главное меню.

### Архитектура
- **Нет facade-обёрток** — controller работает напрямую через `app` объект.
- **`app` объект** с дженериком в методе вызова UC для типизации команд.
- Проверить нужен ли отдельный controller слой (как в CLI). Если нет — handlers напрямую используют `app`.
- Конфигурация через `.env.development` / `.env.production` (Bun подтягивает автоматически через `NODE_ENV`).

## Нефункциональные требования
- grammY + @grammyjs/conversations.
- Session storage в JSON (восстановление состояния при перезапуске).
- Покрытие тестами где возможно (моки grammY).

## Критерии приёмки
- [ ] Бот отвечает на `/start`, создаёт/находит пользователя.
- [ ] Inline-меню работает корректно.
- [ ] Conversation проходит все вопросы анкеты.
- [ ] Множественный выбор и ветвление работают корректно.
- [ ] «Изменить ответы» позволяет вернуться к нужному шагу.
- [ ] Заявка создаётся в БД, пользователь получает CANDIDATE.
- [ ] `/cancel` прерывает опросник и очищает session.
- [ ] Повторный `/start` корректно обрабатывает существующих кандидатов и подписчиков.
- [ ] Конфигурация через .env.development/.env.production.

## За рамками
- Выдача роли SUBSCRIBER при вступлении в группу (отдельный процесс).
- Розыгрыш/распределение по потокам.
- Admin-меню для просмотра заявок (будущий трек).
