# Спецификация: Контроллер бота и интеграция Stream

## Обзор
Реализовать полноценный `StreamController` для Telegram-бота с рендерингом карточек, прогресс-баров и навигацией согласно user-stories. Интегрировать кнопки потоков в главное меню `u7-bot`.

## Архитектурное ограничение
**Строгое разделение слоёв:**
- **Controller** — только приём команд от бота, вызов UC, форматирование ответа. Никакой доменной логики.
- **UC** — координирует поток выполнения, вызывает AR/DomainService/Policy.
- **AR/DomainService/Policy** — управляют состоянием и принимают доменные решения.

## Функциональные требования

### FT-1: StreamController — обработка update
- Маршрутизация: `command` → метод, `callback` → парсинг `action:params`
- Вызов соответствующих UC через `StreamApiModule`
- Обработка ошибок с человекочитаемыми сообщениями

### FT-2: Витрина потоков (US-1)
- Кнопка `📚 Наши потоки` → список потоков
- Inline-клавиатура: `🟢 Название (Набор)`, `🔵 Название (Идёт)`, `⚪ Название (Завершён)`
- callback_data: `stream:view:<streamId>`

### FT-3: Карточка потока (US-2)
- Детальная карточка: название, ментор, описание, дата старта, кол-во студентов
- GUEST/CANDIDATE на enrollment: кнопки `📝 Записаться` и `📖 Программа`
- STUDENT/MENTOR: скрыть кнопку записи, показать статус

### FT-4: Запись на поток (US-3)
- Кнопка `📝 Записаться` → EnrollStudentUc
- Сообщение об успехе с ссылкой на чат группы

### FT-5: Моя учёба (US-4, US-5)
- Кнопка `📖 Моя учёба` (только STUDENT)
- Текущий шаг с заданием и кнопкой `✅ Выполнено`
- При завершении: следующий шаг / поздравление с уроком/проектом/потоком
- Прогресс-бар: `📊 Мой прогресс`

### FT-6: Панель ментора (US-6, US-7, US-8)
- Кнопка `🛠️ Панель ментора` (MENTOR/ADMIN)
- Создание потока: пошаговый диалог (`menu = 'create_stream'` + шаги в `session.data`)
- Запуск потока: `🚀 Запустить` → ActivateStreamUc
- Список студентов с прогресс-барами и пометкой отстающих

### FT-7: Интеграция с top-menu-handler.ts
- `📚 Наши потоки` — всем ролям
- `📖 Моя учёба` — только STUDENT
- `🛠️ Панель ментора` — только MENTOR/ADMIN
- Существующие команды переведены на кнопки где уместно

## Критерии приёмки
- [ ] Ментор создаёт поток через бота (US-6)
- [ ] Ментор запускает поток — студентам открывается первый шаг (US-7)
- [ ] Посетитель видит витрину и записывается (US-1..3)
- [ ] Студент видит `📖 Моя учёба`, проходит шаги, видит прогресс (US-4..5)
- [ ] Ментор видит список студентов с прогрессом (US-8)
- [ ] Кнопки меню корректны для каждой роли
- [ ] Контроллер не содержит доменной логики

## За рамками
- Автоматическая рассылка уведомлений при запуске потока
- Написание ментору напрямую из списка студентов (кнопка-ссылка на tg user)
