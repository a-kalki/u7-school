# Спецификация: Расширение ролей пользователя

## Обзор
Расширить домен `@u7/user` новыми ролями жизненного цикла onboarding и добавить UC управления ролями.

## Функциональные требования
1. Добавить в `Role` enum: `GUEST`, `SUBSCRIBER`, `CANDIDATE`.
2. Добавить метод `UserAr.register(cmd)` — регистрация пользователя при первом `/start` в боте. Создаёт пользователя с `roles: [GUEST]`.
3. Добавить метод `UserAr.addRole(role)` — идемпотентное добавление роли в `roles[]`. Если роль уже есть — noop.
4. Создать `RegisterUserUc` — делегирует `UserAr.register`.
5. Создать `AddRoleToUserUc` — делегирует `UserAr.addRole`.
6. Обновить тесты существующих UC, где затрагивается `Role`.

## Нефункциональные требования
- Покрытие кода >80%.
- Следовать DDD-конвенциям проекта (valibot, JSON-repo, модули).
- `RegisterUserUc` НЕ заменяет `CreateUserUc`, а дополняет.

## Критерии приёмки
- [ ] `Role` содержит `GUEST`, `SUBSCRIBER`, `CANDIDATE`.
- [ ] `UserAr.register()` создаёт пользователя с ролью `GUEST`.
- [ ] `UserAr.addRole()` добавляет роль без дублирования.
- [ ] `RegisterUserUc` и `AddRoleToUserUc` работают корректно.
- [ ] Все тесты `@u7/user` проходят.

## За рамками
- Telegram-UI, onboarding-анкета, домен заявок.
- Логика выдачи SUBSCRIBER при вступлении в группу (отдельный процесс).
