# Спецификация трека: Мониторинг группы для ментора

> **Контекст эволюции:** `conductor/architecture-evolution.md` (§2.3). Прочитать перед началом.
> **Зависимости:** трек 1 (ЖЦ студента — корректные статусы, abandonReason, completedAt шагов).

## Обзор
Дать ментору инструменты оперативного контроля группы: автоматическое выявление отстающих, сводка здоровья группы, сортировка проблемных сверху, детальная история шагов студента.

**Роли:** MENTOR, ADMIN.

## Функциональные требования

### F1. Логика «Отстаёт» (S07, S08)
- Статус «⚠️ Отстаёт» если:
  - `completedAt` последнего шага старше 5 дней (константа, configurable), статус `active`; ИЛИ
  - прогресс отстаёт от медианного по группе на 30%+.
- В S07 — эмодзи-маркер у студента. В S08 — причина («нет активности 6 дней» / «отставание от группы 35%»).
- `abandoned(by_mentor, inactivity)` — ментор может поставить статус через `mark-abandoned` (UC из трека 1) прямо из карточки отстающего.

### F2. Сводка по группе (S07)
- Кроме «Всего: N»: медиана прогресса, число отстающих, число завершивших, число active.
- Ментор видит здоровье группы одним взглядом.

### F3. Сортировка списка студентов (S07)
- Отстающие вверху, затем по убыванию прогресса. Ментору важны проблемные первыми.

### F4. История шагов студента (S08 `history:*`)
- Реализовать обработчик `monitor:history:{studentId}` (сейчас заглушка).
- Список шагов студента: позиция (ContentPath), статус (issued/completed), даты выдачи/завершения.
- Ментор видит темп и где студент застрял.

### F5. Действия ментора в S08 (интеграция с треком 1)
- «✅ Завершить обучение» → `complete-student` (confirm).
- «❌ Отчислить» → `mark-abandoned(by_mentor)` (confirm) — переименовать/семантика из трека 1.
- «📁 История шагов» → F4.

## Критерии приёмки
- [ ] Отстающие (5+ дней без активности / 30%+ от медианы) помечаются с причиной.
- [ ] S07 показывает сводку здоровья группы.
- [ ] Список отсортирован: отстающие сверху.
- [ ] История шагов студента работает.
- [ ] Действия complete/abandoned через confirm-хелпер.
- [ ] Покрытие >80%, TDD.

## За рамками
- Broadcast/рассылка группе (бэклог, трек 7).
- Автоматическое отчисление по неактивности (только ручное mark-abandoned).
- Графики/аналитика во времени.
