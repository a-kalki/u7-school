# Спецификация: Оптимизация воркфлоу пакета w3school

## Обзор (Overview)
Данный трек направлен на превращение пакета `w3school` в надежный автономный модуль управления контентом. Мы объединим разрозненные скрипты в единый CLI, упорядочим локальное хранение данных внутри пакета и предоставим программный API для интеграции с другими частями системы (например, ботом).

## Функциональные требования (Functional Requirements)
1. **Унифицированный CLI (Bun):**
   - Реализовать CLI-интерфейс внутри пакета.
   - Подкоманды:
     - `parse`: Извлечение структуры и контента из локально скачанных HTML-файлов.
     - `enrich`: Генерация сводок (summary) и метаданных для уроков через ИИ.
     - `status`: Отчет о наличии скачанного контента и степени его обработки.
2. **Локальное хранение (Local Storage):**
   - Все данные (сырой HTML и обработанный JSON) хранятся внутри директории пакета `packages/w3school/`.
   - Структурировать папки для удобной навигации ментора.
3. **Программный API (Public API):**
   - Экспортировать из пакета типизированные методы (например, `getCourseSyllabus`, `getLessonContent`), которые будут скрывать детали файлового хранения от других модулей.
4. **Инструкции для ментора:**
   - Добавить подробное руководство по использованию: от скачивания через `wget` до финального формирования структуры курсов.

## Критерии приемки (Acceptance Criteria)
- [ ] Рабочий CLI-инструмент в `packages/w3school`.
- [ ] Публичный API (index.ts) предоставляет доступ ко всем данным курсов.
- [ ] Написана пошаговая документация для менторов.
- [ ] Сохранен и упорядочен текущий механизм парсинга и обогащения.
- [ ] Весь новый функционал покрыт тестами (>80%).

## Вне рамок (Out of Scope)
- Перенос данных в корень проекта (остаемся в границах пакета).
- Автоматизация процесса скачивания (остается на стороне пользователя).
