# Спецификация: Архитектурные улучшения бота (bot_ux_arch_20260616)

## Обзор
Архитектурные улучшения ядра бота: удаление старых кнопок при навигации, контекст предыдущего сообщения, отключение экранирования кнопок (конвенция), задержка `sendMessages`.

## Функциональные требования

### А1. Удаление старых кнопок при навигации
- В `BotResponse` добавляется поле `removePrevKeyboard?: boolean`.
- При `true` — `executeResponses()` перед отправкой новых сообщений редактирует предыдущее сообщение бота (messageId + текст из `SessionData.lastBotMessage`), убирая клавиатуру (`reply_markup: undefined`).
- Механизм создаётся, но в этом треке НЕ применяется к конкретным story (например, «Моя учёба»). Применение — в следующих треках.

### А2. Хранение предыдущего сообщения в контексте
- В `SessionData` добавляется поле `lastBotMessage?: SendMessageDescription & { messageId: number }`.
- `executeResponses()` при отправке (`ctx.reply()`) сохраняет `SendMessageDescription` + `message_id` последнего отправленного сообщения в `ctx.session.lastBotMessage`.
- При `editMessage` — lastBotMessage не обновляется.
- При `sendMessages` (массив) — сохраняется последнее.

### А3. Отключение экранирования кнопок (конвенция)
- Текст кнопок в `KeyboardDescription.rows[].text` — всегда plain text. Telegram не парсит MarkdownV2 в кнопках, экранирование не нужно и портит отображение.
- Из всех story во всех модулях убираются вызовы `escapeMarkdown()` для текста кнопок.
- Валидация `assertResponseMarkdownSafe()` обновляется: не проверяет текст кнопок на неэкранированные символы.
- Документация фиксирует конвенцию.
- Архитектурных запретов на уровне типов не вводим.

### А4. Задержка `sendMessages`
- В `BotResponse` добавляется поле `sendDelayMs?: number`.
- По умолчанию — 1000 мс между сообщениями в массиве `sendMessages`.
- Можно задать `0` (без задержки).
- Реализуется в `executeResponses()` через `await sleep(delay)`.

## Нефункциональные требования
- Строгая типизация, никаких `any` без крайней необходимости.
- Все новые механики покрыты тестами.
- Обратная совместимость не обязательна — готовы к временной неработоспособности.
- Документация обновляется после завершения трека.

## Критерии приёмки
- `SessionData.lastBotMessage` сохраняет полный `SendMessageDescription` + `messageId` последнего сообщения.
- При `removePrevKeyboard: true` клавиатура предыдущего сообщения убирается, текст остаётся.
- `sendMessages` приходят с паузой 1000 мс (или с заданной в `sendDelayMs`).
- Кнопки во всех story показываются без бэкслешей (без ручного `escapeMarkdown`).
- `assertResponseMarkdownSafe` не ругается на неэкранированный текст кнопок.
- Все тесты проходят: `bun test`
- Линтер и типы чисты: `bun run lint && bun run tslint`
