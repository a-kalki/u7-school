# Спецификация трека: Навигация студента

> **Контекст эволюции:** `conductor/architecture-evolution.md` (§2.2). Прочитать перед началом.
> **Зависимости:** трек 1 (ЖЦ — drop студента, статусы abandoned/advanced/not_advanced), трек 2 (ContentPath — резолвер).

## Обзор
Реструктурировать «Моя учёба» в хаб, добавить древовидную навигацию по урокам кнопками (через ContentPath), навигацию ◀️/▶️ по истории шагов, прогресс урока и самостоятельный выход из потока.

**Роли:** STUDENT.

## UX-конвенции
- **editMessage** — все переходы внутри дерева «📂 Уроки» (проект→урок→шаг→просмотр→◀️/▶️) происходят в одном сообщении через `editMessage`. Без спама в чате.
- **sendMessage** — вход в дерево из хаба и выход из него — новые сообщения.
- **Список шагов в теле** — при просмотре шага внизу всегда показывается полный список шагов урока с маркерами ✅/▶️/🔒.
- **Кнопки — только доступные шаги** — ✅ и ▶️. 🔒-шаги видны только в тексте (Telegram не поддерживает disabled-кнопки).

## Функциональные требования

### F1. Хаб «Моя учёба» (S05)
- «📖 Моя учёба» → промежуточный экран (не прыжок в шаг):
  - «▶️ Продолжить» — текущий шаг (S05a)
  - «📂 Уроки» — навигация по программе (S05b)
  - «📊 Мой прогресс» (S06)
  - «🚪 Покинуть поток» (confirm → drop-student)
- Для завершивших студентов (advanced/not_advanced/abandoned): «▶️ Продолжить» и «📂 Уроки» скрыты.

### F2. Навигация по урокам (S05b) — дерево через ContentPath
- Иерархическая навигация в одном сообщении (editMessage):
  1. **Проекты:** список проектов модуля с прогрессом `(пройдено/всего уроков)`. Только проекты, в которых есть ≥1 пройденный или текущий урок.
  2. **Уроки проекта:** список уроков с прогрессом `(пройдено/всего шагов)`. Кнопка «⬅️ Назад к проектам».
  3. **Шаги урока:** список шагов с маркерами `✅` completed (read-only), `▶️` current (active), `🔒` будущий (только в тексте, не кнопкой). Кнопка «⬅️ Назад к урокам».
  4. **Просмотр шага:** контент шага (read-only для ✅, active для ▶️). Внизу — полный список шагов урока с маркерами. Кнопки: ◀️/▶️ (для completed), «⬅️ Назад к уроку», «↩️ Главное меню».
- Использует `ContentSnapshot` из `get-stream` + `CourseDs` (findStepPosition, findLessonTitle, findProjectTitle).
- **НЕ использует** `resolve-content-path` UC (это чистые вычисления на снапшоте).

### F3. Навигация ◀️/▶️ по истории (editMessage)
- Доступна только при просмотре completed шага (зашли через дерево «📂 Уроки»).
- Кнопки: `◀️ Назад` / `▶️ Вперёд`. Листают **только completed шаги** в хронологическом порядке.
- ◀️ скрыта на первом completed шаге. ▶️ скрыта на последнем completed (перед current).
- При листании — **editMessage** (меняется контент, список шагов и кнопки в том же сообщении).
- Кнопки «📊 Мой прогресс» на этом экране **нет** — фокус на программе.
- На активном (▶️) шаге кнопок ◀️/▶️ **нет** — только «✅ Выполнено».

### F4. Прогресс урока в шаге
- В просмотре шага: бар урока `[██░░] X/Y` в строке «Шаг X из Y».
- На transition-экранах (урок/проект завершён): общий прогресс потока («осталось N уроков»).

### F5. Самостоятельный выход
- «🚪 Покинуть поток» → confirm → `drop-student` (status: abandoned, abandonDetails: { who: 'self', cause: 'voluntary' }, −STUDENT).
- После выхода: сообщение с благодарностью, кнопка «↩️ Главное меню».

## Критерии приёмки
- [x] Хаб «Моя учёба» показывает 3–4 кнопки вместо прыжка в шаг. (Фаза 1 ✅)
- [x] Самостоятельный выход работает (confirm). (Фаза 1 ✅)
- [ ] Дерево навигации: проекты→уроки→шаги в одном сообщении (editMessage).
- [ ] Студент доходит до любого пройденного шага кнопками (read-only).
- [ ] ◀️/▶️ листает историю в одном сообщении (editMessage).
- [ ] Будущие шаги заблокированы — только в тексте с `🔒`, не кнопкой.
- [ ] Список шагов урока виден при просмотре любого шага.
- [ ] Покрытие >80%, TDD.

## За рамками
- Повторное прохождение урока (restart) — бэклог.
- Поиск по шагам — бэклог.
- Шаблонный ввод `mA:pB:lC:sD` — бэклог.
