# Спецификация трека: Навигация студента

> **Контекст эволюции:** `conductor/architecture-evolution.md` (§2.2). Прочитать перед началом.
> **Зависимости:** трек 1 (ЖЦ — drop студента, статусы abandoned/advanced/not_advanced), трек 2 (ContentPath — резолвер).

## Обзор
Реструктурировать «Моя учёба» в хаб, добавить древовидную навигацию по урокам кнопками (через ContentPath), навигацию ◀️/▶️ по истории шагов, прогресс урока и самостоятельный выход из потока.

**Роли:** STUDENT.

## Функциональные требования

### F1. Хаб «Моя учёба» (S05)
- «📖 Моя учёба» → промежуточный экран (не прыжок в шаг):
  - «▶️ Продолжить» — текущий шаг (S05a)
  - «📂 Уроки» — навигация по программе (S05b)
  - «📊 Мой прогресс» (S06)
  - «🚪 Покинуть поток» (confirm → drop-student)

### F2. Навигация по урокам (S05b) — кнопки через ContentPath
- Иерархическая навигация: проекты (текущий + пройденные) → уроки проекта → шаги урока → контент шага.
- Маркеры шагов: `✅` completed (read-only), `▶️` current (active), `🔒` будущий (скрыт/заголовок).
- Использует `resolve-content-path` (трек 2).
- Шаблонный ввод `mA:pB:lC:sD` — опц. power-user фича через `handleMessage` (если решим включить).

### F3. Навигация ◀️/▶️ по истории (S05a)
- На архивном (completed) шаге: кнопки `◀️ Назад` / `▶️ Вперёд` (read-only, без «Выполнено»).
- На текущем шаге: только «✅ Выполнено» + «📊 Прогресс».
- Листание в пределах пройденной истории.

### F4. Прогресс урока в шаге
- В S05a: бар урока `[██░░] 2/5` рядом с «Шаг X из Y».
- На переходах (урок/проект завершён): общий прогресс потока («осталось N уроков»).

### F5. Самостоятельный выход
- «🚪 Покинуть поток» → confirm → `drop-student` (status: abandoned, abandonDetails: { who: 'self', cause: 'voluntary' }, −STUDENT).

## Критерии приёмки
- [ ] Хаб «Моя учёба» показывает 3–4 кнопки вместо прыжка в шаг.
- [ ] Студент доходит до любого пройденного шага кнопками (read-only).
- [ ] ◀️/▶️ листает историю.
- [ ] Будущие шаги заблокированы (`🔒`).
- [ ] Самостоятельный выход работает (confirm).
- [ ] Покрытие >80%, TDD.

## За рамками
- Повторное прохождение урока (restart) — бэклог.
- Поиск по шагам — бэклог.
