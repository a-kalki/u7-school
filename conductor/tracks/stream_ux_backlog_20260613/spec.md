# Спецификация трека: Stream UX Backlog

## Обзор
Реализовать UX-улучшения из раздела 5 `user-stories.md`: навигация по истории шагов, broadcast-уведомления, логика «Отстает».

**Роли:** STUDENT, MENTOR.

## Функциональные требования

### UX-1: Навигация ◀️/▶️ по истории шагов (US-4)
- Студент нажимает `[◀️ Назад]` — видит предыдущий выполненный шаг (read-only, без кнопки «Выполнено»).
- На архивном шаге: кнопки `[▶️ Вперёд]` и `[◀️ Назад]`.
- Студент может листать историю до начала текущего урока и вернуться к активному шагу.

### UX-2: Broadcast при запуске потока (US-7)
- При активации потока ментором, всем зачисленным студентам отправляется сообщение:
  «🎉 Поток {StreamTitle} запущен! Перейдите в '📖 Моя учёба', чтобы получить первое задание.»
- Рассылка через Telegram API (бот должен иметь список telegramId студентов).

### UX-3: Логика «Отстает» в MonitorStory (US-8)
- Статус «⚠️ Отстает» присваивается, если:
  - Студент не отмечал шаги 5+ дней
  - Или прогресс отстаёт от медианного по группе на 30%+
- Отображать в списке студентов и в детальной карточке.

### UX-4: Кнопки «Завершить» / «В архив» для ментора (US-7)
- На карточке активного потока для ментора: кнопка `[✅ Завершить]` → `complete-stream` UC.
- На карточке любого своего потока: кнопка `[📁 В архив]` → `archive-stream` UC.

## Критерии приёмки
- [ ] Студент может листать историю шагов ◀️/▶️
- [ ] При запуске потока студенты получают уведомление
- [ ] В мониторинге отображается статус «Отстает» с причиной
- [ ] Ментор может завершить и архивировать поток через карточку

## За рамками
- Double-confirm (отказались — случайное нажатие некритично)
- Сбор обратной связи (Feedback Loop)
- Разнообразие типов шагов (тесты, ревью)
- Заморозка/отчисление студента
- Синхронизация контента «на лету»
