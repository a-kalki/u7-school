# Спецификация: Полная интеграция жизненного цикла потоков

## Контекст
Полная реализация жизненного цикла потоков курсов (@u7-scl/stream), включая управление ими ментором и прохождение обучения студентом через Telegram-бота.

## Основные User Stories (Детали см. `packages/stream/src/user-stories.md`)
1. **US-1, US-2 (Посетитель)**: Витрина потоков, детальная карточка.
2. **US-3 (Студент)**: Запись на поток, автоматическое получение роли `STUDENT`.
3. **US-4, US-5 (Студент)**: Прохождение обучения, получение шагов, прогресс.
4. **US-6, US-7, US-8 (Ментор)**: Создание потоков, запуск (активация), управление списком студентов.

## Доменные инварианты
1. **Ролевая модель**: 
    - `GUEST` / `CANDIDATE` -> могут только просматривать витрину и записываться.
    - `STUDENT` -> может проходить обучение, смотреть прогресс (на 1 активном потоке).
    - `MENTOR` -> может создавать, запускать, архивировать потоки, смотреть прогресс группы.
2. **Активация потока**: При переходе в `active`, для всех `StreamStudent` (со статусом `active`) автоматически выдается первый шаг из `contentSnapshot` модуля.
3. **Завершение шага**: Только `issued` шаг может быть завершен. После завершения последнего шага — курс завершается (`status: completed`).

## Требования к UI (Bot)
1. **Динамическое меню**: Появление кнопок `📖 Моя учёба` и `🛠️ Панель ментора` строго зависит от роли пользователя.
2. **Stateless**: Контроллер не хранит состояние диалога, все запросы к домену идут через `ApiApp`.
3. **Безопасность**: Передача `actorId` (UUID пользователя, полученного из `UserFacade` по `telegramId`) во все API-вызовы.
