# Спецификация трека: Stream Mentor (US-6..US-8)

## Обзор
Доработать менторские сторис до полного соответствия `user-stories.md`: создание потока с реальным списком модулей, интеграция кнопки запуска в карточку, мониторинг с именами студентов.

**Роль:** MENTOR.

**Зависит от:** `stream_infra_20260613` (CourseFacade должен быть подключён).

## Функциональные требования

### US-6: Реальный список модулей в CreateStreamStory
- **Шаг выбора модуля** (`#handleModuleMessage`) должен запрашивать модули ментора через `CourseFacade` или API `list-modules` и показывать inline-кнопки с названиями.
- **Превью перед созданием:** после шага 5 (ввод группы) показать карточку потока с кнопками `[✅ Создать]` и `[⬅️ Изменить]`.
- Сейчас: шаг выбора модуля возвращает заглушку «Обновите список», нет превью.

### US-7: Интеграция кнопки «Запустить» в ViewStreamStory
- Для ментора на **своём** потоке (`actorId === stream.mentorId`) добавить менторские кнопки:
  - `[🚀 Запустить]` — для `enrollment`
  - `[👥 Студенты]` — всегда
  - `[✅ Завершить]` — для `active`
  - `[📁 В архив]` — всегда
- Нажатие «Запустить» → callback `activate-stream:activate:<streamId>` (уже реализован в `ActivateStreamStory`).
- Нажатие «Студенты» → callback `monitor:students:<streamId>` (уже реализован в `MonitorStory`).

### US-8: Имена студентов в MonitorStory
- Вместо `userId.slice(0, 8)` показывать имена студентов, полученные через `UserFacade.getUserByUuid()`.
- Добавить callback `monitor:detail:<studentId>` для детальной карточки студента.
- Детальная карточка: имя, Telegram, статус, прогресс, кнопки `[✉️ Написать]`, `[📁 История шагов]`, `[⬅️ Назад к списку]`.

## Критерии приёмки
- [ ] При создании потока ментор видит список своих модулей (реальные данные)
- [ ] Перед созданием показывается превью с подтверждением
- [ ] Ментор на карточке своего потока видит менторские кнопки
- [ ] Кнопка «Запустить» активирует поток
- [ ] Кнопка «Студенты» показывает список с именами
- [ ] Можно кликнуть на студента и увидеть детальную карточку
