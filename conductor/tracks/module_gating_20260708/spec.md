# Спецификация трека: Gating модулей

> **Контекст эволюции:** `conductor/architecture-evolution.md` (§2.1, §2.3). Прочитать перед началом.
> **Зависимости:** трек 0 (Course — порядок модулей), трек 1 (ЖЦ студента — статусы completed/advanced).

## Обзор
Реализовать продуктовое правило: студент может записаться на модуль N только завершив модуль N-1 в курсе. При завершении модуля — предложение перейти на следующий; при зачислении на следующий — статус `advanced` на предыдущей записи.

**Роли:** STUDENT, MENTOR.

## Функциональные требования

### F1. CoursePolicy — gating
- `CoursePolicy.canEnrollNextModule(student, course, targetModuleId)`:
  - targetModule — первый в курсе → разрешён всем.
  - иначе: существует предыдущий модуль в course.phases, и у студента есть Student-запись с `status: 'completed' | 'advanced'` для потока предыдущего модуля. Если нет → отказ с причиной.
- Учитывает, что Student-запись привязана к stream→module.

### F2. Gate в enroll-student
- `enroll-student` для потока модуля N: проверить `canEnrollNextModule`. Если не пройден → ошибка «Сначала пройдите модуль {prevModuleTitle}».
- Решить: что считать «завершением модуля» — Student.status in (`completed`, `wants_next`, `advanced`, `not_advanced`). `abandoned` — НЕ завершение.

### F3. Переход completed→advanced
- После `completed` студента на модуле N → UI предлагает «Записаться на следующий модуль» (если есть следующий и есть открытый поток набора).
- При успешном `enroll-student` на модуль N+1 → предыдущая Student-запись → `advance()` (+ `advanced`), выдача STUDENT на новую запись.

### F4. UI (S05/S05a — после complete)
- При завершении модуля: «🎉 Модуль завершён! Следующий модуль: {title}.» + кнопки:
  - «📝 Записаться на следующий» (если есть enrollment-поток модуля N+1 и gate пройден)
  - «⏳ Ждать набор» (если нет открытого потока) → `wants_next`
  - «↩️ Главное меню» → `not_advanced` (или оставить completed — решить)

### F5. Тесты
- E2E: студент проходит Синтаксис → может записаться на Алгоритмику; не прошедший Синтаксис → отказ.

## Критерии приёмки
- [ ] Студент без завершённого Синтаксиса не может записаться на Алгоритмику (отказ с причиной).
- [ ] Студент с completed Синтаксис → записывается на Алгоритмику, предыдущая запись → `advanced`.
- [ ] UI предлагает переход после завершения модуля.
- [ ] Покрытие >80%, TDD.

## За рамками
- Сложные prerequisite-графы (деревья зависимостей) — пока только линейная последовательность по phases.
- Автоматический набор на следующий поток (только ручная запись).
