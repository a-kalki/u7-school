# Спецификация: Доменная модель и парсинг материалов курсов

## Обзор (Overview)
Цель данного трека — спроектировать доменную модель для образовательных материалов (курсы, модули, уроки, практические задания), определить структуру папок для их хранения и разработать скрипт парсинга структуры курсов HTML и CSS с сайта W3Schools (только навигация).

## Функциональные требования (Functional Requirements)
1.  **Доменная модель (Domain Model):**
    *   **Курс (Course):** Состоит из модулей. Идентификаторы курсов имеют префикс `w3c-` (например, `w3c-html`, `w3c-css`).
    *   **Модуль (Module):** Логическая группировка уроков (например, "HTML Basic").
    *   **Урок (Lesson):** Базовая единица контента. Каждый урок имеет:
        *   Внутренний уникальный идентификатор (`id`).
        *   Оригинальный URL на W3Schools (`url`).
        *   Название из контекста W3C (`name`).
        *   Другие релевантные атрибуты из W3C (например, порядок следования).
    *   **Практический материал (Practice):** Дополнительные локальные задания, связанные с конкретным уроком (маппинг по `id` урока).
2.  **Хранение данных (Data Storage):**
    *   Все структурные данные курсов (список модулей, уроков, маппинг) хранятся в формате JSON.
    *   Файловая иерархия организована по курсам (изолированно), с использованием префикса `w3c-`.
        *   Пример структуры:
            *   `data/courses/w3c-html/index.json` (структура курса HTML)
            *   `data/courses/w3c-html/practice/` (практические материалы)
            *   `data/courses/w3c-css/index.json` (структура курса CSS)
            *   `data/courses/w3c-css/practice/` (практические материалы)
3.  **Парсинг W3Schools (Parsing):**
    *   Разработать Bun-скрипт(ы) для парсинга Sidebar-меню W3Schools.
    *   Глубина парсинга: только навигация (извлечение иерархии модулей, названий уроков, ссылок).
    *   Целевые курсы для первого парсинга: HTML и CSS.
    *   Результат парсинга должен сохраняться в соответствующие JSON файлы в директории `data/courses/`.

## Нефункциональные требования (Non-Functional Requirements)
*   Парсер должен корректно обрабатывать структуру DOM W3Schools на текущий момент.
*   Код парсера должен быть надежным, с базовой обработкой ошибок сетевых запросов.
*   Структура JSON должна валидироваться с помощью схем (Valibot, согласно техстеку проекта).

## Критерии приемки (Acceptance Criteria)
*   [ ] Описана и зафиксирована структура JSON-схемы (Valibot) для `Course`, `Module`, `Lesson` в проекте.
*   [ ] Создана базовая структура папок в `data/courses/` (w3c-html, w3c-css).
*   [ ] Написан исполняемый скрипт (в `src/parser/` или аналогичной папке), который успешно парсит структуру курсов HTML и CSS с w3schools.com.
*   [ ] Результат работы скрипта сохраняется в валидные файлы `data/courses/w3c-html/index.json` и `data/courses/w3c-css/index.json`.
*   [ ] У каждого урока в JSON есть внутренний `id`, `name`, `url`.

## Вне рамок (Out of Scope)
*   Парсинг текстового контента, примеров кода или интерактивных элементов со страниц уроков W3Schools.
*   Реализация команд бота для работы с этими материалами (выдача заданий, проверка).
*   Парсинг других языков (JS, Python и т.д.) на данном этапе.
