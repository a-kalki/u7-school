# Спецификация: Реорганизация документации conductor

## Обзор
Реорганизация и упрощение документации в `conductor/code_styleguides/` согласно скиллу `conductor-docs`.
Удаление дублирующих, пустых и раздутых файлов. Объединение смежных тем.
Создание недостающего styleguide по bot-контроллерам.

**Главная цель:** создать легковесную систему документации, удалить лишние файлы и лишнее в файлах.

**Принципы:**
- Описывать в основном: цели, архитектурные решения, ответственности, принципы, правила, соглашения.
- Не раздувать документацию, не дублировать одно и то же в разных файлах.
- Не делать документацию хрупкой, описывать высокоуровнево, для деталей дать явные ссылки в код.
