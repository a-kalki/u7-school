# Спецификация трека: Onboarding Bot Tests

## Обзор
Написать интеграционные и E2E тесты для модуля onboarding, покрывающие полный сценарий анкетирования пользователя через Telegram-бота.

Текущее состояние:
- Unit-тесты на use-case'ы написаны (`start-uc`, `handle-action-uc`, `abandon-uc`, `get-current-question-uc`).
- Интеграционный тест контроллера (`onboarding-controller.test.ts`) покрывает базовые сценарии взаимодействия с контроллером.
- E2E тесты с Grammy-ботом **отсутствуют**.

**Роли:** GUEST, CANDIDATE.

## Функциональные требования

### FR-1: E2E тест полного сценария анкетирования
- Тест проходит полный цикл: `/start` → кнопка «Заполнить анкету» → прохождение всех вопросов → завершение анкеты → роль CANDIDATE.
- Проверяется корректность сообщений на каждом шаге.
- Проверяется финальное сообщение о завершении.
- Используется реальный Grammy-бот в тестовом режиме (без polling/webhook).

### FR-2: E2E тест прерывания анкеты
- Начало анкеты → `/cancel` → анкета помечается как abandoned.
- Проверяется сообщение о прерывании.

### FR-3: E2E тест возобновления анкеты
- Начало анкеты → ответ на первый вопрос → `/cancel` → снова «Заполнить анкету» → анкета продолжается с того же места.

### FR-4: Интеграционный тест: edge cases
- Пустой ответ на обязательный вопрос.
- Слишком длинный ответ.
- Множественный выбор: выбор нескольких опций.

## Критерии приёмки
- [ ] E2E тест полного сценария проходит
- [ ] E2E тест прерывания проходит
- [ ] E2E тест возобновления проходит
- [ ] Интеграционные тесты edge cases проходят
- [ ] Все существующие тесты по-прежнему проходят

## За рамками
- Нагрузочное тестирование
- Тестирование с production-конфигурацией
