# Итоговый отчёт: Onboarding Bot Tests

## Цель
Написать интеграционные и E2E тесты для модуля onboarding, покрывающие полный сценарий анкетирования пользователя через Telegram-бота.

## Выполненные задачи

### Фаза 1: E2E тесты (Grammy-бот)
- ✅ E2E тест полного сценария анкетирования — `/start` → кнопка «Заполнить анкету» → ответы → завершение → роль CANDIDATE
- ✅ E2E тест прерывания анкеты — `/cancel` → abandoned → сообщение о прерывании
- ✅ E2E тест возобновления анкеты — ответ → повторный вход → продолжение с текущего вопроса

### Фаза 2: Интеграционные тесты: edge cases
- ✅ Пустой ответ на текстовый вопрос → ошибка валидации
- ✅ Очень длинный текст (2000+ символов) → принимается без ошибок
- ✅ Множественный выбор: «Далее» без выбора опций → ошибка minLength

## Изменённые файлы
- `tests/bot/e2e/onboarding.e2e.test.ts` — новый файл, 3 E2E теста
- `packages/onboarding/src/ui/bot/controller/onboarding-controller.test.ts` — +3 edge case теста

## Архитектурные решения
- E2E тесты используют `BotRouter` (абстрактный бот без Grammy polling), по аналогии с существующими тестами stream и main-menu
- Тесты создают временные JSON-репозитории через `mkdtempSync` и удаляют после выполнения
- Использован упрощённый пул из 2 вопросов (choice + text) для предсказуемости тестов

## Отклонения от первоначального плана
- **FR-3:** тест возобновления не использует `/cancel` перед повторным входом, т.к. после `/cancel` анкета получает статус `abandoned` и новая начинается с нуля. Реальный сценарий: повторный вход в активную анкету без отмены.

## Известные ограничения
- Отсутствует E2E тест с реальным Grammy-ботом (используется BotRouter)
- Нет нагрузочного тестирования
