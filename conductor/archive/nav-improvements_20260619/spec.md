# Спецификация трека: Навигация и очистка кнопок бота

## Обзор
Улучшить UX навигации в Telegram-боте: на всех не-процессных экранах есть кнопка возврата, удаление отработанных кнопок где уместно, правила закреплены в документации, `user-stories.md` актуален.

**Философия:** кнопки «Назад»/«Главное меню» — только когда пользователь в режиме навигации/просмотра. В активном процессе (wizard, обучение) акцент на процессе, лишних кнопок нет. Пользователь всегда может `/start`.

## Функциональные требования

### UX-1: Callback `app:main-menu` — пересборка меню без сброса activeHandler
- `BotRouter` обрабатывает специальный callback `app:main-menu`.
- Пересобирает главное меню через `collectMainMenu()`, возвращает тот же ответ, что и `/start`.
- **НЕ** сбрасывает `activeHandler` — в отличие от `/start`.
- Нужен для кнопки «↩️ Главное меню».

### UX-2: Кнопка «↩️ Главное меню» на первом уровне и терминальных экранах
Добавляется последней строкой клавиатуры (`app:main-menu`):

| Story | Экран |
|---|---|
| `catalog` | `list` |
| `learning` | `my-study` |
| `learning` | `complete` → урок/проект завершён |
| `learning` | `complete` → поток завершён |

### UX-3: Кнопка «⬅️ Назад к {предыдущий уровень}» на тупиковых экранах
| Story | Экран | Кнопка |
|---|---|---|
| `view-stream` | `complete` | `⬅️ Назад к списку` → `catalog:list` |
| `view-stream` | `archive` | `⬅️ Назад к списку` → `catalog:list` |
| `progress` | `progress` | `⬅️ Назад к обучению` → `learning:my-study` |
| `monitor` | `students` | `⬅️ Назад к потоку` → `view-stream:view:{id}` |
| `activate-stream` | `activate` | `⬅️ Назад к потоку` → `view-stream:view:{id}` |

### UX-4: Где кнопка «Назад» НЕ добавляется (пользователь в процессе)
| Story | Экран | Причина |
|---|---|---|
| `learning` | `complete` → следующий шаг | студент в процессе обучения |
| `create-stream` | шаги 0–10 | пользователь в wizard-процессе |
| `onboarding` | все шаги | свой механизм навигации |
| `enroll` | `enroll` | делегирует в `learning:my-study` |

### UX-5: Удаление отработанных кнопок в CreateStreamStory
- Шаги 4–8 (кнопки «Принять»/«Пропустить»): при нажатии — `removePrevKeyboard: true`.
- Шаг 9 (кнопка «Пропустить» для группы): при нажатии — `removePrevKeyboard: true`.
- Остальные шаги (0, 10) — без изменений.

### UX-6: Документация правил навигации бота
- Создать `packages/core/src/ui/bot/README.md`:
  - Когда и как добавлять кнопку «Назад» / «Главное меню».
  - Когда НЕ добавлять (процесс vs навигация).
  - Как использовать `app:main-menu`.
  - Как story может управлять `removePrevKeyboard`.
  - Правило: story сама решает об удалении кнопок, единого автоматизма нет.

### UX-7: Актуализация user-stories
- Привести `packages/stream/src/user-stories.md` в соответствие с реальным состоянием кода:
  - US-4 п.5: убрать кнопку `[ ◀️ Назад ]` (не реализована, будет в треке `stream_ux_backlog_20260613`).
  - US-4 п.9: пометить как «запланировано (трек stream_ux_backlog)».
  - US-7: broadcast пометить как «запланировано».
  - US-8: статус «Отстает» пометить как «запланировано».
  - Добавить актуальную информацию о кнопке «↩️ Главное меню».
