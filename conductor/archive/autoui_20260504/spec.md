# Спецификация: Архитектура Auto-UI

## Обзор
Реализовать архитектуру автоматического построения пользовательских интерфейсов (`auto-ui`) внутри абстрактного каркаса приложения (`@u7/core`). Слой `ui` должен автоматически генерировать меню и обеспечивать взаимодействие с пользователем (навигация, запуск UseCase) на основе метаданных доменных модулей из слоя `api`. Текущий фокус — разработка абстракций `UIApp`/`UIModule`, роутинга и конкретной реализации Console UI, которая будет максимально переиспользуема в Bot UI (Telegram) и Web UI.

## Функциональные требования

1. **Абстракции UIApp и UIModule:**
   - Базовая логика для всех дочерних интерфейсов (`auto-ui`, `bot-ui` и т.д.).
   - `UIApp` обеспечивает глобальное управление, роутинг на уровне приложения и читает приветственное сообщение `about.md` из корня приложения.
   - `UIModule` оборачивает доменный модуль (`api.Module`), предоставляет метаданные (в том числе загруженные из `about.md` рядом с модулем) и агрегирует UI-логику для UseCases.

2. **Абстракция навигации (Router):**
   - Реализовать систему навигации на основе путей (например, `/app`, `/module-name`, `/module-name/aggregate/use-case`).
   - Роутер должен быть независимым от конкретного интерфейса, чтобы переиспользоваться в console, bot (callback_data) и web (URL).

3. **Описания (about.md):**
   - Реализовать механизм в `@u7/core/ui`, который автоматически загружает и парсит файл `about.md`, находящийся в соответствующих папках (для App и Module).
   - Эти файлы используются для вывода приветственного текста и информации (в том числе по ролям) на стартовых экранах.

4. **Иерархическое меню Приложения:**
   - **Уровень Приложения:** Приветственная информация и меню со списком доступных модулей.
   - **Уровень Модуля:** Приветственная информация модуля (из `about.md`) и список доступных UseCase, сгруппированных по агрегатам (или ролям).
   - **Уровень UseCase:** Пошаговый сбор данных (или формы) на основе валидационной схемы `UcDocType`, предоставленной слоем `api`.

5. **Console UI (auto-ui):**
   - Реализовать консольную адаптацию `auto-ui` как первый рабочий пример.
   - Консольный UI должен уметь рендерить текущий маршрут (выводить текст) и принимать пользовательский ввод для перехода по роутам или выполнение UseCase.

## Структура пакета
- Логика должна располагаться в `@u7/core/src/ui` (базовые абстракции) и `@u7/core/src/ui/auto-ui` (реализация для консоли).

## Критерии приёмки
- Написаны базовые абстракции `UIApp` и `UIModule` в слое `ui`.
- Написаны абстракции `Router` (или аналогичные сущности).
- Добавлен механизм чтения `about.md` для приложения и модулей.
- Написаны модульные/интеграционные тесты для роутинга и загрузки описаний.
- Создан тестовый консольный интерфейс для демо-приложения, который позволяет перемещаться по меню и запускать UseCase.

## За рамками
- Фактическая реализация Telegram (Grammy) или Web (Elysia) интерфейсов.
- Сложная система прав доступа и аутентификации.
