# Спецификация: Исправления доменного слоя Stream

## Обзор
Дополнить доменный и API слой модуля `@u7-scl/stream` недостающей функциональностью для полного покрытия пользовательских сценариев US-2, US-4, US-7, US-8 из `packages/stream/src/user-stories.md`.

## Функциональные требования

### FT-1: StreamDs.completeStep — определение уровней lesson/project
При завершении шага `StreamDs` должен определять, завершён ли урок или проект, и возвращать соответствующий `CompletionResult`:
- `level: 'step'` — следующий шаг в том же уроке
- `level: 'lesson'` — переход на первый шаг следующего урока (возвращается `completedLessonId`)
- `level: 'project'` — переход на первый шаг следующего проекта (возвращается `completedProjectId`)
- `level: 'stream'` — все шаги пройдены, студент завершает поток

### FT-2: ActivateStreamUc — выдача первого шага студентам
При вызове `ActivateStreamUc`:
1. Поток переводится `enrollment → active`
2. Для всех студентов потока без выданных шагов — вызывается `issueStep(firstStepId)`
3. Все изменения сохраняются атомарно

### FT-3: GetStreamUc — получение одного потока
Новый use-case для получения потока по `streamId`. Без авторизации (публичный).

### FT-4: GetStudentByUserUc — поиск активной записи студента
Новый use-case: поиск единственной активной (`status = 'active'`) записи студента по `userId`. Нужен для `📖 Моя учёба`.

### FT-5: ListStreamStudentsUc — список студентов потока
Новый use-case: список всех студентов указанного потока. Доступен ментору потока или админу.

### FT-6: Проверки авторизации в UC
Добавить проверку прав актора в:
- `ActivateStreamUc` — только ментор потока или админ
- `ArchiveStreamUc` — только ментор потока или админ
- `CompleteStreamUc` — только ментор потока или админ
- `GetStudentProgressUc` — сам студент, ментор потока или админ

### FT-7: EnrollStudentUc — проверка существующей активной записи
При зачислении проверять через `StudentRepo.getByUser`, нет ли у пользователя уже активной записи в другом потоке. Если есть — ошибка.

## Критерии приёмки
- [ ] 61 существующий тест продолжает проходить
- [ ] `StreamDs.completeStep` корректно определяет все 4 уровня
- [ ] `ActivateStreamUc` выдаёт первый шаг всем зачисленным студентам
- [ ] `GetStreamUc` возвращает поток по UUID
- [ ] `GetStudentByUserUc` возвращает активную запись по userId
- [ ] `ListStreamStudentsUc` возвращает список студентов потока
- [ ] Неавторизованные вызовы `activate/archive/complete` отклоняются
- [ ] Попытка записи студента с уже активным потоком отклоняется
