# Аудит базовых классов core: обработка ошибок и логирование

Дата: 2026-06-16

## BotUserStory

- **Статус:** ✅ Унифицирован
- Метод `handleError` перенесён из `U7BotUserStory` в `BotUserStory` и использует `this.logger?.error('bot', ...)`.
- Геттер `logger` добавлен через `getGlobalLogger()`.
- Все наследники автоматически получают доступ к логгеру и унифицированной обработке.
- `console.error` заменён на структурированное логирование.

## ApiModule.dispatch() (метод `execute`)

- **Статус:** ✅ Логирование присутствует, ошибки не подавляются
- Успешные запросы логируются через `this.logger.info()` с временем выполнения.
- Ошибки **не** перехватываются — пробрасываются до вызывающего кода (контроллер или глобальный middleware).
- Ad-hoc try/catch отсутствует.

## UseCase (`handle`)

- **Статус:** ✅ Корректная проброска ошибок
- `handle()` вызывает последовательно: `checkAuth` → `validateInput` → `execute` → `validateOutput`.
- Ошибки авторизации и валидации выбрасываются через `throwBaseErrors`.
- Ошибки из `execute()` пробрасываются без изменений.
- Подавления ошибок нет.

## BotController

- **Статус:** ✅ Улучшен в рамках трека
- Добавлен `logger` через `getGlobalLogger()`.
- `handleCallback` и `handleMessage` обёрнуты в try/catch.
- `handleError` улучшен: использует `fromError()` для различения типов ошибок.
- `internal`, `unauthorized`, `default` ошибки логируются; остальные — нет.

## Вывод

Базовые классы core корректно обрабатывают и пробрасывают ошибки.
Необработанные ошибки из story перехватываются на уровне контроллера (BotController)
и дополнительно на уровне глобального middleware в main.ts.
