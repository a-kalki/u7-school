# Спецификация: stream_api_bot_20260601

## Обзор

Этот трек покрывает создание API-слоя (UseCases, Module) для модуля `@u7-scl/stream`, реализацию UI-контроллера для Telegram-бота (`StreamController`) и полную интеграцию в бота `u7-bot`.

Цель трека — связать доменную логику потоков с интерфейсом Telegram-бота так, чтобы Любопытный Посетитель мог записаться на курс, Студент — проходить шаги, а Ментор — создавать и запускать потоки.

---

## Функциональные требования

### FR-1: API-слой (UseCases)

Необходимо создать следующие сценарии использования (UseCases):

| Название UseCase | Параметры (DTO) | Описание |
|---|---|---|
| `CreateStreamUc` | `title`, `description`, `moduleId`, `startDate`, `telegramGroupId?` | Создаёт поток. Использует `CourseFacade.getModuleSnapshot()` для снимка структуры модуля. Сохраняет поток. |
| `ActivateStreamUc` | `streamId` | Меняет статус потока на `active`. Для всех записанных студентов активирует первый шаг обучения. |
| `CompleteStreamUc` | `streamId` | Завершает поток (статус `completed`). |
| `ArchiveStreamUc` | `streamId` | Архивирует поток (статус `archived`). |
| `EnrollStudentUc` | `streamId`, `userId` | Записывает пользователя в поток. Создаёт `StreamStudentAr` со статусом `active` и первым шагом. **Меняет роль пользователя на `STUDENT`** в модуле `user`. |
| `CompleteStepUc` | `userId`, `stepId` | Помечает шаг выполненным, находит следующий и выдаёт его. Если шагов больше нет, помечает студента `completed`. |
| `GetStudentProgressUc`| `userId` | Возвращает информацию об активном потоке студента, текущем шаге, прогрессе (пройдено X из Y шагов). |
| `ListStreamsUc` | `status?`, `mentorId?` | Возвращает список потоков для витрины или панели ментора. |

### FR-2: Контроллер Telegram-бота (`StreamController`)

Класс `StreamController` располагается в `@u7-scl/stream/src/ui/bot/controller/stream-controller.ts` и работает в stateless-режиме:

1. **Команда `/streams` (Витрина потоков):**
   - Отображает список потоков со статусом `enrollment` и `active`.
   - Позволяет кликнуть на поток, посмотреть карточку (цели, результаты, ментор, статистика записавшихся).
   - Для Guest/Candidate показывает кнопку `[ 📝 Записаться на поток ]`.
   - Для зачисленных показывает плашку `[ Вы уже учитесь ]`.

2. **Команда `/my_study` (Учёба студента):**
   - Показывает текущий активный шаг, урок, проект.
   - Выводит кнопку `[ ✅ Выполнено! ]`.
   - При завершении шага/урока/курса красиво форматирует MarkdownV2-ответ.

3. **Команда `/mentor_panel` (Панель ментора):**
   - Доступна только для ролей `MENTOR` и `ADMIN`.
   - Позволяет запустить пошаговое создание потока (через меню ввода u7-bot).
   - Показывает список потоков ментора с кнопками `[ 🚀 Запустить ]` и `[ 👥 Список студентов ]`.

### FR-3: Интеграция в `u7-bot`

1. **Регистрация обработчиков:**
   - Зарегистрировать `StreamController` в `apps/u7-bot/src/api-app.ts` (DI-контейнер).
   - Создать `StreamHandler` в `apps/u7-bot/src/handlers/stream-handler.ts` для перехвата сообщений и callback-запросов.
   - Подключить обработчик в `bot.ts`.

2. **Главное меню u7-bot:**
   - Настроить меню в `top-menu-handler.ts` так, чтобы:
     - Кнопка `📚 Потоки курсов` была доступна всем.
     - Кнопка `📖 Моя учёба` динамически появлялась у роли `STUDENT`.
     - Кнопка `🛠️ Панель ментора` появлялась у ролей `MENTOR`/`ADMIN`.

## Нефункциональные требования

- **Безопасность:** Проверка прав доступа через `StreamPolicy` на уровне UseCase перед выполнением любого действия.
- **Интеграция между модулями:** Взаимодействие с модулем `user` (для обновления ролей) и `course` (для получения снимка модуля) только через фасады (`UserFacade`, `CourseFacade`), следуя архитектурному гайдлайну.
- **Форматирование:** Текст сообщений должен соответствовать `product-guidelines.md` (разметка MarkdownV2, эмодзи, понятные ошибки).

## Критерии приёмки

- [ ] Все UseCase имеют модульные тесты с моками репозиториев и фасадов.
- [ ] `StreamController` корректно переводит `BotUpdate` в вызовы UseCase и формирует красивую MarkdownV2 разметку.

- [ ] При сдаче последнего шага студент получает сообщение об окончании курса.

- [ ] Все тесты проходят, Biome линтинг и TSLint не выдают ошибок.
