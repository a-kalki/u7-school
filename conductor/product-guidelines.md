# Руководство по продукту (Guidelines)

## UX и Навигация
- **Гибридный подход в Telegram:** В основном чате бота приоритет отдается минимализму (использование инлайн-кнопок, редактирование сообщений вместо отправки новых для чистоты истории).
- **Пошаговость (Wizard):** Сложные действия разбиваются на последовательные, понятные шаги (вопрос-ответ).
- **Интерфейсы для разных ролей:**
  - *Студенты:* Взаимодействуют преимущественно через Telegram-бота (получение заданий, отправка ответов). Уроки могут ссылаться на внешние ресурсы (например, W3Schools).
  - *Менторы:* На начальном этапе управление контентом и создание курсов может осуществляться через CLI или другие легковесные/простые инструменты. В будущем для сложных задач будет разработан Web-интерфейс.

## Обработка ошибок
- **Явное информирование:** Бот не должен скрывать ошибки, он должен понятно сообщать о том, что произошло.
- **Помощь ментора:** Если ошибка возникает на стороне студента, бот должен подска подсказку, что за разъяснениями можно обратиться напрямую к своему ментору.

## Оформление текста
- **MarkdownV2 Акценты:** Использование возможностей форматирования Telegram (жирный шрифт, курсив, моноширинный текст для кода) для акцентирования внимания на важных учебных терминах.
- **Эмодзи-навигация:** Применение стандартизированных эмодзи в меню и кнопках для быстрой визуальной ориентации (например, 📚 для курсов, 👤 для профиля).
