# Модуль User

Модуль "User" (Пользователи) отвечает за управление учетными записями, ролями и идентификацией пользователей в системе U7.

## Ответственность
- Хранение информации о пользователях (имя, UUID, Telegram ID и т.д.).
- Управление ролевой моделью (ADMIN, MENTOR, USER).
- Обеспечение базовой информации о пользователе для других модулей (например, проверка прав в политиках курса).

## Как взаимодействовать
Действия выполняются путем вызова команд (UseCases) модуля `user`.
В CLI это делается с помощью ввода: `/user/<команда>`
Например, чтобы получить список пользователей, введите: `/user/list-users`
Если команда требует данных, CLI запросит ввод необходимых полей (payload).

## Доступные действия и права по ролям

### 1. Для всех (включая неавторизованных)
Эти действия доступны любому пользователю, даже без авторизации.
- **`list-users`** — Просмотр списка всех пользователей. Поддерживает фильтрацию по ролям (например, можно запросить только список менторов).
- **`get-user`** — Получение детальной информации о пользователе по его UUID.
- **`get-user-by-telegram-id`** — Поиск и получение информации о пользователе по его Telegram ID.

### 2. Только для роли ADMIN (Администратор)
Администраторы обладают исключительными правами на создание и управление учетными записями.
- **`create-user`** — Регистрация/создание нового пользователя в системе. При создании администратор может сразу назначить пользователю нужные роли (например, сделать его MENTOR или ADMIN).
*(Обычные пользователи и менторы не могут создавать других пользователей).*
