# Руководство по работе с контентом W3Schools

Данный пакет предназначен для скачивания, парсинга и ИИ-обогащения учебных материалов с сайта W3Schools.

## 1. Скачивание контента

Для скачивания курсов используется утилита `wget`. Скачивание необходимо производить в папку `packages/w3school/www.w3schools.com/`.

### Команда для скачивания (пример для курса SQL):
```bash
cd packages/w3school/www.w3schools.com/
wget --recursive --level=1 --no-parent --page-requisites --adjust-extension --convert-links --no-check-certificate https://www.w3schools.com/sql/
```

**Важные флаги:**
- `--recursive`: рекурсивное скачивание.
- `--level=1`: скачиваем только страницы самого курса (без ухода на внешние ссылки).
- `--adjust-extension`: сохраняет файлы с расширением `.html`.
- `--convert-links`: подправляет ссылки для локального просмотра.

---

## 2. Использование CLI инструмента `w3s`

Инструмент `w3s` доступен через `bun`. Все команды запускаются из корня проекта.

### Просмотр текущего состояния
```bash
bun w3s status
```
Показывает, какие курсы скачаны, какие уже спарсены и какой процент уроков обогащен через ИИ.

### Парсинг (конвертация HTML -> Markdown)
```bash
bun w3s parse --course <название>
```
**Опции:**
- `--course <name>`: парсить только указанный курс (например, `sql`).
- `--force`: полностью перезаписать папку курса в `output`.
- `--new`: парсить только те файлы, которых еще нет в `output`.

### Обогащение (генерация summary через Gemini AI)
```bash
GEMINI_API_KEY=your_key bun w3s enrich --course <название>
```
Эта команда анализирует содержимое `.md` файлов и добавляет поле `summary` в `syllabus.json`.
*Примечание: Если урок уже имеет summary, запрос к ИИ не отправляется (экономия токенов).*

### Очистка исходников
```bash
bun w3s clean --course <название>
```
Удаляет скачанные HTML-файлы из папки `www.w3schools.com`. **Внимание:** результат парсинга в папке `output` при этом сохраняется.

---

## 3. Рекомендуемый рабочий процесс (Workflow)

1. **Скачивание:** Используйте `wget` для загрузки нужного раздела в `www.w3schools.com/`.
2. **Проверка:** Запустите `bun w3s status`, чтобы убедиться, что курс виден как `[+] (новое)`.
3. **Парсинг:** Запустите `bun w3s parse --course <name>`. В папке `output/<name>` появятся Markdown-файлы и `syllabus.json`.
4. **Обогащение:** Запустите `bun w3s enrich --course <name>`, чтобы добавить ИИ-описания.
5. **Финализация:** Проверьте результат и, если исходные HTML больше не нужны, выполните `bun w3s clean --course <name>`.

---

## Структура выходных данных

Результат работы находится в `packages/w3school/output/<course_name>/`:
- `*.md`: уроки в формате Markdown.
- `syllabus.json`: структурированный список всех уроков с заголовками, путями к файлам и описаниями.
- `README.md`: автоматически сгенерированное оглавление курса со ссылками на локальные файлы.
