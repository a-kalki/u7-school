# Спецификация экранов Stream (UI Spec)

Документ описывает все экраны Telegram-бота модуля Stream: их содержимое, кнопки, условия видимости и доменную логику.  
Служит **отправной точкой для разработки** — что уже реализовано, что запланировано.

Условные обозначения:
- ✅ реализовано
- 🔧 в текущем треке (`stream_mentor_control_20260703`)
- 📋 в бэклоге

---

## Карта экранов

```
Главное меню
├─ 📚 Наши потоки → Каталог (S01)
│   └─ поток → Карточка потока (S02)
│       ├─ 📖 Программа курса → Программа (S03)
│       ├─ 📋 Детали → Детали потока (S04)
│       ├─ 👥 Студенты → Мониторинг (S07)
│       └─ студент → Карточка студента (S08)
├─ 📖 Моя учёба → Обучение (S05)
│   └─ 📊 Мой прогресс → Прогресс (S06)
└─ 🛠️ Создать поток → Wizard (S09)
```

## S01 — Каталог потоков

**Как попасть:** кнопка `📚 Наши потоки` в главном меню.  
**Кому:** всем ролям.

**Содержание:**
- Заголовок `📚 Наши потоки`
- Легенда: `🟢 — идёт набор   🔵 — идёт обучение   ⚪ — завершён`
- Кнопки-потоки: `{статус} {StreamTitle}` — только `enrollment` и `active`

**Кнопки:**

| Текст | Код | Условие | Статус |
|-------|-----|---------|--------|
| `{эмодзи} {StreamTitle}` | `view-stream:view:{id}` | поток enrollment/active | ✅ |
| `🔍 Включить завершённые` | `catalog:toggle-completed` | есть completed-потоки | ✅ |
| `🔍 Скрыть завершённые` | `catalog:toggle-completed` | режим «с завершёнными» | ✅ |

**Доменная логика:**
- `list-streams` → фильтр `['enrollment', 'active']` (или `['enrollment', 'active', 'completed']` при включённом режиме)

## S02 — Карточка потока

**Как попасть:** клик на поток в S01.  
**Кому:** всем ролям.

**Содержание:**
```
📋 *{StreamTitle}*
_{StreamDescription}_
👤 Ментор: {MentorName}
📅 Старт: {StartDate}
🕐 Время: {HH:MM}
👥 Студентов: {N}
📌 Статус: {StatusLabel}
🔗 {TelegramGroupInvite}
```

**Кнопки — менторские (только владелец потока):**

| Текст | Код | Статус потока | Статус |
|-------|-----|---------------|--------|
| `🚀 Запустить` | `activate-stream:activate:{id}` | `enrollment` | ✅ |
| `✅ Завершить` | `view-stream:complete:{id}` | `active` | ✅ |
| `📁 В архив` | `view-stream:archive:{id}` | любой | ✅ |

> **Подтверждения:** `🚀 Запустить` — без подтверждения.  
> `✅ Завершить` → «⚠️ Завершить поток?» → `Да, завершить` / `Отмена`.  
> `📁 В архив` → «⚠️ Отправить в архив?» → `Да, в архив` / `Отмена`.

**Кнопки — публичные (всем):**

| Текст | Код | Статус потока | Статус |
|-------|-----|---------------|--------|
| `👥 Студенты` | `monitor:students:{id}` | любой | ✅ |

**Кнопки — гостевые (только НЕ владелец потока):**

| Текст | Код | Статус потока | Доп. условие | Статус |
|-------|-----|---------------|-------------|--------|
| `📝 Записаться` | `enroll:enroll:{id}` | `enrollment` | `canEnroll(actor)` | ✅ |
| `📖 Программа курса` | `view-stream:program:{id}` | `enrollment` | — | ✅ |
| `🔔 Уведомить о наборе` | `view-stream:notify:{id}` | `active` | `canEnroll(actor)` | ✅ |
| `📋 Детали` | `view-stream:details:{id}` | любой | — | 🔧 |

**Кнопки — всегда:**

| Текст | Код | Статус |
|-------|-----|--------|
| `⬅️ Назад к списку` | `catalog:list` | ✅ |

**Доменная логика:**
- `get-stream` → карточка
- `list-stream-students` → подсчёт студентов (если нет доступа → 0)
- `get-user` (ментор) → имя ментора
- `StreamPolicy.canEdit(actor, stream)` → менторские кнопки
- `StreamPolicy.canEnroll(actor)` → кнопки записи/уведомлений

## S05 — Обучение (активный шаг)

**Как попасть:** `📖 Моя учёба` в главном меню.  
**Кому:** только `STUDENT`.

**Содержание:**
```
📖 Поток: {StreamTitle}
📁 Проект: {ProjectTitle}
📚 Урок: «{LessonTitle}»
🔢 p{projectIndex}-l{lessonIndex}

{StepContent}
```

**Кнопки:**

| Текст | Код | Статус |
|-------|-----|--------|
| `✅ Выполнено` | `learning:complete` | ✅ |
| `📊 Мой прогресс` | `progress:show` | ✅ |
| `↩️ Главное меню` | `app:main-menu` | ✅ |

**Доменная логика — выполнение шага:**
1. `complete-step` → `StreamDs.completeStep`
2. Текущий шаг → `completed` в `StepRecord`
3. Поиск следующего шага в `contentSnapshot`
4. Если найден → выдача (`issued`), уровень перехода: `step` / `lesson` / `project`
5. Если всё пройдено → `StudentAr.complete()` (статус `completed`)

**Рендеринг после выполнения:**
- Обычный шаг → сразу следующий, без промежуточного сообщения
- Конец урока → «🎉 Урок завершён!» + `▶️ Начать следующий урок`
- Конец проекта → «🚀 Проект завершён!» + `▶️ Начать следующий проект`
- Конец потока → «🏆 Поток полностью завершён!»

## S08 — Карточка студента

**Как попасть:** клик на студента в S07.  
**Кому:** ментор / админ.

**Содержание:**
```
👤 {StudentName}
📌 Статус: Активен
📊 Прогресс: 12/30 (40%)
```

**Кнопки:**

| Текст | Код | Условие | Статус |
|-------|-----|---------|--------|
| `📁 История шагов` | `monitor:history:{studentId}` | — | ✅ (заглушка) |
| `❌ Отчислить` | `monitor:expel:{studentId}` | `StudentPolicy.canExpel(actor)` | 🔧 |
| `⬅️ Назад к списку` | `monitor:students:{streamId}` | — | ✅ |

> **Подтверждение отчисления 🔧:** «⚠️ Вы уверены, что хотите отчислить {StudentName}?» → `Да, отчислить` / `Отмена`.

**Доменная логика отчисления 🔧:**
- `expel-student` → `StudentAr.expel()` (статус `expelled`)
- `UserFacade.removeRole(userId, STUDENT)` — **без** добавления другой роли
- Запись **не удаляется** из репозитория

## S09 — Wizard создания потока

**Как попасть:** `🛠️ Создать поток` в главном меню.  
**Кому:** `MENTOR`, `ADMIN`.

**Шаги wizard'а:**

| # | Поле | Источник по умолчанию | Кнопки |
|---|------|----------------------|--------|
| 0 | Выбор модуля | список опубликованных | — |
| 1 | Название | `module.title` | — |
| 2 | Описание | `module.description` | — |
| 3 | Дата старта (YYYY-MM-DD) | — | — |
| 4 | Цель | `module.goal` | ✅ Принять / ⏭️ Пропустить |
| 5 | Результат | `module.result` | ✅ Принять / ⏭️ Пропустить |
| 6 | Правила | `module.rules` | ✅ Принять / ⏭️ Пропустить |
| 7 | Целевая аудитория | `module.targetAudience` | ✅ Принять / ⏭️ Пропустить |
| 8 | Дополнительно | `module.additional` | ✅ Принять / ⏭️ Пропустить |
| 9 | Telegram-группа | — | ⏭️ Пропустить |
| 10 | Кодовое слово 🔧 | — | ⏭️ Пропустить |

**Доменная логика:**

## S10 — Проверка кодового слова 🔧

**Как попасть:** `📝 Записаться` на потоке с `enrollmentKey` (S02).  
**Кому:** Guest / Candidate.  
**Статус:** 🔧 в текущем треке.

**Сценарий:**
1. Бот: «Введите кодовое слово для записи на поток»
2. Пользователь вводит слово
3. Верно → вызов `enroll-student` с `enrollmentKey`
4. Неверно → «❌ Неверное слово. Осталось попыток: {N}» (всего 3)
5. 3 неудачи → «❌ Попытки исчерпаны.» → возврат в S02

**Кнопки:**

| Текст | Код | Статус |
|-------|-----|--------|
| `❌ Отмена` | `view-stream:view:{id}` | 🔧 |

**Архитектура:**
- Story (UI) проверяет слово для UX и подсчёта попыток
- Use-case `enroll-student` **тоже** проверяет слово — безопасность на уровне домена

## Бэклог

### 📋 Навигация по истории шагов (S05)
Кнопки `◀️ Назад` / `Вперед ▶️` для повторения пройденного материала.

### 📋 Сбор обратной связи
Мини-опрос после урока/проекта: оценка сложности 1–5.

### 📋 Типы шагов
Тест (квиз), практическое задание (ревью).

### 📋 Заморозка студента
Академический отпуск — приостановка обучения.

### 📋 Снятие ролей при завершении потока
При завершении потока ментором — автоматическое снятие `STUDENT` у всех студентов. Требует анализа: кто может менять роль (ментор, админ, сам пользователь) и как это встраивается в существующие use-case'ы user-модуля.

### 📋 Самостоятельный выход из потока
Студент покидает поток сам.

### 📋 Синхронизация контента
Обновление `contentSnapshot` активного потока при изменении модуля.

### 📋 Роли при зачислении
Снятие `CANDIDATE` при `+STUDENT` (сейчас: только добавление `STUDENT`).
