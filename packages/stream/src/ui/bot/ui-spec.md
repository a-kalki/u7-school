# Спецификация экранов Stream (UI Spec)

Документ описывает все экраны Telegram-бота модуля Stream: их содержимое, кнопки, условия видимости и доменную логику.
Служит **отправной точкой для разработки** — что уже реализовано, что запланировано.

Условные обозначения:
- ✅ реализовано
- ❌ не реализовано
- 📋 в бэклоге / запланировано в треке

---

## Карта экранов

```
Главное меню
├─ 📚 Программы → Каталог курсов (S00)                     ← 📋 трек 5
│   └─ курс → Карточка курса (S00a) → Программа курса (S00b)
├─ 📚 Потоки → Каталог потоков (S01)
│   └─ поток → Карточка потока, curious-режим (S02)
│       ├─ 📖 Программа → Программа модуля (S03)
│       ├─ 📋 Детали → Детали потока (S04)
│       ├─ 📊 Студенты → Мониторинг (S07) → Карточка студента (S08)
│       └─ 📚 Курсы → Карточка курса (S00a)
├─ 📖 Моя учёба → Хаб (S05) → Прогресс (S06) / Дерево уроков (S05b)    ← ✅ хаб, 📋 дерево
├─ 🛠️ Инструменты ментора → Мои потоки (S02m) / Создать поток (S09) / Мониторинг (S07)   ← 📋 трек 5
├─ ✍️ Инструменты автора → (отложено, см. раздел ниже)      ← 📋 техдолг
├─ 📝 Заполнить анкету (onboarding)
├─ 💬 Сообщество школы
└─ ❓ Помощь
```

**Принцип веток (см. `architecture-evolution.md` §2.10, §2.11):**
- «Потоки» — только curious-функциональность, **без менторских инструментов**.
- Управление потоком — только в «Инструментах ментора» → «Мои потоки» → S02m (mentor-режим).
- S07/S08 (студенты) открыты всем ролям — прозрачность; кнопки действий — только ментору/админу.

## S00 — Каталог курсов (📋 трек 5)

**Как попасть:** «📚 Программы» в главном меню.
**Кому:** всем ролям.

**Содержание:**
- Список опубликованных курсов: title, краткое описание, эмодзи направления (tech/business), сводка объёма.

**Кнопки:**

| Текст | Код | Статус |
|-------|-----|--------|
| `{эмодзи} {CourseTitle}` | `course-catalog:view:{id}` | 📋 трек 5 |

**Как попасть:** клик на курс в S00, либо `📚 Курсы` из S02.

## S00b — Программа курса (drill-down) (📋 трек 5)

**Как попасть:** `📖 Развернуть программу` в S00a.
**Кому:** всем ролям. Curious-уровень ContentPath: заголовки и объём видны, тела шагов — нет.

> ✅ ContentPath (VO + UC resolve-content-path + CourseDs) реализован в треке `content_path_20260708`. Интеграционные тесты: `tests/bot/integration/course/content-path.integration.test.ts`.

**Содержание (drill-down):**
```
📖 *Программа курса: {CourseTitle}*
Этап 1: {PhaseTitle} (tech)
  📁 {ModuleTitle} — {P} проектов · {L} уроков · {S} шагов
    📁 {ProjectTitle}
      📝 {LessonTitle} (5 шагов: 📄📄💻📄📄)
        • {StepTitle} 📄
        • {StepTitle} 💻
```
- У урока: число шагов, типы шагов иконками (📄 текст / 💻 код / 📎 файл).
- У шага: только заголовок + тип, **без тела** (`content`/`code` скрыты).
- Пагинация при превышении лимита 4096 символов.

**Кнопки:**

| Текст | Код | Статус |
|-------|-----|--------|
| `📝 Записаться` | `enroll:enroll:{streamId}` | 📋 трек 5 (если у модуля есть enrollment-поток) → S10 |
| `⬅️ Назад` | `course-catalog:view:{courseId}` | 📋 трек 5 |

## S01 — Каталог потоков

**Как попасть:** кнопка `📚 Потоки` в главном меню (переименование «Наши потоки», 📋 трек 5).
**Кому:** всем ролям.

**Содержание:**
- Заголовок `📚 Потоки`
- Легенда: `🟢 — идёт набор   🔵 — идёт обучение   ⚪ — завершён`
- Кнопки-потоки: `{статус} {StreamTitle}` — только `enrollment` и `active`

**Кнопки:**

| Текст | Код | Условие | Статус |
|-------|-----|---------|--------|
| `{эмодзи} {StreamTitle}` | `view-stream:view:{id}` | поток enrollment/active | ✅ |
| `🔍 Включить завершённые` | `catalog:toggle-completed` | есть completed-потоки | ✅ |
| `🔍 Скрыть завершённые` | `catalog:toggle-completed` | режим «с завершёнными» | ✅ |

**Доменная логика:**
- `list-streams` → фильтр `['enrollment', 'active']` (или `['enrollment', 'active', 'completed']` при включённом режиме)

## S02 — Карточка потока (curious-режим)

**Как попасть:** клик на поток в S01 (ветка «Потоки»).
**Кому:** всем ролям. **Только curious-функциональность, без менторских инструментов.**

**Содержание:**
```
📋 *{StreamTitle}*
_{StreamDescription}_
👤 Ментор: {MentorName}
📅 Старт: {StartDate}
🕐 Время: {HH:MM}
👥 Студентов: {N}
📌 Статус: {StatusLabel}
```

**Кнопки — публичные (всем ролям, включая ментора как зрителя):**

| Текст | Код | Статус потока | Статус |
|-------|-----|---------------|--------|
| `📖 Программа` | `view-stream:program:{id}` | любой (📋 трек 5: сейчас только `enrollment`) → S03 | ✅ / 📋 |
| `📋 Детали` | `view-stream:details:{id}` | любой (📋 трек 5: сейчас только `enrollment` не-владельцу) → S04 | ✅ / 📋 |
| `📊 Студенты` | `monitor:students:{id}` | любой → S07 (прозрачность) | ✅ |
| `📚 Курсы` | `view-stream:courses:{id}` | любой → S00a (курсы, в которые входит модуль) | 📋 трек 5 |

**Кнопки — гостевые (только НЕ владелец потока, `canEnroll`):**

| Текст | Код | Статус потока | Статус |
|-------|-----|---------------|--------|
| `📝 Записаться` | `enroll:enroll:{id}` | `enrollment` → S10 | ✅ |
| `🔔 Уведомить о наборе` | `view-stream:notify:{id}` | `active` | ✅ |
| `🔔 Ждать новый набор` | `view-stream:notify:{id}` | `completed` | 📋 трек 5 (предзапись) |

**Кнопка — всегда:**

| Текст | Код | Статус |
|-------|-----|--------|
| `⬅️ Назад к списку` | `catalog:list` | ✅ |

> ⚠️ В этом режиме **нет** lifecycle-кнопок ментора (`🚀 Запустить` / `✅ Завершить` / `📁 В архив`). Управление — через S02m (ветка «Инструменты ментора»).

**Доменная логика:**
- `get-stream` → карточка
- `list-stream-students` → подсчёт студентов
- `get-user` (ментор) → имя ментора

## S02m — Карточка потока (mentor-режим) (📋 трек 5)

**Как попасть:** «🛠️ Инструменты ментора» → «📋 Мои потоки» → поток (где я ментор).
**Кому:** только владелец потока / ADMIN (`StreamPolicy.canEdit`). Тот же поток, что в S02, но в режиме управления.

**Содержание:** как в S02 + `📊 Сводка потока` (счёт по статусам: active/advanced/not_advanced/abandoned/enrolled; медиана/отстающие/сортировка — трек 6).

**Кнопки — панель управления:**

| Текст | Код | Статус потока | Статус |
|-------|-----|---------------|--------|
| `🚀 Запустить` | `activate-stream:activate:{id}` | `enrollment` | ✅ (перенос из S02) |
| `✅ Завершить` | `view-stream:complete:{id}` | `active` | ✅ (перенос из S02) |
| `📁 В архив` | `view-stream:archive:{id}` | любой | ✅ (перенос из S02) |
| `👥 Студенты` | `monitor:students:{id}` | любой → S07/S08 **с действиями** | ✅ |

> **Подтверждения:** `🚀 Запустить` — без подтверждения.
> `✅ Завершить` → «⚠️ Завершить поток?» → `Да, завершить` / `Отмена`.
> `📁 В архив` → «⚠️ Отправить в архив?» → `Да, в архив` / `Отмена`.

**Кнопка — всегда:** `⬅️ Назад к моим потокам` → `mentor:my-streams`.

## S05 — Хаб «Моя учёба»

**Как попасть:** `📖 Моя учёба` в главном меню.
**Кому:** только `STUDENT`.

**Содержание:**
```
📖 *Моя учёба*

Выберите действие:
```

**Кнопки — активный студент (enrolled/active):**

| Текст | Код | Статус |
|-------|-----|--------|
| `▶️ Продолжить` | `learning:my-study:continue` → S05a (текущий шаг) | ✅ |
| `📂 Уроки` | `learning:my-study:lessons` → S05b (дерево) | 📋 трек 4 |
| `📊 Мой прогресс` | `progress:progress:{streamId}` → S06 | ✅ |
| `🚪 Покинуть поток` | `learning:my-study:leave-confirm` → confirm | ✅ |
| `↩️ Главное меню` | `app:main-menu` | ✅ |

**Кнопки — завершивший студент (advanced/not_advanced/abandoned):**

| Текст | Код | Статус |
|-------|-----|--------|
| `📊 Мой прогресс` | `progress:progress:{streamId}` → S06 | ✅ |
| `🚪 Покинуть поток` | `learning:my-study:leave-confirm` → confirm | ✅ |
| `↩️ Главное меню` | `app:main-menu` | ✅ |

> «▶️ Продолжить» и «📂 Уроки» скрыты для завершивших — обучение окончено.

**Confirm-диалог выхода:**
```
🚪 *Покинуть поток?*

Вы уверены, что хотите покинуть поток? Это действие нельзя отменить.
```

| Текст | Код | Статус |
|-------|-----|--------|
| `🚪 Да, покинуть` | `learning:my-study:leave` → drop-student | ✅ |
| `❌ Отмена` | `learning:my-study` → хаб | ✅ |

**Результат выхода:** `👋 Вы покинули поток. Если захотите вернуться — обратитесь к ментору.`

## S05a — Просмотр шага (активный / read-only)

**Как попасть:** `▶️ Продолжить` в S05, или клик на шаг в S05b.
**Кому:** `STUDENT`.

**Содержание:**
```
📖 *Поток:* {StreamTitle}
📁 *Проект:* {ProjectTitle}
📚 *Урок:* «{LessonTitle}»
🔢 p{projectIndex}-l{lessonIndex}

――――――――――――――

📝 *Шаг {S} из {Total}:* {StepDescription}

{StepContent}

――――――――――――――

_Шаги урока:_
✅ _Уже пройденный шаг_
▶️ _Текущий шаг_
🔒 _Будущий шаг_
```

**Кнопки — активный шаг (режим «Продолжить»):**

| Текст | Код | Статус |
|-------|-----|--------|
| `✅ Выполнено` | `learning:complete:{streamId}:{stepId}` | ✅ |
| `📊 Мой прогресс` | `progress:progress:{streamId}` | ✅ |
| `↩️ Главное меню` | `app:main-menu` | ✅ |

**Кнопки — просмотр пройденного шага (read-only, из S05b):**

| Текст | Код | Условие | Статус |
|-------|-----|---------|--------|
| `◀️ Назад` | `learning:back:{streamId}:{stepId}` | скрыта на первом completed | 📋 трек 4 |
| `▶️ Вперёд` | `learning:forward:{streamId}:{stepId}` | скрыта на последнем completed | 📋 трек 4 |
| `⬅️ Назад к уроку` | `learning:my-study:lesson:{lessonId}` → S05b (шаги урока) | 📋 трек 4 |
| `↩️ Главное меню` | `app:main-menu` | ✅ |

> ◀️/▶️ листают **только completed шаги** в одном сообщении (editMessage). Текущий шаг через листание недоступен.
> Список шагов урока всегда виден внизу — студент понимает, где находится.

## S05b — Дерево навигации «📂 Уроки» (📋 трек 4)

**Как попасть:** `📂 Уроки` в S05.
**Кому:** `STUDENT`.

**UX-конвенция:** все переходы внутри дерева — **editMessage** (одно сообщение на весь сеанс). Вход из хаба и выход в хаб — sendMessage.

### Уровень 1: Проекты

```
📂 *Уроки*

Выберите проект:
```

| Текст | Код | Статус |
|-------|-----|--------|
| `📁 {ProjectTitle} ({N}/{M})` | `learning:my-study:project:{projectIndex}` | 📋 |
| `⬅️ Назад к учёбе` | `learning:my-study` → S05 (sendMessage) | 📋 |

> В скобках — прогресс: пройдено/всего уроков. Только проекты, где есть ≥1 пройденный или текущий урок.

### Уровень 2: Уроки проекта

```
📂 *Уроки* › {ProjectTitle}

Выберите урок:
```

| Текст | Код | Статус |
|-------|-----|--------|
| `📝 {LessonTitle} ({N}/{M})` | `learning:my-study:lesson:{lessonId}` | 📋 |
| `⬅️ Назад к проектам` | `learning:my-study:projects` → уровень 1 | 📋 |

### Уровень 3: Шаги урока

```
📂 *Уроки* › {ProjectTitle} › {LessonTitle}

✅ _Пройденный шаг_
▶️ _Текущий шаг_
🔒 _Будущий шаг_

Выберите шаг:
```

| Текст | Код | Условие | Статус |
|-------|-----|---------|--------|
| `✅ {StepTitle}` | `learning:view:{streamId}:{stepId}` | completed | 📋 |
| `▶️ {StepTitle}` | `learning:my-study:continue` → S05a | current | 📋 |
| `⬅️ Назад к урокам` | `learning:my-study:project:{projectIndex}` → уровень 2 | 📋 |

> 🔒-шаги — только в тексте, не кнопками. Telegram не поддерживает disabled-кнопки.

## S07 — Мониторинг (список студентов)

## S08 — Карточка студента

**Как попасть:** клик на студента в S07.
**Кому:** **всем ролям** (прозрачность — кто как учился, кого отчислили, кто каких успехов достиг). Кнопки действий — только ментору потока / админу.

**Содержание:**
```
👤 {StudentName}
📊 Статус: {StatusLabel}
📈 Прогресс: {completed} из {totalSteps} шагов ({pct}%)
📁 Проект: {ProjectTitle}      ← только если есть выданный шаг
📝 Урок: {LessonTitle}
```

`StatusLabel`: `🟢 Активен` / `🔴 Выбыл` / `✅ Прошёл` / `↩️ Не прошёл`
(соответствуют статусам `active` / `abandoned` / `advanced` / `not_advanced`).

> 📋 трек 6: история шагов студента (позиция, статус, даты выдачи/завершения), динамика обучения, причина отставания.

**Кнопки — всегда:**

| Текст | Код | Статус |
|-------|-----|--------|
| `📁 История шагов` | `monitor:history:{studentId}` | ✅ (заглушка «ещё не реализована»; полноценно — трек 6) |
| `⬅️ Назад к списку` | `monitor:students:{streamId}` | ✅ |

**Кнопки — действия ментора** (только ментор потока или админ, `StudentPolicy`):

| Текст | Код | Условие | Статус |
|-------|-----|---------|--------|
| `⚠️ Неактивен` | `monitor:mark-abandoned:{studentId}` | студент `active` | ✅ |
| `✅ Завершить` | `monitor:complete:{studentId}` | студент `active` | ✅ |
| `🔄 Сменить исход` | `monitor:complete:{studentId}` | студент `advanced`/`not_advanced` | ✅ |

> **`⚠️ Неактивен` → подтверждение:** «⚠️ Отметить *{StudentName}* как неактивного?» → `⚠️ Да, неактивен` / `❌ Отмена` → `mark-abandoned-confirm` → выполняет `mark-abandoned` (cause: `inactivity`). Работает end-to-end.

> **`✅ Завершить` / `🔄 Сменить исход` → выбор исхода:** экран «Выберите исход для студента» с кнопками `✅ Прошёл` / `↩️ Не прошёл` / `🔴 Выбыл` / `❌ Отмена` → `complete-confirm:{studentId}:{outcome}` → confirm-диалог → `complete-confirm-confirm:{studentId}:{outcome}` → выполняет `complete-student` UC. Работает end-to-end.

**Доменная логика действий:**
- `mark-abandoned` → `StudentAr.markAbandoned()` (статус `abandoned`, `abandonDetails: { who: 'mentor', cause: 'inactivity' }`), `−STUDENT`. Запись **не удаляется**.
- `complete-student` → `StudentAr.advance()` / `markNotAdvanced()` / `markAbandoned()` по выбранному исходу, `−STUDENT`; для `advanced`/`not_advanced` — TgFacade-сообщение студенту (предложение следующего шага, трек 3) — **отложено** (см. техдолг трека `module_gating_20260708`).

## ✍️ Инструменты автора (📋 ОТЛОЖЕНО — техдолг)

> Не реализуется в Релизе 2. Функциональность зафиксирована здесь, чтобы система документации явно содержала эту потребность. UC создания уже реализованы (трек `author_role_20260708`), UI-входа нет. См. `architecture-evolution.md` §2.8, §2.9.

**Ответственность (когда встанет потребность):**
- Создание/редактирование **модулей** (title, description, goal, result, rules, targetAudience, additional).
- Создание/редактирование **курсов**: phases, направления (tech/business), порядок модулей (`add-module-to-course`, `add-phase`).
- Создание/редактирование **проектов, уроков, шагов** внутри модуля (типы шагов: текст/код/файл).
- Роль `AUTHOR`. Создание — прерогатива роли; редактирование — ADMIN или автор (по `authorId`).

**Вход:** «✍️ Инструменты автора» в главном меню (role-gated `AUTHOR`). Подменю: модули / курсы / уроки и шаги.
