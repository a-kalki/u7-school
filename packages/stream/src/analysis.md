## 2. Детальные расхождения по сценариям

### US-1 — Витрина потоков
- **Проблема:** `handleListStreams` бота вызывает `list-streams` **без фильтра**, показывая все статусы. По сценарию — только `enrollment` и `active`.
- **Решение:** передавать `{ status: 'enrollment' }` или делать два запроса и объединять.

### US-2 — Карточка потока
- **Нет кнопки «Программа курса»:** требуется отображать срез `contentSnapshot` (проекты → уроки).
- **Нет ролевого разделения:** кнопка «Записаться» видна всем, включая уже зачисленных студентов и менторов.
- **Решение:** контроллер должен получать роль пользователя и условно рендерить кнопки.

### US-3 — Запись на поток
- **Нет даты старта** в сообщении об успешной записи. Название потока выводится.
- **Решение:** добавить `startDate` в ответ `handleEnroll`.

### US-4 — Прохождение обучения
- **Нет названий уроков/проектов** в сообщениях о переходе. Сейчас просто «Урок завершён!», а нужно «Урок {OldLessonTitle} завершён. Начинаем: {NewLessonTitle}».
- **Решение:** добавить `contentSnapshot` в `handleCompleteStep`, вычислять названия из контекста.

### US-5 — Прогресс
- **Не хватает данных:** имя ментора, дата старта, ссылка на чат группы, текущий проект и урок.
- **Решение:** добавить соответствующие поля в рендеринг `handleProgress`.

### US-6 — Создание потока 🔴
- **`registerStreamHandler` в `stream-handler.ts` — пустая функция.**
- **`handleCreateStream` не вызывается** из `handleUpdate` и не маршрутизируется.
- **Нет пошагового диалога** (выбор модуля → название → описание → дата → chat ID).
- **`CourseFacade` в `api-app.ts` — заглушка:** `{ getModuleSnapshot: async () => [] }`.
- **Решение:** реализовать диалог через `ctx.session.menu = 'create_stream'`, подключить реальный `CourseFacade`.

### US-7 — Запуск потока
- **Кнопка «Запустить» не интегрирована** в карточку потока для ментора. US-7 описывает карточку с кнопками `[🚀 Запустить]`, `[👥 Список]`, `[📁 В архив]`.
- **Решение:** в `handleStreamView` добавить менторские кнопки при совпадении `actorId === stream.mentorId`.

### US-8 — Мониторинг группы
- **Нет имён студентов** — показывается обрезанный `userId` вместо имени.
- **Нет кликабельности** — нельзя нажать на студента для деталей.
- **Решение:** добавить `UserFacade` в контроллер для получения имён; добавить callback `student:detail:<id>`.

## 3. Рекомендации по порядку реализации

Учитывая приоритеты пользователя:

### Этап 1: Вспомогательные сценарии (US-1, US-2)

> **Цель:** гость может увидеть потоки и изучить их.

| Задача | Файлы | Суть |
|--------|-------|------|
| 1.1. Починить фильтр в `handleListStreams` | `stream-controller.ts` | Запрашивать только `enrollment` + `active` |
| 1.2. Добавить роль в `handleStreamView` | `stream-controller.ts`, сигнатура метода | Передавать `actorId`, загружать роль, скрывать кнопки для STUDENT/MENTOR |
| 1.3. Кнопка «Программа курса» | `stream-controller.ts` | Рендерить `contentSnapshot` при клике (новый callback) |

### Этап 2: Запуск потока вручную / через CLI

> **Цель:** ментор может создать поток и запустить его без бота, чтобы студенты могли учиться.

| Задача | Файлы | Суть |
|--------|-------|------|
| 2.1. Реализовать `CourseFacade` | `packages/course` + `api-app.ts` | Настоящий сниминг контента модуля, а не заглушка |
| 2.2. CLI-команда для создания потока | `apps/u7-cli` | Обёртка над `CreateStreamUc` |
| 2.3. CLI-команда для запуска потока | `apps/u7-cli` | Обёртка над `ActivateStreamUc` |

### Этап 3: Зачисление студентов на поток (US-3)

> **Цель:** гость может записаться через бота и стать студентом.

| Задача | Файлы | Суть |
|--------|-------|------|
| 3.1. Доработать `handleEnroll` | `stream-controller.ts` | Добавить `startDate` и `StreamTitle` в сообщение |
| 3.2. Ролевая проверка при записи | `stream-controller.ts` | Скрыть кнопку «Записаться» для STUDENT/MENTOR |

### Этап 4: Запуск потока ментором через бота (US-7, US-6)

> **Цель:** ментор может создать и запустить поток полностью через интерфейс бота.

| Задача | Файлы | Суть |
|--------|-------|------|
| 4.1. Пошаговый диалог создания потока | `stream-handler.ts`, `stream-controller.ts` | `ctx.session.menu = 'create_stream'`, шаги: модуль → название → описание → дата → chat ID |
| 4.2. Менторские кнопки в карточке потока | `stream-controller.ts` | `[🚀 Запустить]`, `[👥 Студенты]`, `[📁 В архив]` для ментора |
| 4.3. Подключить `registerStreamHandler` | `stream-handler.ts` | Вынести stream-callback'и из `top-menu-handler.ts` |

### Этап 5: Завершение потока ментором (US-7 complete)

> **Цель:** ментор может завершить активный поток.

| Задача | Файлы | Суть |
|--------|-------|------|
| 5.1. Кнопка «Завершить» для active-потока | `stream-controller.ts` | Аналогично запуску, только `complete-stream` |
| 5.2. Кнопка «В архив» | `stream-controller.ts` | `archive-stream` |

## 4. Что пропущено в приоритетах?

Из полного списка User Stories **отсутствуют** в твоих приоритетах:

| US | Почему важно |
|----|-------------|
| **US-6 (Создание потока)** | Критически! Без создания потока нечего запускать. Если создание будет через CLI (этап 2), то этот US можно отложить для бота. Но `CourseFacade` нужно сделать в любом случае. |
| **US-5 (Прогресс студента)** | Вспомогательный, но студент ожидает видеть свой прогресс. Частично покрывается US-4. |
| **US-8 (Мониторинг группы)** | Нужен ментору для отслеживания успеваемости. Можно отложить до post-MVP. |

## 6. Технический долг и критические точки

| # | Проблема | Где | Влияние |
|---|----------|-----|---------|
| 1 | `registerStreamHandler` — пустая функция | `stream-handler.ts` | Все stream-callback'и висят в `top-menu-handler`, нарушает разделение ответственности |

## 7. Итог

**Фундамент готов:** domain-слой, API-слой и базовый контроллер реализованы на 80-90%. Все агрегаты, use-case'ы и репозитории спроектированы правильно.

**Основной фронт работ** — это:
1. **Интеграция бота** (подключение `registerStreamHandler`, ролевые проверки, диалог создания потока).
2. **Устранение заглушек** (`CourseFacade`, имена студентов).
3. **Доводка UX-деталей** (названия уроков/проектов в сообщениях, даты, фильтры).

Рекомендуемый порядок: **Этап 1 → Этап 2 → Этап 3 → Этап 4 → Этап 5**, как описано в разделе 3.
