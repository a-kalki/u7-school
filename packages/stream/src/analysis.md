## 2. Детальные расхождения по сценариям

### US-1 — Витрина потоков (CatalogStory)
**Файл:** `ui/bot/stories/catalog.story.ts`

**Расхождение:** `handleCallback('list')` вызывает `list-streams` **без фильтра по статусам**. По US нужно показывать только `enrollment` и `active`.

**Решение:** передавать `{ status: 'enrollment' }` или делать два запроса и объединять результаты. Либо добавить поддержку фильтра `status: ['enrollment', 'active']` в `ListStreamsCmd`.

---

### US-2 — Детальная карточка потока (ViewStreamStory)
**Файл:** `ui/bot/stories/view-stream.story.ts`

**Расхождения:**
1. **Нет ролевого разделения кнопок.** Кнопка «Записаться» показывается для всех ролей при `enrollment`. По US:
   - Для GUEST/CANDIDATE на `enrollment`: `[📝 Записаться]`, `[📖 Программа курса]`, `[⬅️ Назад]`
   - Для GUEST/CANDIDATE на `active`: `[🔔 Уведомить о наборе]`, `[⬅️ Назад]`
   - Для STUDENT/MENTOR: кнопка «Записаться» скрывается
   - Для MENTOR на своём потоке: `[🚀 Запустить]` (enrollment), `[👥 Студенты]`, `[✅ Завершить]` (active), `[📁 В архив]`
2. **Нет кнопки «Программа курса»** — не выводится `contentSnapshot`.
3. **Нет имени ментора** — поле `mentorId` есть в данных, нужно резолвить через `UserFacade`.

**Решение:**
- Передавать `actor` в `handleCallback` (уже доступен, тип `unknown` — нужно кастить).
- По роли актора и `actorId === stream.mentorId` вычислять набор кнопок.
- Для кнопки «Программа курса» — новый экшен `view-stream:program:<streamId>`, рендерит `contentSnapshot`.

---

### US-3 — Запись на поток (EnrollStory)
**Файл:** `ui/bot/stories/enroll.story.ts`

**Расхождение:**
1. **Нет даты старта** в сообщении об успешной записи. По US: «Обучение начнётся {StartDate}».

**Решение:** добавить `startDate` из ответа `get-stream` в сообщение.

---

### US-4 — Прохождение обучения (LearningStory)
**Файл:** `ui/bot/stories/learning.story.ts`

**Расхождения:**
1. **Нет названий уроков/проектов в переходах.** Сейчас сообщения шаблонные:
   - `step` → «✅ Шаг выполнен! Следующее задание уже ждёт.»
   - `lesson` → «🎉 Урок завершён! Отличная работа!»
   - `project` → «🚀 Проект завершён! ...»
   По US нужно: «Урок {OldLessonTitle} завершён. Начинаем: {NewLessonTitle}».
2. **Нет навигации ◀️/▶️** по истории шагов. По US студент может листать назад/вперёд.
3. **Нет подтверждения** (double-confirm) перед завершением шага.

**Решение:**
- Для названий: после `complete-step` делать ещё один `get-stream` и вычислять названия из `contentSnapshot`.
- Или расширить выход `CompleteStepUc`, чтобы возвращал `completedLessonId`/`completedProjectId` (уже возвращает для `lesson`/`project`), и добавить названия.
- Навигация: новый экшен `learning:back:<studentId>:<streamId>:<stepId>` и `learning:forward:...`.

---

### US-5 — Прогресс студента (ProgressStory)
**Файл:** `ui/bot/stories/progress.story.ts`

**Расхождения:**
1. **Нет имени ментора, даты старта, ссылки на чат.** По US: «Ментор: {MentorName}», «Дата старта: {StartDate}», «Учебный чат: [ссылка]».
2. **Нет текущего проекта и урока.** По US: «Текущий проект и урок».

**Решение:** добавить поля в рендеринг. Для имени ментора нужен `UserFacade` (запросить пользователя по `stream.mentorId`).

---

### US-6 — Создание потока (CreateStreamStory) 🔴
**Файл:** `ui/bot/stories/create-stream.story.ts`

**Проблема:** Wizard-диалог реализован, НО:
1. **`CourseFacade` — заглушка** в `u7-bot/src/api-app.ts`:
   ```ts
   courseFacade: { getModuleSnapshot: async () => [] },
   ```
   При этом в `packages/course/src/infra/course-in-proc-facade.ts` есть **реальная реализация** `CourseInProcFacade`. Нужно просто подключить её.
2. **Шаг выбора модуля** (`#handleModuleMessage`) возвращает заглушку с одной кнопкой «Обновить список» — не показывает реальные модули ментора.
3. **Нет превью перед созданием** (US требует подтверждение с кнопками «Всё верно, создать» / «Изменить данные»).

**Решение:**
- Подключить `CourseInProcFacade` в `api-app.ts` (нужны lessonRepo + moduleRepo из course).
- В `#handleModuleMessage` запрашивать модули через API (`list-modules` с фильтром по автору) и выводить inline-кнопки.
- Добавить шаг 6: превью + подтверждение.

---

### US-7 — Запуск потока (ActivateStreamStory)
**Файл:** `ui/bot/stories/activate-stream.story.ts`

**Расхождение:**
1. **Кнопка «Запустить» не интегрирована** в `ViewStreamStory`. Ментор не видит кнопку в карточке потока.

**Решение:** в `ViewStreamStory` добавить менторские кнопки при совпадении `actorId === stream.mentorId`.

---

### US-8 — Мониторинг группы (MonitorStory)
**Файл:** `ui/bot/stories/monitor.story.ts`

**Расхождения:**
1. **Показывается `userId` вместо имени студента.**
2. **Нет кликабельности** — нельзя нажать на студента для деталей.
3. **Нет детальной карточки студента** (имя, Telegram, статус, история шагов, кнопки управления).
4. **Нет логики «Отстает»** (5+ дней без активности или отставание на 30%).

**Решение:**
- `UserFacade.getUserByUuid()` для получения имён студентов.
- Добавить callback `monitor:detail:<studentId>` для детальной карточки.
- Реализовать детальную карточку с кнопками: «Написать», «История шагов», «Назад к списку».

## 3. Инфраструктурные проблемы

| # | Проблема | Где | Влияние |
|---|----------|-----|---------|
| 1 | `CourseFacade` — заглушка в `api-app.ts` | `apps/u7-bot/src/api-app.ts:55` | US-6 не видит модули, `CreateStreamUc` создаёт потоки с пустым `contentSnapshot` |
| 2 | `ViewStreamStory` не получает роль актора | `view-stream.story.ts` | Нет ролевых кнопок (US-2, US-7) |
| 3 | `MonitorStory` не имеет доступа к `UserFacade` | `monitor.story.ts` | Показывает userId вместо имён (US-8) |
| 4 | `ProgressStory` не имеет доступа к `UserFacade` | `progress.story.ts` | Нет имени ментора (US-5) |
| 5 | Тесты сторис падают (13/124) | `*.story.test.ts` | `story.init(mockApi)` не вызван в ряде тестов |

**Примечание к п.2-4:** `BotUserStory` имеет `this.api` (доступ к API-модулю), но НЕ имеет доступа к `UserFacade` напрямую. Варианты решения:
- (А) Добавить фасады в API-модуль как use-case'ы (например, `get-user-name`).
- (Б) Пробросить `UserFacade` в `StreamApiModuleResolver` и дать доступ через `this.resolve` в сторис.
- (В) Сделать составные use-case'ы, которые возвращают сразу все нужные данные (например, `get-stream-view` возвращает поток + имя ментора + список студентов с именами).

**Рекомендация:** вариант (В) — наиболее чистый архитектурно. Но для скорости можно пойти по пути (Б) или (А).
