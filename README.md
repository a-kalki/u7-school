# u7-school

Образовательная платформа, направленная на подготовку специалистов через курсы и личное менторство. Проект предоставляет инфраструктуру для менторов (создание и управление контентом) и студентов (прохождение обучения).

## О проекте

u7-school — это современная LMS (Learning Management System), построенная на принципах DDD (Domain-Driven Design). Основная цель проекта — обеспечить удобную среду для обмена знаниями, автоматизируя рутинные процессы управления курсами, уроками, пользователями и onboarding-заявками.

## Модули проекта

Проект организован как монорепозиторий на базе Bun Workspaces:

### Приложения (`apps/`)
- **[u7-cli](./apps/u7-cli/)**: Интерактивный CLI для работы с приложением.
- **[u7-bot](./apps/u7-bot/)**: Telegram-бот для студентов и менторов. Реализует витрину потоков, запись, прохождение обучения, панель ментора.

### Пакеты (`packages/`)
- **[core](./packages/core/)**: Ядро платформы. Содержит базовые абстракции DDD (`Aggregate`, `UseCase`, `ApiModule`), UI-компоненты для ботов (`BotController`, `BotUserStory`, `BotRouter`) и общие утилиты.
- **[app](./packages/app/)**: Связующий слой приложения. Определяет `U7BotController`, `U7BotUserStory`, общие типы (`User`, `Role`, `U7BotAppMeta`) для Telegram-бота.
- **[user](./packages/user/)**: Модуль управления пользователями (роли, авторизация, профили, `UserFacade` для других модулей).
- **[onboarding](./packages/onboarding/)**: Модуль сбора заявок от кандидатов. Управление onboarding-анкетами, пуллом кандидатов и автоматическим присвоением роли CANDIDATE.
- **[course](./packages/course/)**: Конструктор учебного контента. Управление модулями, проектами, уроками и шагами. Предоставляет `CourseFacade` для получения снимков контента.
- **[stream](./packages/stream/)**: Жизненный цикл потоков обучения. Запись на потоки, выдача шагов, отслеживание прогресса студентов, мониторинг группы ментором.
- **[w3school](./packages/w3school/)**: Контент курсов и уроков из ресурса W3Schools. Позволяет планировать уроки ведущие на этот ресурс, все данные находятся в папке `output`.
- **[video-summarizer](./packages/video-summarizer/)**: Утилита для суммаризации видео-контента (экспериментальный).

## Роли пользователей

Платформа поддерживает иерархию ролей с чётким разделением прав:

| Роль | Описание | Ключевые права |
|---|---|---|
| **GUEST** | Гость, только что начавший взаимодействие с ботом | Минимальный доступ, может стать подписчиком или кандидатом |
| **SUBSCRIBER** | Подписчик Telegram-канала школы | Получает анонсы, может подать заявку на обучение |
| **CANDIDATE** | Кандидат, заполнивший onboarding-анкету | Ожидает зачисления в поток, может обновить свои ответы |
| **STUDENT** | Ученик, проходящий обучение | Доступ к материалам курсов, прохождение уроков |
| **MENTOR** | Наставник, создающий курсы и управляющий потоками | Создание и управление модулями, проектами, уроками; создание, запуск и мониторинг потоков |
| **ADMIN** | Администратор платформы | Полный доступ: управление пользователями, ролями, курсами, потоками, заявками |

## Подготовка к запуску

### Требования
- [Bun](https://bun.sh/) (последняя стабильная версия)

### Установка
1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd u7-school
   ```
2. Установите зависимости:
   ```bash
   bun install
   ```
3. Настройте переменные окружения:
   Создайте файл `.env` в корне проекта:
   ```env
   # ══ Обязательные для u7-bot ══
   BOT_TOKEN=your_telegram_bot_token
   BOT_ADMIN_UUID=uuid_пользователя_с_ролью_ADMIN
   NEWS_GROUP_URL=https://t.me/your_school_group

   # ══ Опциональные для u7-bot ══
   BOT_MODE=polling              # polling (по умолчанию) или webhook
   WEBHOOK_URL=https://...        # только для webhook
   WEBHOOK_PATH=/webhook          # путь webhook (по умолчанию)
   WEBHOOK_PORT=8443              # порт webhook (по умолчанию)
   DB_DIR=./data                  # директория JSON-хранилищ (по умолчанию)
   ADMIN_TELEGRAM_IDS=123,456     # ID админов для логов в Telegram
   LOGGER_BOT_TOKEN=...           # токен отдельного бота для логов

   # ══ API ключи для ИИ (если используется w3school) ══
   GEMINI_API_KEY=your_gemini_key
   DEEPSEEK_API_KEY=your_deepseek_key
   ```
4. Начальный пользователь с правами admin:
    Создайте файл `data/users/users.json` с содержимым:
    ```json
    [
      {
        "uuid": "39549ec2-68c3-4bfe-8594-6e9874d418cd",
        "name": "Fox",
        "telegramId": 123456789,
        "roles": ["ADMIN", "MENTOR"],
        "createdAt": "2026-05-13T12:56:00Z"
      }
    ]
    ```
    Замените `uuid`, `name`, `telegramId` на свои. Укажите этот же `uuid` в `BOT_ADMIN_UUID`.

## Развертывание и работа

### Telegram-бот (`u7-bot`)

#### Запуск бота
```bash
cd apps/u7-bot
bun start
```
Бот запустится в режиме polling (по умолчанию) или webhook (если `BOT_MODE=webhook`).

Бот обслуживает:
- **Гостей** — витрина потоков, карточки, запись на обучение.
- **Студентов** — прохождение шагов, просмотр прогресса.
- **Менторов** — создание и запуск потоков, мониторинг группы.

### CLI (`u7-cli`)

#### Запуск CLI
Для управления пользователями и контентом через терминал:
```bash
cd apps/u7-cli
bun start
```
Интерфейс предложит выбрать доступные действия (UseCase) в зависимости от ваших прав доступа. При первом запуске система автоматически инициализирует необходимые данные (создает JSON-файлы в папке `data/` если используются JSON-репозитории).

#### Тестирование и проверка кода
```bash
bun test        # Запуск всех тестов
bun run lint    # Проверка линтером (Biome)
bun run tslint  # Проверка типов (TSC)
bun run check   # Полная проверка (lint + tslint + test)
```
