# u7-school

Образовательная платформа, направленная на подготовку специалистов через курсы и личное менторство. Проект предоставляет инфраструктуру для менторов (создание и управление контентом) и студентов (прохождение обучения).

## О проекте

u7-school — это современная LMS (Learning Management System), построенная на принципах DDD (Domain-Driven Design). Основная цель проекта — обеспечить удобную среду для обмена знаниями, автоматизируя рутинные процессы управления курсами, уроками, пользователями и onboarding-заявками.

## Модули проекта

Проект организован как монорепозиторий на базе Bun Workspaces:

### Приложения (`apps/`)
- **[u7-cli](./apps/u7-cli/)**: Интерактивный CLI для работы с приложением.
- **[u7-bot](./apps/u7-bot/)**: Telegram-бот для студентов (в процессе разработки).

### Пакеты (`packages/`)
- **[core](./packages/core/)**: Ядро платформы. Содержит базовые абстракции DDD, логику Auto-UI и общие утилиты.
- **[user](./packages/user/)**: Модуль управления пользователями (роли, авторизация, профили).
- **[onboarding](./packages/onboarding/)**: Модуль сбора заявок от кандидатов. Управление onboarding-анкетами, пуллом кандидатов и автоматическим присвоением роли CANDIDATE.
- **[course](./packages/course/)**: Основной домен обучения. Управление курсами, модулями, проектами, уроками и шагами.
- **[w3school](./packages/w3school/)**: Контент курсов и уроков из ресурса W3Schools. Позволяет планировать уроки ведущие на этот ресурс, все данные находятся в папке `output`.

## Роли пользователей

Платформа поддерживает иерархию ролей с чётким разделением прав:

| Роль | Описание | Ключевые права |
|---|---|---|
| **GUEST** | Гость, только что начавший взаимодействие с ботом | Минимальный доступ, может стать подписчиком или кандидатом |
| **SUBSCRIBER** | Подписчик Telegram-канала школы | Получает анонсы, может подать заявку на обучение |
| **CANDIDATE** | Кандидат, заполнивший onboarding-анкету | Ожидает зачисления в поток, может обновить свои ответы |
| **STUDENT** | Ученик, проходящий обучение | Доступ к материалам курсов, прохождение уроков |
| **MENTOR** | Наставник, создающий курсы | Создание и управление курсами, проектами, уроками |
| **ADMIN** | Администратор платформы | Полный доступ: управление пользователями, курсами, заявками |

## Подготовка к запуску

### Требования
- [Bun](https://bun.sh/) (последняя стабильная версия)

### Установка
1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd u7-school
   ```
2. Установите зависимости:
   ```bash
   bun install
   ```
3. Настройте переменные окружения:
   Создайте файл `.env.production` (или `.env`) в корне проекта:
   ```env
   # API ключи для ИИ (если используется w3school пакет)
   GEMINI_API_KEY=your_gemini_key
   DEEPSEEK_API_KEY=your_deepseek_key
   ```
4. Начальный пользователь с правами admin:
    - Создайте файл `data/user/users.json`;
    - введите такие данные;
```json
[
  {
    "uuid": "39549ec2-68c3-4bfe-8594-6e9874d418cd",
    "name": "Fox",
    "telegramId": 123456789,
    "roles": [
      "ADMIN",
      "MENTOR"
    ],
    "createdAt": "2026-05-13T12:56"
  }
]
```

Замените данные на ваши соблюдая валидность данных.

## Развертывание и работа

### Через пакет `u7-cli`.

#### Запуск CLI
Для управления пользователями и контентом выполните:
```bash
cd apps/u7-cli
bun start
```
Интерфейс предложит выбрать доступные действия (UseCase) в зависимости от ваших прав доступа. При первом запуске система автоматически инициализирует необходимые данные (создает JSON-файлы в папке `data/` если используются JSON-репозитории).

#### Тестирование и проверка кода
```bash
bun test        # Запуск всех тестов
bun run lint    # Проверка линтером (Biome)
bun run tslint  # Проверка типов (TSC)
bun run check   # Полная проверка (lint + tslint + test)
```
