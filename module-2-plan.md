### Как реализовать мутабельные функции массивов без стандартного API

Базовый примитив — собственная `len(arr)`, которая считает элементы до первого `undefined`:

```js
function len(arr) {
  let count = 0;
  for (let i = 0; arr[i] !== undefined; i++) count++;
  return count;
}
```

Имея `len`, все мутабельные функции реализуются через прямое обращение по индексам и `delete`:

```js
// push: вставить в конец
function push(arr, item) {
  const length = len(arr);
  arr[length] = item;        // JS сам увеличит внутренний .length
  return length + 1;
}

// pop: удалить с конца
function pop(arr) {
  const length = len(arr);
  if (length === 0) return undefined;
  const item = arr[length - 1];
  delete arr[length - 1];    // «дырка» → arr[last] === undefined → len() остановится
  return item;
}

// unshift: вставить в начало (сдвиг всех элементов вправо)
function unshift(arr, item) {
  const length = len(arr);
  for (let i = length; i > 0; i--) arr[i] = arr[i - 1];
  arr[0] = item;
  return length + 1;
}

// shift: удалить из начала (сдвиг всех элементов влево)
function shift(arr) {
  const length = len(arr);
  if (length === 0) return undefined;
  const item = arr[0];
  for (let i = 0; i < length - 1; i++) arr[i] = arr[i + 1];
  delete arr[length - 1];
  return item;
}
```

## Подход к TDD в модуле

### Проект 1, реализация `len()` — детальный TDD

В первом проекте студент изучает TDD на практике: каждый шаг урока ведёт через цикл red → green → refactor. Урок с реализацией `len(str)` полностью посвящён процессу TDD. Текстовые объяснения подробно описывают процесс.

### Остальные реализации — текстовые тест-кейсы

Начиная со второй реализации первого проекта (`isEqual()`), TDD уже знаком студенту. Поэтому шаги дают:

- Текстовое описание задачи: «Реализовать функцию `fn(args)`. Функция принимает ..., возвращает ...»
- Текстовые тестовые кейсы: «Должен вернуть X для входных данных Y», «Должен выкинуть исключение если аргумент не string»
- Студент сам пишет тесты и реализацию
- Предусмотреть базовые и граничные тестовые случаи. Студент должен начинать чувствовать опасные тестовые места

Формат описания функции:

```
Реализовать функцию fn(args).

Функция принимает:
  - arg1: тип — описание

Функция возвращает: тип — описание

Тестовые случаи:
  - Должен вернуть X для входных данных Y
  - Должен вернуть Z для граничного случая W
  - Должен выкинуть исключение TypeError если arg1 не string
```

  - Должен вернуть пустой массив если ни один элемент не прошёл

#### Урок 7.4 — Функция some()

**Цель:** реализовать проверку «хотя бы один».

**Содержание:**
- Текстовое описание задачи и тест-кейсов:
  - Должен вернуть true если хотя бы один элемент удовлетворяет callback
  - Должен вернуть false если ни один не удовлетворяет
  - Должен вернуть false для пустого массива
  - Должен остановить обход при первом true (ранний выход)

#### Урок 7.5 — Функция every()

**Цель:** реализовать проверку «все».

**Содержание:**
- Текстовое описание задачи и тест-кейсов:
  - Должен вернуть true если все элементы удовлетворяют callback
  - Должен вернуть false если хотя бы один не удовлетворяет
  - Должен вернуть true для пустого массива
  - Должен остановить обход при первом false (ранний выход)

#### Урок 7.6 — PR и ревью

**Содержание:** пуш ветки, создание PR, приглашение ревьюеров, pin-сообщение.

### Проект 8: «Итеративные методы: find, reduce, flat»

**Цель:** реализовать продвинутые итеративные методы: поиск, свёртку, разглаживание.

**Результат:** PR с 4 функциями.

**Функции (4):** `find(arr, callback)`, `findIndex(arr, callback)`, `reduce(arr, callback, initialValue)`, `flat(arr, depth)`.

> **Примечание:** `flat` перенесён из проекта 6 — он логически ближе к reduce (оба про рекурсию и накопление).

#### Урок 8.1 — Функции find() и findIndex()

**Цель:** реализовать поиск первого подходящего элемента/индекса.

**Содержание:**
- `find` — вернуть первый элемент, для которого callback → true. Если не найден → undefined
- `findIndex` — вернуть индекс первого подходящего элемента. Если не найден → -1
- Тест-кейсы: найден, не найден, пустой массив

#### Урок 8.2 — Функция reduce()

**Цель:** реализовать свёртку массива в одно значение.

**Содержание:**
- Текстовое описание задачи и тест-кейсов:
  - Должен свести массив к одному значению через аккумулятор
  - Callback получает (аккумулятор, элемент, индекс, массив)
  - Если initialValue передан — аккумулятор = initialValue, обход с первого элемента
  - Если initialValue не передан — аккумулятор = первый элемент, обход со второго
  - Должен выкинуть ошибку если массив пуст и initialValue не передан
- Пример: сумма чисел, конкатенация строк, группировка

#### Урок 8.3 — Функция flat()

**Цель:** реализовать разглаживание вложенных массивов рекурсивно.

**Содержание:**
- Текстовое описание задачи и тест-кейсов:
  - Должен разгладить массив на указанную глубину (depth)
  - depth = 1: разгладить один уровень
  - depth = 2: разгладить два уровня
  - depth = Infinity: разгладить полностью
  - Должен вернуть новый массив
  - Пустой массив → пустой массив

#### Урок 8.4 — Финальный прогон: все функции

**Цель:** запустить тесты для всех реализованных функций.

**Содержание:**
- Полный прогон тестов: все функции из проектов 1–8
- Исправление найденных ошибок
- Проверка совместимости функций друг с другом

#### Урок 8.5 — PR и ревью

**Содержание:** пуш ветки, создание PR, приглашение ревьюеров, pin-сообщение.

### Проект 9: «Бизнес-утилиты»

**Цель:** применить все полученные навыки в реальных бизнес-задачах.

**Результат:** финальный PR с 5 бизнес-функциями, полный набор из ~51 функции проходит все тесты.

**Функции (5):** `getQueryParams(url)`, `parseUrl(url)`, `toCSV(data)`, `fromCSV(csvString)`, `groupBy(array, key)`.

#### Урок 9.1 — Функция getQueryParams(url)

**Цель:** реализовать разбор query-строки URL.

**Содержание:**
- Текстовое описание задачи и тест-кейсов:
  - Должен разобрать query-строку в объект
  - Ключи и значения разделены `=`, пары разделены `&`
  - Должен вернуть пустой объект если query-строки нет
  - Должен обрабатывать URL-кодирование (%20, %D0 и т.д.)

#### Урок 9.2 — Функция parseUrl(url)

**Цель:** реализовать разбор URL на составные части.

**Содержание:**
- Текстовое описание задачи и тест-кейсов:
  - Должен выделить протокол, хост, порт, путь, query, хэш
  - Должен работать с https и http
  - Должен обрабатывать URL без пути
  - Должен обрабатывать URL с портом

#### Урок 9.3 — Функция toCSV(data)

**Цель:** реализовать преобразование массива объектов в CSV-строку.

**Содержание:**
- Текстовое описание задачи и тест-кейсов:
  - Должен преобразовать массив объектов в CSV
  - Первая строка — заголовки (ключи первого объекта)
  - Должен экранировать значения с запятыми (обернуть в кавычки)
  - Должен экранировать кавычки внутри значений (удвоение)

#### Урок 9.4 — Функция fromCSV(csvString)

**Цель:** реализовать парсинг CSV-строки в массив объектов.

**Содержание:**
- Текстовое описание задачи и тест-кейсов:
  - Должен разобрать CSV-строку в массив объектов
  - Первая строка — заголовки (ключи объектов)
  - Должен обрабатывать экранированные запятые и кавычки
  - Должен быть обратным к toCSV (round-trip)

#### Урок 9.5 — Функция groupBy(array, key)

**Цель:** реализовать группировку массива объектов по ключу.

**Содержание:**
- Текстовое описание задачи и тест-кейсов:
  - Должен сгруппировать объекты по значению указанного ключа
  - Результат — объект, где ключи — значения группировки, значения — массивы объектов
  - Должен работать с числами и строками как ключами группировки
  - Должен вернуть пустой объект для пустого массива

#### Урок 9.6 — Финальный PR и полный прогон

**Цель:** завершить модуль — запустить все тесты, отправить последний PR.

**Содержание:**
- Полный прогон всех функций из проектов 1–9
- Исправление ошибок
- Финальный PR с бизнес-утилитами
- Просмотр истории всего репозитория через `git log`

## Правила создания уроков и шагов

См. `conductor/guides/lesson-design.md` — полные правила.

Ключевые моменты для этого модуля:

1. **Папки уроков:** `p<номер_проекта>-l<номер_урока>-<kebab-name>` в `data/fullstack-js/m2-algorithm/`
2. **Видео-ссылки:** `[p<проект>-l<урок>. Название.mp4](...)`
3. **Первый шаг урока:** ссылка на внешний ресурс (видео/статья) с кратким описанием
4. **Последующие шаги:** самостоятельный разбор (вопросы, код)
5. **Практические шаги:** текстовое описание задачи + тест-кейсы, без готового кода
6. **Менторские файлы:** 20–30 строк, 10–15 console.log, без вопросов-подсказок в коде
7. **JSDoc:** обязателен для всех функций начиная с проекта 4

### Формат описания функции (проекты 1–9, начиная с isEqual)

```
Реализовать функцию fn(args).

Функция принимает:
  - arg1: тип — описание

Функция возвращает: тип — описание

Тестовые случаи:
  - Должен вернуть X для входных данных Y
  - Должен вернуть Z для граничного случая W
  - Должен выкинуть исключение TypeError если arg1 не string
```
